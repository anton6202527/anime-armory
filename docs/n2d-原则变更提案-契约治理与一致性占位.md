# n2d 原则变更提案：契约层治理 + 一致性 / 占位（三件套）

> 状态：**提案（待决）** · 日期：2026-06-08 · 适用：novel2drama / n2d 全线
> 性质：这是**原则层变更提案**，不是实现。每条给「现行原则 → 问题 → 提议新原则 → 影响面 → 分阶段迁移 → 风险缓解 → 待决点」。

## 0. 摘要：三件套 + 依赖关系

| 提案 | 一句话 | 性质 |
|---|---|---|
| **一·契约层治理** | 契约项区分 `invariant`（可硬化）/ `contested`（保持 soft）；版本升级必带迁移函数 | **元治理**——决定 二/三 怎么落 |
| **二·一致性杠杆** | 一致性主力从「prompt 工艺」转向「后端原生角色原语 + 多抽择优」，重审 Codex-only 垄断 | 降一致性税 / 返工 |
| **三·配音先行自洽** | 「配音先行 vs 占位放行」两条互斥铁律 → 诚实化为「rough → refine 两遍制」 | 降时长返工 |

**关键依赖**：提案一**必须先行**（哪怕只做它的阶段0）。因为提案二、三要改的恰是「Codex-only 垄断」「占位驱动付费」——而这两条**正在被契约层（`COSTLY_HINTS` / `PRODUCTION_MODES` + gate BLOCK）硬化成既成事实**。先不把它们进一步焊死，二/三的落地成本才不会翻倍。

> **已定决策（2026-06-08）**：**二a = 先解锁仍默认 Codex**（后端成为选择点，换默认留到阶段2 用数据验证）· **三a = A 方案（rough→refine 两遍制）** · **生图后端持久化 = 档2「每次出图前弹完整后端菜单」**，配套「中途换后端触发跨集一致性 gate 守护」防漂 · **一a 新增 contested：`cinematic_contract` 写实画风**（美学=contested/derive from global_style，工艺=invariant；拆分见 P1.5；**一b=改名独立**「摄影质感契约」、光/色/运镜归 visual 去重叠 · **二b=select_best 全自动取最优**（floor 守底+与生成粒度停审正交））。其余决策点（三b / 三c / 落地）见文末决策表，仍待决。

**共同的阶段0 是零行为变化的**：标注 contested + 解除原则锁 + 加水位线，行为不变、随时可回退。建议三个阶段0 一起做，先让架构自洽，再按数据推进。

## 提案一 — 契约层治理：区分 invariant / contested，不硬化待决原则

### 背景
`skills/common/n2d_contract.py`（commit 397936f）抽出了机器契约层，是真进步：阶段图 / 列名 / gate stage / 回退 / 产物快照终于有了单一真值源。本提案不否定它，而是给它**加一条治理原则**，避免它把好事做过头。

### 现行原则（要补的缺口）
- 契约层把**所有**生产规则一视同仁地编码：`COSTLY_HINTS["出图"]="…先确认 Codex only…"`、`PRODUCTION_MODES` 收录「先出视频后配音」为一等模式、gate 把「非 Codex 后端」判 BLOCK。
- `CONTRACT_VERSION = 1` / `schema_version = 1` 引入了版本号，但**没有任何迁移函数**，在产作品的 `_进度.md` / `storyboard.json` / 旧 manifest 都早于它。

### 问题
1. **过早承诺**：契约把**仍在争论的设计决策**（生图后端垄断、占位能否驱动付费）编码成既成事实，赋予它们**虚假权威**（"它在契约里，所以对"）。一旦想改（提案二/三），blast radius 从「改几句 SKILL」放大到「改契约 + 闸门 + 多处复述」。
2. **版本号无迁移 = 改不动**：`CONTRACT_VERSION` 一旦 bump 到 2（如提案二/三、或列名调整），旧作品集体破。几十部在产剧的"工厂"，schema 演进没有 migration 等于 schema 永久冻结。

### 提议新原则
- **P1.1 契约项分层标注 `status`**：每个契约项（铁律 / COSTLY_HINTS / 模式 / gate BLOCK）显式标 `invariant`（已定不变量，可硬化）或 `contested`（待决原则，保持 soft）。
  - **invariant 示例**：阶段顺序、`_进度.md` 列名、产物路径、接力契约「出点=下一入点」、合规闸门（克隆授权 / AI 标识水印）、prompt/产物分离。
  - **contested 示例（当前应标）**：① **生图后端垄断**（"图必须 Codex"——见提案二）② **占位驱动付费生成**（"先出视频后配音"作为一等路径——见提案三）③ **真实电影感画风**（`cinematic_contract` 默认写实低饱和 + 禁插画/游戏CG——见 P1.5）。
- **P1.2 contested 项只能进 choice point，禁止进 BLOCK / "必须"措辞**：`gate.py` 不得对 contested 原则判 BLOCK（最多 WARN）；`COSTLY_HINTS` / SKILL 铁律不得用"必须 / 只能 / 不可选"措辞描述 contested 项。把"对错判断"降级成"选择 + 知情权衡"。
- **P1.3 立即把两条待决原则改标 contested**：在你对提案二/三拍板**之前**，先把 Codex-only 与 占位-驱动-付费 从 invariant 语气降为 contested——**这是本三件套的最高优先动作**，因为它解除原则锁、让后续提案便宜落地。
- **P1.4 版本治理：bump 必带迁移**：`CONTRACT_VERSION` 每升一级，配一个 `migrate_v{N}_to_v{N+1}(work_root)`；`n2d-progress` / 路由脚本启动时检测 work 的 `schema_version` 落后即提示（可选自动迁移）。**版本号不配迁移函数等于没版本号。**
- **P1.5 首个 contested-硬化纠偏案例：`cinematic_contract`（真实电影感契约）拆分**。新近落地的 `cinematic_contract` 正是 P1 要防的反模式现行实例——它把**「写实电影感」这一画风**（默认 `摄影基调=写实低饱和胶片颗粒`、`真实感禁忌=插画感/游戏CG/过饱和霓虹`，`formats.md:147/152`）硬化进**契约 + BLOCK gate（`gate.py:259/312`）+ 负向词**，撞国漫 / 二次元 / Q版 / 水墨 / 赛博霓虹等合法画风。**注意**：n2d_contract 登记字段、gate 从契约读字段、GATE_RECOVERY 同步——"管道形式化"做对了；错在**把一种画风当成全线 invariant**。纠偏三步：
  - **(a) 拆 style-agnostic 工艺 vs per-project 美学**：① **工艺**（光源有动机 / 焦段可信 / 运镜克制 / 禁塑料·过锐·无来源强光·背景贴图）——任何画风都成立，留近 invariant；② **美学**（摄影基调 / 色彩饱和 / 是否胶片颗粒）——**contested，真值 derive from `global_style.md`**（已存在的画风真值源），不默认写实、不对非写实画风 BLOCK。
  - **(b) 改名 + 解 BLOCK + 改禁忌语义**：契约去掉"真实"预设、**改名「本集摄影质感契约」**（一b 决策=改名独立，与 visual 并存、不合并）；gate 由"必须叫真实电影感契约且全字段在场" → "工艺字段在场 + 美学字段与 `global_style` 不冲突"；`真实感禁忌` → `画风一致性禁忌（违背本项目 global_style 的元素）`。
  - **(c) 去重叠（一b=改名独立，靠裁剪而非合并）**：光（光位锚↔光源动机）、色（色调基线↔色彩策略）、运镜（起幅余量·景别阶梯↔运镜边界）现两契约**各有一份**，违背"单变量单一真值"。既不合并，就靠**裁剪**去重叠——**光 / 色 / 运镜交还 `visual_contract` 唯一持有**；改名后的「摄影质感契约」只留 `visual_contract` 不覆盖的**增量**：镜头焦段 / 景深纪律、材质 / 反塑料反过锐、以及由 `global_style.md` 决定的摄影基调。两契约**职责正交、字段不重叠**。

### 影响面
| 改哪 | 怎么改 |
|---|---|
| `skills/common/n2d_contract.py` | 契约项加 `status` 维度；新增 `CONTESTED` 集合；`migrate_*` 框架 + `schema_version` 检测 |
| `skills/common/n2d_contract.py` `COSTLY_HINTS` | "出图"项措辞从"确认 Codex only"改为"确认生图后端（默认 Codex，可选更强一致性后端）" |
| `gate.py --stage image` | 非 Codex 后端从 BLOCK 降为「混用才 BLOCK / 逆向 CLI 才 BLOCK」（与提案二阶段0 一致） |
| `references/contract.md` | 补「invariant vs contested 治理原则」+「版本迁移原则」两节 |
| 各 SKILL「不可选/硬规则」措辞 | contested 项改为选择点语气 |
| `n2d_contract.py` `CINEMATIC_CONTRACT_FIELDS` | 拆 style-agnostic 工艺字段 vs per-project 美学字段；美学字段标 contested（P1.5a） |
| `gate.py` cinematic 检查（`check_storyboard/markdown_cinematic_contract`） | 美学字段不再硬 BLOCK；改查「工艺字段在场 + 美学与 global_style 不冲突」（P1.5b） |
| `n2d-script/formats.md` + `n2d-image/n2d-video prompt_format.md` | cinematic 种子去写实预设、美学 derive from `global_style.md`、`真实感禁忌`→`画风一致性禁忌`；目标合并 visual+cinematic 为「视听契约」（P1.5b/c） |

### 迁移路径
1. **阶段0（纯文档/零行为变化）**：加 `status` 维度 + 标注 contested 项 + 写迁移框架（空壳）。**不改任何判定**，只解除"语气锁"。
2. **阶段1**：把 contested 项的 gate BLOCK 降级为 WARN/choice（与提案二/三阶段0 合并做）。
3. **阶段2**：首次需要 schema 演进时落地第一个 `migrate_v1_to_v2`。

### 风险与缓解
- **过度标 contested → 失去约束力**：只标**确实有公开提案在争**的两项；其余默认 invariant。治理原则是"先证明有争议再降级"，不是"默认都软"。
- **可逆**：阶段0 纯标注，随时回退。

### 待决点
- (a) 除 Codex-only / 占位驱动付费外，你还认为哪些现行"铁律"其实是 contested？（如「即梦只能做视频后端」「LoRA 仅核心角色」）
  - **已答（2026-06-08）**：**`cinematic_contract` 的写实画风**确认为 contested（见 P1.5）——它的美学部分入 contested 清单、工艺部分留 invariant。其它候选继续征集。
- (b) cinematic 纠偏的目标态 — **已答（2026-06-08）：改名独立**（「摄影质感契约」与 visual 并存），去重叠靠裁剪：光/色/运镜归 visual 唯一持有，摄影质感契约只留焦段/材质/基调（见 P1.5(c)）。

## 提案二 — 一致性杠杆：prompt 工艺 → 后端原生原语 + 多抽择优

> 依赖提案一 P1.3：先把 Codex-only 标 contested，本提案才好落。

### 现行原则（要改的三条）
1. **生图 Codex-only 硬规则**（`n2d-image/SKILL.md`「生图调用硬规则（不可选）」）：图必须 Codex/官方 OpenAI，禁即梦/Dreamina，且"不装第三方逆向 CLI"。
2. **后端原生角色 ID / 主体库是 opt-in 重武器**：可灵主体库 / Seedream Universal Reference / Nano Banana 多参考只在"参考图压不住"时绕道启用。
3. **重抽 = 抽 1 张、命中硬伤才重抽**（「重抽预算铁律」）。

### 问题
- 三条把一致性主杠杆焊死在 **prompt 工艺 + 参考图派生**上，而模型矩阵自己记录：一致性第一梯队恰是 Seedream（14 图参考免 LoRA 锁人）/ Nano Banana Pro（多参考 + 原生 SynthID）/ 可灵主体库（按 ID 跨集 ~90% 稳）——**最强的一致性原语被默认规则挡在门外**。
- Codex-only 把两件事捆死：①「不装逆向 CLI」（安全，对）②「只能用 Codex 一个后端」（一致性垄断，可疑）。②牺牲的正是①根本不涉及的**官方**多参考后端。
- 「抽 1 + 硬伤才重抽」在关键镜上是采样浪费：已有 `face/outfit/quality_consistency` 自标定检测器，却只用在事后 QA，没用在生成时择优。

### 提议新原则
- **P2.1 单后端纪律保留，后端降为 choice point，默认取"当前可用的最强一致性原语"**。"同一项目不混用生图后端"（防跨镜画风/脸漂）**继续 invariant**；"那个后端必须是 Codex"撤销——`生图AI` 成正式选择点，按「可用性 × 原生一致性能力」选，有官方多参考/主体库后端则优先，否则回退 Codex。
  > **决策落点（2026-06-08）**：① **阶段0 默认仍 Codex**（二a）；"换默认到最强可用"留到阶段2 用 A/B 数据验证后再决定。② **选择点持久化 = 档2「每次出图前弹完整后端菜单」**（二a'，用户选项 ii）——即不走"沉默沿用"，每集出图前主动呈现后端菜单、默认高亮本项目已用后端。③ **配套护栏（关键）**：档2 每次可重选 ✕ 单后端纪律 invariant 的冲突，由 gate 协调——`gate.py --stage image` 检测到本集所选后端 ≠ 本剧已用后端时 BLOCK/WARN（"本剧已用 X，换 Y 会跨集漂；要么整部统一、要么已出集一并重出"）。菜单每次弹（满足"选择每次都在、可重选"），但**真正中途换**被守护，不静默漂。
- **P2.2 Codex-only 拆成两条独立铁律**：① **安全铁律（invariant，保留）**：只用官方 API/CLI，绝不装第三方逆向 CLI（即梦逆向 CLI 仍禁）。② **垄断铁律（contested，撤销）**：删除"图必须 Codex"；即梦**官方** web/合规 API 与 Codex 同级可选，逆向 CLI 仍禁。
- **P2.3 关键镜改"多抽择优"**：标 🔑 关键镜默认**并行抽 K 张（K=2~4，按 `重抽预算策略` 档）→ 用现成检测器对定妆组 auto-rank → 取最优落档**；非关键镜维持「抽 1 + 硬伤才重抽」。把"筛选宽容 + 重抽预算"从「抽 1 再决定重不重抽」升级为「先小批择优、再决定补不补抽」。
  > **决策（2026-06-08）：全自动取最优**（二b）——auto-rank 直接落最高分候选，不每张让你点。三条护栏让"全自动"≠"自动接受次品"：① **floor 守底**：若 best-of-K 仍低于本作定妆组**自标定地板**（即命中硬伤），不静默落档——按 `重抽预算` 档补抽或标 🔴 待人判；② **与 `生成粒度` 正交**：全自动只是"K 张里自动选一张"，**不跳过** `生成粒度` 的停审颗粒——选 逐个/小批/按场景 停审档时，被选中那张仍在停审点展示+可否决；选 整集档才真无人值守；③ **可追溯**：落档记录写明"抽了 K 张 / 选中哪张 / 各候选分"。

### 影响面
| 改哪 | 怎么改 |
|---|---|
| `n2d-image/SKILL.md` | 「生图调用硬规则」拆成 安全铁律(invariant) + 撤销垄断(contested)；`生图AI` 提为正式选择点；「后端原生角色ID」从 opt-in 提为核心长线角色的默认推荐；新增「多抽择优」段 |
| `_偏好约定.md` / `创作偏好-默认.md` | `生图AI` 默认改为「最强可用一致性后端，回退 Codex」；新增 `关键镜择优K` 选择点 |
| `gate.py --stage image` | BLOCK「非 Codex」改为 BLOCK「项目内**混用**多后端」+ BLOCK「逆向 CLI 口径」 |
| `n2d-review/scripts/` | 新增 `select_best.py`（或扩 `consistency_audit.py`）：N 候选 + 定妆组 → 排名，供 n2d-image 生成时调用（复用检测器，不重写） |
| 模型矩阵 / `platforms.md` | 把"后端原生角色ID/主体库"从增强节提到默认推荐路径 |

### 迁移路径
1. **阶段0（零行为变化）**：拆 Codex-only 为两条铁律 + `生图AI` 写成选择点，**默认值仍 Codex**——只解原则锁。
2. **阶段1**：gate 从「禁非 Codex」→「禁混用 + 禁逆向 CLI」；放行官方多参考后端进默认链路。
3. **阶段2**：实现 `select_best.py`，关键镜接「多抽择优」；一部 demo A/B（Codex 单抽 vs 多参考后端+择优）量一致性税与成本，再决定是否换默认后端。

### 风险与缓解
- **跨后端漂移** → P2.1 保留"单后端纪律"，gate 改为拦混用，风险不升反降。
- **多参考后端合规**（如 Nano Banana SynthID）→ 合规闸门照旧；SynthID 反而是天然 AI 标识，利合规。
- **成本上升**（多抽 K）→ 仅限关键镜，与成本档联动；阶段2 用数据验证 ROI 再扩。

### 待决点
- (a) 默认后端先阶段0解锁、仍 Codex，还是直接换"最强可用"？
- (b) `select_best.py`：**已答（2026-06-08）= 全自动取最优**（floor 守底 + 与生成粒度停审正交 + 可追溯，见 P2.3）。

## 提案三 — 配音先行 vs 占位：二选一并自洽

> 依赖提案一 P1.3：先把「占位驱动付费」标 contested。

### 现行原则（互相矛盾的两条）
1. **配音先行不变量**：镜头时长**必须**由真实配音实测时长驱动；整套时长清单/finalize 闸门/reflow 都为它服务。
2. **占位放行铁律**（`n2d-image`「占位配音可放行」+ `FINALIZE_ALLOW_PLACEHOLDER=1` + `ALLOW_PLACEHOLDER_COMPOSE=1` + `制作模式=先出视频后配音`）：允许用**估算/占位时长**穿过出图/出视频/合成。

### 问题
同一架构里「时长必须真」与「用假时长跑最贵的步骤」同时成立。占位逃逸口让**出图/出视频（最花钱·最不可逆）能跑在架构本身判定为无效的时长上**——正是配音先行要消灭的返工。现状靠散落的"不推荐"措辞勉强压住，原则不自洽。

### 提议新原则（方案 A 推荐）

**方案 A（推荐）— 诚实化为「两遍制：rough timeline → real refine」**
- **P3.A1 两遍是设计内的正式路径**：第一遍=`rough timeline`（占位/估算，只为分镜骨架 + 字幕初定时 + 节奏预览）；第二遍=`real refine`（真实配音 → 重定时 → **增量重出受影响镜头**）。
- **P3.A2 rough 产物全程带"未定稿"水位线**：占位驱动的 PNG/clip 落 `_rough/` 子层或带 `rough=true` 标记，**进度表用 `⏳rough` 而非 `✅`**——从状态机层面分清"骨架"与"定稿"。
- **P3.A3 第二遍是增量不是重来**：定义"时长漂移阈值"——真音 vs 占位每镜差 < 阈值的镜头**免重出**，只重出超阈镜（`asset_impact` 同思路）。把"换真音=全重出"降成"换真音=重出 20%"。

**方案 B（保守）— 占位严格禁止驱动付费生成**
- **P3.B1** 占位只驱动不花钱的步骤；`gate.py` 在 image/video stage **BLOCK 占位**（不再 WARN 放行），`制作模式=先出视频后配音` 标 deprecated。

> **推荐 A**：B 干净但砍掉"快速 demo/比稿"这个真需求；A 既保留快速预览，又从状态机层面分开 rough 与定稿，从根上消除"占位 PNG 被当成片"的错觉。

### 影响面（按方案 A）
| 改哪 | 怎么改 |
|---|---|
| `n2d-voice/SKILL.md` | 「占位分阶段」升级为「两遍制」正式定义；rough 产物标 `rough=true` |
| `n2d-script`（finalize/storyboard） | 占位产物标 rough；新增「时长漂移阈值」计算 → 输出"需重出镜头清单" |
| `n2d-image` | 占位驱动 PNG 落 `_rough/` 或带标记；第二遍按"漂移清单"**只重出超阈镜** |
| `_进度.md` 状态机 / `n2d_contract.py` | 占位驱动列显示 `⏳rough` 而非 `✅`；真音定稿才转 ✅（动 `n2d_route` cell_state 语义） |
| `gate.py` | 占位放行 WARN 文案改为"rough 水位线"；compose 对 rough 成片打"非定稿"标 |
| `n2d-progress` | 仪表盘区分"rough 完成度"与"定稿完成度" |

### 迁移路径
1. **阶段0**：只加 `rough=true` 标记 + 进度表 `⏳rough` 显示，**不改逃逸口行为**——先让"骨架 vs 定稿"可见。
2. **阶段1**：实现"时长漂移阈值 + 需重出清单"。
3. **阶段2**：第二遍走增量重出；compose/gate 文案与水位线对齐。

### 风险与缓解
- **状态机语义改动**牵动 4 个 skill 读 `_进度.md` → 阶段0 只加显示不改判定，`is_done` 后置到阶段2 改。
- **漂移阈值误判** → 阈值默认保守 + 关键镜/爽点镜无论漂移多少都强制复核。

### 待决点
- (a) 走 **A（两遍制）** 还是 **B（占位禁驱动付费）**？
- (b) 若 A：rough 用**目录隔离**（`_rough/`）还是**仅标记位**（`rough=true` 不挪目录）？
- (c) "时长漂移阈值"默认粒度（如每镜 ±0.5s 免重出 / 关键镜一律复核）认不认？
