# Anime Armory Factory

Anime Armory Factory 是 `anime-armory` 的 VS Code 制作面板，用来把作品按固定目录组织起来，并配合 AI agent 按 workflow skills 批量生产小说和 AI 漫剧。

扩展内置 `写小说`（novel-*）与 `制漫剧`（n2d-*）两条线的 workflow skills，并自带一个**种子工作根**：空的 `写小说/` + 一部只含原著小说的 `制漫剧/`，装好即可试跑。创作区（作品区）默认就用这个自带工作根。

> ⚠️ 自带工作根位于已安装的扩展目录内，**升级或重装扩展会被覆盖**。要长期保存自己的作品，请把创作区切到你自己的目录（见下文「选择作品目录」）。

完整源码、skill 细节和脚本实现见 GitHub：
https://github.com/anton6202527/anime-armory

## 快速开始

1. 点开左侧 anime-armory 图标——面板会自动开一个终端，定位到当前创作区。
2. （推荐）用 `File → Open Folder` 打开你自己的项目根，或执行命令 `anime-armory: 选择仓库目录` 指定创作区——这样产物保存在你自己的目录里，且面板实时刷新最稳。
3. 在 `写小说` / `制漫剧` 目录下新建作品文件夹。
4. 在终端里让本地 AI agent 读取作品目录和 `_进度.md`，按当前阶段继续生产。

目录结构：

    your-workspace/
    ├── 写小说/作品名/
    └── 制漫剧/作品名/

每个作品目录建议保留：

    _进度.md    当前阶段、已完成产物、下一步
    _设置.md    平台、模型、分辨率、音色等选择

## 两条工作流

### 写小说

从想法、题材、人物或旧稿开始，逐步生成设定、章纲、正文、改写稿和评分。

    想法 -> 设定 -> 章纲 -> 正文 -> 审稿 -> 改写 -> 评分

### 制漫剧

把小说改成 AI 漫剧短视频。推荐配音先行，用真实音频时长倒推分镜，减少音画错位。

    小说 -> 拆集 -> 台词 -> 配音 -> 分镜 -> 出图 -> 出视频 -> 合成

## 批量生产

同一条线可同时维护多个作品目录。把每个作品状态写清楚，让 agent 按 `_进度.md` 逐个推进：

    读取 写小说/仙门旧案/_进度.md，继续下一步。
    批量检查 制漫剧/ 下所有项目，列出每个项目下一步。

## 选择作品目录

创作区按以下优先级解析：

1. 经 `anime-armory: 选择仓库目录` 指定的外部作品根（会自动开启 `Use External Works`）；
2. 否则用扩展自带的种子工作根；
3. 否则用当前打开的工作区。

无论创作区在哪，面板开的终端都会 cd 到同一个根，产物始终落在该根的 `写小说/` `制漫剧/` 下——和自带样例同一层级、同一套 `_进度.md` / `_设置.md` 约定。

**实时刷新**：面板会同时监听「当前工作区」和「解析出的创作区」两处，新文件通常在约 300ms 内自动出现——既覆盖你 `Open Folder` 打开的工作区，也覆盖工作区之外的自带根 / 外部根。若偶尔没刷出来，点面板顶部的「刷新」按钮即可。

## 面板说明

- 作品线目录置顶显示，读取当前创作区。
- `README.md` 显示在面板根层级，作为使用说明入口。
- 首次打开面板会自动开一个终端，定位到当前创作区目录。
- `Skills` 文件树默认隐藏；调试时在设置里搜索 `anime-armory` 打开 `Show Skills`。

## 合规提醒

请只使用自有、公版或已授权素材。换脸、仿声、真人歌手嗓音克隆等能力必须有授权，并保留必要的 AI 标识。
