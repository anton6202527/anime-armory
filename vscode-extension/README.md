# Anime Arsenal Factory

Anime Arsenal Factory 是 `anime-armory` 的 VS Code 制作面板，用来把作品按固定目录组织起来，并配合 AI agent 按 workflow skills 批量生产小说、AI 漫剧和音乐 MV。

扩展自带一套独立样例工程，不依赖上层源码仓库：`写小说`、`制漫剧`、`写歌`、`制MV` 四条线各内置一个完整作品，安装后可以直接展开查看目录、进度文件和产物组织方式。

完整源码、skill 细节和脚本实现见 GitHub：
https://github.com/anton6202527/anime-armory

## 快速开始

1. 点开左侧 Anime Arsenal 图标。
2. 直接查看内置的 `写小说`、`制漫剧`、`写歌`、`制MV` 样例作品。
3. 复制任意样例作品，或在对应目录下新建作品文件夹。
4. 打开终端，让本地 AI agent 读取作品目录和 `_进度.md`，按当前阶段继续生产。

内置样例：

```text
写小说/本宫才是这皇宫最大的妖/
制漫剧/本宫才是这皇宫最大的妖/
写歌/仗剑下山/
制MV/仗剑下山/
```

如果要维护自己的独立制作目录，也可以按同样结构创建：

```text
your-workspace/
├── 写小说/作品名/
├── 制漫剧/作品名/
├── 写歌/歌曲名/
└── 制MV/歌曲名/
```

每个作品目录建议保留：

```text
_进度.md    当前阶段、已完成产物、下一步
_设置.md    平台、模型、分辨率、音色等选择
```

## 四条工作流

### 写小说

适合从想法、题材、人物或旧稿开始，逐步生成设定、章纲、正文、改写稿和评分结果。

常用节奏：

```text
想法 -> 设定 -> 章纲 -> 正文 -> 审稿 -> 改写 -> 评分
```

### 制漫剧

适合把小说改成 AI 漫剧短视频。推荐配音先行，用真实音频时长倒推分镜，减少音画错位。

常用节奏：

```text
小说 -> 拆集 -> 台词 -> 配音 -> 分镜 -> 出图 -> 出视频 -> 合成
```

### 写歌

适合从主题、风格或一句 hook 直接创作歌曲，也可以继续编辑已有歌曲项目。

常用节奏：

```text
主题 -> 歌词 -> 作曲/人声 -> 多版挑选 -> 质检
```

### 制MV

适合把成品歌做成音乐 MV，包括节拍分析、画面计划、歌词对齐、卡拉 OK 字幕和最终合成。

常用节奏：

```text
歌曲 -> 节拍 -> MV 计划 -> 出图 -> 出视频 -> 歌词同步 -> 合成
```

## 批量生产方式

同一条线可以同时维护多个作品目录。批量生产时，把每个作品的状态写清楚，让 agent 根据 `_进度.md` 逐个推进。

示例：

```text
写小说/
├── 仙门旧案/
├── 夜雨长安/
└── 海底碑文/

制漫剧/
├── 仙门旧案/
├── 夜雨长安/
└── 海底碑文/
```

给 agent 的指令可以保持简单：

```text
读取 写小说/仙门旧案/_进度.md，继续下一步。
批量检查 制漫剧/ 下所有项目，列出每个项目下一步。
按 _设置.md 的分辨率和模型，继续生成 制MV/仗剑下山 的剩余镜头。
```

## 切换外部作品目录

默认情况下，面板读取扩展自带的独立样例工程。

如果以后要切到外部作品目录，可执行命令：

```text
Anime Arsenal: 选择仓库目录
```

选择后会自动开启：

```text
Anime Arsenal: Use External Works
```

要切回内置样例工程，把 VS Code 设置里的 `Anime Arsenal: Use External Works` 关掉即可。

## 面板说明

- 作品线目录默认置顶显示，读取扩展自带的独立样例工程，不默认读取上层源码仓库。
- `README.md` 直接显示在面板根层级，作为使用说明入口。
- 首次打开面板会自动开一个终端，默认定位到扩展自己的样例工程目录。
- `Skills` 文件树默认隐藏，避免普通用户看到内部实现。需要调试时在 VS Code 设置里开启 `Anime Arsenal: Show Skills`。
- 完整 workflow skill 的源码、职责说明和脚本都在 GitHub 仓库中。

## 合规提醒

请只使用自有、公版或已授权素材。换脸、仿声、真人歌手嗓音克隆等能力必须有授权，并保留必要的 AI 标识。
