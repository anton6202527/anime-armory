# 续写法

在原作末章后**接着写新章节**。和扩写（章节内加细节）、精简、视角续写都不一样。

## 核心原则

1. **续写不是改写**——主角仍是主角，世界规则不动，已发生的事不能反悔。
2. **续写必须根植原作钩子**——续编要用原作未回收伏笔 ≥ 50%；接更要尊重作者既定走向。
3. **文风优先于情节**——读者打开续写第一章，会先判断"是不是同一本书"。文风不像，再好的情节也救不回。

## 续编 vs 接更

| 维度 | 续编（sequel） | 接更（continuation） |
|---|---|---|
| 原作状态 | 已完结 | 未完结（卡更 / 烂尾） |
| 自由度 | 大 | 小 |
| 主线 | 可开新主线，但要根植旧伏笔 | 延续作者既定主线轨迹 |
| 文风 | 可微调（毕竟读者已经读完了原作） | 必须严丝合缝（让读者错觉作者还在更） |
| 伏笔 | 可回收旧伏笔 + 埋新伏笔 | 重点回收旧伏笔；少埋新的 |

## 第 2 步：吸收原作要建的 5 张卡

续写最大的工作量在第 2 步。**末章状态卡 + 作者口吻卡** 比 spinoff 的锚点表还重要。

- **人物卡**：每个主要人物的截止末章状态——所处境界 / 关系 / 持有物 / 心结。续写第一章里这些必须严丝合缝。
- **世界观卡**：原作已确立的规则。新章不要扩展或推翻。
- **主线骨架**：已发生的关键事件，时序排好。新章要承接末尾不要反悔前面。
- **末章状态卡**：⭐ 最关键 ⭐ —— 末章结束时：
  - 主角在哪 / 在做什么 / 心境如何
  - **未回收的伏笔清单**（这是续写章纲的输入）
  - 未回答的悬念
  - 章末留下的钩子
- **作者口吻卡**：句长 / 段落节奏 / 高频词汇 / 每个角色的标志性短句 / 描写风格。**子代理 prompt 必塞这张卡**。

## 第 3 步：续写方向（最重要的 gate）

给用户 2-3 个候选方向，每个方向：

```markdown
### 方向 A：<主线一句话>
- 用上的未回收伏笔：[A、B、C]（占总伏笔 60%）
- 新冲突：[一句话]
- 新角色（如有）：[一句话，不可主导主角线]
- 风险点：[一句话]
```

用户选 → 写入 `_meta.json.direction_chosen`。

**反模式**：
- 没让用户选方向就开写章纲（最常见 / 最致命）
- 候选都同一类（应当覆盖 2-3 种不同张力方向）
- 凭空开新故事线，伏笔覆盖率 < 30%

## 单章续写的关键差异（vs 普通章节）

子代理 prompt 比标准 chapter.md 多塞这些：

- **原作末 1-2 章全文**（文风锚点；不是设定锚点）
- **作者口吻卡**全文
- **末章状态卡**（让子代理知道每个人物现在在哪）
- **承诺要回收的伏笔列表**（本章承担哪几条）

写章守则的差异：
- **承接 ≠ 复述**：续写章第一段不要复述末章发生了什么——直接接下来发生的事。回顾用一两句简短带过即可，比如"三日后，他从云霄峰下来"。
- **文风对齐 >> 情节推进**：每段写完自问"这一段如果塞回原作末章后面读者会觉得突兀吗"。
- **伏笔回收节奏**：不要一章回收 5 个伏笔。一章 1-2 个，分布到全本。
- **新角色**：可加，但不能压主角戏份。新角色最好出现在前 1/3，给后面留发挥空间。

## 第 7 步：一致性回扫的额外清单

除了 chapter.md 的标准清单，续写还要扫：

- [ ] 文风：抽 3-5 章续写章 vs 抽 3-5 章原作，盲读对比——能不能听出是不是一个人写的？
- [ ] 伏笔回收：章纲承诺要回收的，是不是都回收了？回收方式合不合理？
- [ ] 主角状态推进：从末章状态卡到续写终章，主角的境界 / 关系 / 心境是不是线性合理推进？没跳跃 / 没倒退？
- [ ] 新设定：有没有不小心加了原作没有的设定？有就删。
- [ ] 末章状态对齐：续写第 1 章的开头，是不是从末章状态卡 1:1 接续的？

## 反模式总结

- **凭空续编**：原作末章是 A，续写第 1 章直接跳到完全不相干的 B。
- **改主角性格**：续写时主角忽然变得更"现代"或"成熟"。续写章主角必须接续末章状态。
- **大段照搬原作**：续写章里"为了让读者回忆"贴大段原作文本。一两句即可，不许大段。
- **文风跟随作者写完结又跑偏**：续写不像续，像同人。
- **新角色抢戏**：续写第 5 章就让新角色成为主线核心人物。
