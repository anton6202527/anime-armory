# 书名候选 + 平台对位评分

设定卡完工后、章纲之前，给用户 5–8 个书名候选，按平台命名习惯打分，让用户选。

## 主流中文平台对名字的偏好

### 起点 / 番茄 / 飞卢（男频长篇）
- 强标签 + 强动作或强身份：`我在X当Y的日子` / `重生成X的我Y` / `X第一Y`
- 长度 4–12 字最佳
- 情绪锚：爽 / 反差 / 不可能
- 反例：纯文艺名（一句诗、长抒情）→ 点击率极低

### 晋江（女频长篇）
- 身份/关系 + 反差：`X大小姐的Y` / `她说不就是X` / `重生后X决定Y`
- 长度 6–15 字
- 情绪锚：逆袭 / 自我 / 双向

### 抖音漫剧 / 红果短剧 / 微短剧
- 极端冲突 + 钩子：`一夜X` / `亿万X的Y` / `被X后我Y`
- 长度 6–14 字
- 情绪锚：撕破脸 / 反转 / 当面打脸
- 关键：**看到题目就知道这一集要爽点什么**

### 仙侠垂类（不论平台）
- 修真大词 + 反差小词：`仙界第一X` / `在X当Y的日子` / `我修仙就为了X`
- 长度 6–12 字
- 情绪锚：大背景 + 小切口

### 历史向（公版史传改写、汉末三国/魏晋南北朝等）
- 人物 + 标签 / 反差：`<人物>本纪` / `<人物>的Y` / `我，<人物>，要Y`
- 长度 5–14 字
- 情绪锚：把史载冷冰冰的几行字翻译成现代爽点

## 候选生成规则

每次给用户 **5–8 个候选**，按这个配比：

- 2–3 个偏向**目标主平台**（用户在第 0 步指定的目标平台；缺省按 SKILL.md 默认推荐）
- 1–2 个**跨平台稳健型**（命中多个平台命名习惯的中庸名）
- 1–2 个**高钩子尝试**（更激进、更可能爆但也更可能扑）
- 0–1 个**保守雅名**（文艺向，作为对照组让用户感受差距）

候选必须包含的元信息：

| 字段 | 说明 |
|---|---|
| 候选名 | 4–15 字 |
| 主推平台 | 起点 / 晋江 / 抖音漫剧 / 番茄 / 红果 / 跨平台 |
| 一句话定位 | 这名字想卖什么（10–20 字内） |
| 评分（5 维 × 5 分） | 见下 |

## 5 维评分（满分 25）

每个候选按以下 5 维各打 1–5 分：

1. **钩子强度** — 看到题目就想点的程度
2. **平台契合** — 与目标平台头部内容命名习惯的吻合度
3. **角色识别** — 是否传达了视角配角的核心身份/张力
4. **重名风险**（反向） — 搜大站后发现同名/近似名越少越高分（5 = 几乎无重名；1 = 严重撞名）
5. **可记忆性** — 读一遍能不能复述

**总分**：5 维加总 / 25。

## 输出给用户的格式

```markdown
## 书名候选（视角配角：<配角名>，目标平台：<主平台>）

| # | 候选名 | 主推平台 | 钩子 | 契合 | 识别 | 抗撞名 | 记忆 | 总分 | 一句话定位 |
|---|---|---|---|---|---|---|---|---|---|
| 1 | ... | ... | 5 | 5 | 4 | 4 | 5 | 23 | ... |
| 2 | ... | ... | 4 | 5 | 5 | 5 | 4 | 23 | ... |
| ... |
**我的推荐**：第 N 个，因为 <一句话理由>。
**备选**：第 M 个，如果你更想 <某倾向>。
```

排名按总分降序。同分按"主推平台契合度 → 钩子强度"破平。给一行推荐 + 一行备选，最后等用户选定。

用户选定后，把书名写入 `_meta.json` 的 `title` 字段（如无则新建）；导出时 `export.py --title <书名>` 读取。

## 实用约束

- **不要给真实在售书的同名**（重名风险扣到 1，淘汰）。
- **不要给纯抒情/纯诗句**（除非用户明确选保守雅名档）。
- **不要让候选都长得像**（5 个候选要走 5 种钩子方向）。
- **派生作品的书名不要直接挂原作 IP**（如 "XX外传"、"XX之X" 之类，除非用户拥有 IP）。视角作品要有自己的名字。

## 何时复用旧规则

如果用户**已经给了暂定名**（如本次的"王敦传"），把它作为候选 #0 一并打分，并明确指出它的弱点（如：传统史传命名 → 历史向平台契合度高，但漫剧/网文平台钩子弱）。让用户在选项里直接看到"沿用 vs 换名"的差异。
