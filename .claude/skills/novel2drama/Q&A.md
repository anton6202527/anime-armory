## Q1：直接把分镜剧本丢给即梦，还是先出图再生成视频？<a id="q1"></a>

**强烈建议先出图再生成视频（图生视频流程），而不是直接文生视频。**

### 1.1 人物一致性问题
直接文生视频，每个镜头的角色长相、服装、发型都会漂移，跨镜头根本对不上。先做"定妆图"锁死角色形象，再用图生视频，整集人物才稳得住。这也是本仓库 `novel2drama` skill 默认的工作流（角色定妆 → 分镜图 → 视频）。

### 1.2 试错成本
- **文生视频**：一条 5s 视频 几毛~几块，废一条就是几块没了，要等几十秒
- **文生图**：几分钱一张，秒出，不满意立刻重抽

把"构图、人物、光线、镜头"这些容易翻车的变量在图阶段就锁死，视频阶段只控制"动作/运镜"，废片率断崖式下降。

### 1.3 可控性
图阶段可以局部重绘、换脸、改服装、统一色调；直接出视频这些都没法精修，只能整条重抽。

### 1.4 即梦的实际表现
即梦的"文生视频"在长剧情、多角色场景下一致性较弱；它的"图生视频（首帧/首尾帧）"才是稳定出片的主力模式。专业短剧团队基本都是这个流程。

### 1.5 唯一例外
纯空镜、转场、氛围镜头（风/雨/火/光效）可以直接文生视频，省一步。

### 1.6 标准流程

```
小说原文
  → 分镜剧本（episode_N.md）
  → 角色定妆 prompt（锁死人物形象，先出定妆图）
  → 分镜图 prompt（基于定妆图，逐镜出图）
  → 视频 prompt（图生视频，控制动作/运镜）
  → 配音 + BGM + 字幕
  → 合成成片
```

## Q2：制作一集的整体步骤是什么？<a id="q2"></a>

**4 个大步骤，前后严格依赖，不能跳。**

```
Step 1 定妆（角色+场景+道具）
   ↓ 所有定妆图存到 第N集/出图/定妆_xxx.png
Step 2 分镜出图（用定妆图作参考图，逐镜出图）
   ↓ 镜头图存到 第N集/出图/镜头N_xxx.png
Step 3 图生视频（每张定稿图喂即梦"图生视频"+ 运镜 prompt）
   ↓ 视频片段存到 第N集/视频/镜头N.mp4
Step 4 合成（配音 + BGM + 字幕 + 剪辑）
   ↓ 成片
```

**关键原则：**
- 上一步不锁死，下一步全废
- Step 1 占整体时间约 30%，但决定整集质量
- 一次只做一个角色/场景的定妆，全部稳了再进 Step 2

## Q3：定妆阶段要先出哪些图？顺序是什么？<a id="q3"></a>

按 **"主角 → 配角 → 反派 → 场景 → 道具"** 顺序，原因：

1. **主角先**：主角是全集出现最多的，定妆稳了才能验证 prompt 模板是否有效
2. **主角的"形态变体"算独立定妆**：比如沈念有常态、觉醒态、银牌态，每个都要单独定妆
3. **反派最后**：反派要跟主角"截然不同"，先做完主角的脸，反派 prompt 才能针对性堵死"长得像主角"的翻车
4. **场景独立做**：场景必须用"空镜"出图，不能带任何人物（即梦经常偷偷画进一个剪影）
5. **道具最后**：因为道具是"特写"，可有可无，分镜阶段也能临时补

**第1集案例：**
```
1.1 沈念·常态定妆   ← 整集人物锚点
1.2 沈念·觉醒态定妆  ← 用 1.1 做参考图
1.3 小禾定妆        ← 不用参考图（新角色）
1.4 柳娘子定妆      ← 不用参考图（新角色，且要跟主角拉开差异）
1.5 冷宫寝殿场景图  ← 严禁带人物
1.6 道具图（赐死三件套等）← 可选
```

## Q4：沈念（女主）常态定妆怎么做？<a id="q4"></a>

### 4.1 正向提示词（来自 `common/characters/沈念.md` 第22行）

```
角色定妆设定图：废妃林婉儿（沈念），20岁出头女子，全身正面站姿A-pose + 半身脸部特写，
乌黑长发半披素布发带、素颜淡唇凤眼眉眼如画、肤白近透明、左手腕淡旧疤，
月白色洗白粗布交领旧宫装窄袖素腰带、无配饰，干净浅灰纯色背景，
柔和均匀打光无强阴影，五官与服装清晰，国风写实漫剧角色设定图，高细节，竖版9:16
```

### 4.2 负向提示词

```
低幼Q版、卡通脸、画风漂移、多余文字水印、多余手指、现代物件、五官崩坏、
欧美脸、写真摄影、纹身、暴露服装
```

### 4.3 即梦参数

| 项 | 值 |
|---|---|
| 模型 | 图片3.0（写实更稳） |
| 比例 | 9:16 |
| 精细度 | 拉满 |
| 数量 | 4 张/轮 |

### 4.4 关键技巧
- **定妆图必须是 A-pose 中性姿态**（不是"最有戏的姿态"），因为后续要做参考图复用
- 背景必须是**干净浅灰纯色**（不是冷宫场景）
- 4 张全废不要改 prompt，**直接同 prompt 再抽一轮**（即梦同 prompt 不同种子差异很大）

## Q5：定妆图怎么挑？检查清单是什么？<a id="q5"></a>

### 5.1 通用 6 条（每张定妆都要核对）

1. ✅ 是**符合设定的脸型**（古风东方脸 vs 欧美脸）
2. ✅ 发型/发色对（披发 vs 双环髻 vs 紧实发髻）
3. ✅ 服装色对（月白 vs 青色 vs 深蓝）
4. ✅ 配饰对（沈念无配饰 / 小禾无金饰 / 柳娘子有银簪）
5. ✅ 设定细节（沈念左手腕疤、小禾雀斑等，能体现最好，丢了也接受）
6. ✅ 背景是干净纯色

### 5.2 挑选优先级（4 选 1 的逻辑）

| 优先级 | 标准 | 为什么 |
|---|---|---|
| ① | **姿态最中性** | 定妆要做参考图复用，侧身/夸张姿势会让后续镜头都歪 |
| ② | **双臂/手可见** | 后面觉醒态、动作戏要画手臂图腾、抓握动作，藏手就废了 |
| ③ | **脸最像设定** | 4 张里挑最匹配人设关键词的那张 |
| ④ | 视觉冲击力 | 排最后，定妆要的不是"最美"，是"最稳" |

### 5.3 已知瑕疵可接受

定妆**不需要完美**，只需要"脸/服装/发型"三大件锚定即可。以下瑕疵接受：
- 设定中的"左手腕一道淡旧疤" → 被画成"一片红斑"（即梦理解偏差，可接受）
- "冷宫三年憔悴感"没画出来（即梦倾向于画健康脸）

**为什么不重抽？**
1. 重抽不一定更好，可能脸变了反而更亏
2. 这些小瑕疵会被后续构图/服装遮盖
3. **已经稳的脸不要轻易换**

## Q6：沈念觉醒态怎么做？为什么要用参考图？<a id="q6"></a>

### 6.1 必须用常态定妆做参考图

**理由：** 觉醒态是"同一个人变身"，脸必须跟常态一致，不然后面镜头"觉醒前 → 觉醒中 → 觉醒后"会变成三个人。

### 6.2 即梦设置

1. 找"**参考图 / 角色参考 / 图片参考**"入口（不是"风格参考"）
2. 上传 `定妆_沈念_常态.png`
3. 参考强度调 **0.75-0.85**（要锁脸）

### 6.3 模型必须跟常态同一个

如果常态用了 图片3.0，觉醒态也用 图片3.0。换模型脸会跑。

## Q7：觉醒态被即梦画成"金属铠甲/护臂"，怎么修正？<a id="q7"></a>

### 7.1 真实翻车记录

第一版 prompt 用了 "**双臂浮现暗金色古老图腾纹路**"，即梦把它理解成了：
- 双臂穿戴黑金图腾铠甲护臂
- 利爪手套
- 金属胸甲/项圈

完全不对——我们要的是"皮肤上的活体纹身（血脉觉醒）"，不是"穿戴装备"。

### 7.2 根本原因

中文"图腾"、"纹路"对即梦来说是**有结构的视觉物件**（容易被画成实物），不是"皮肤纹身"。

### 7.3 修正版 prompt（已验证有效）

```
同一少女（参考图人物），裸露的双前臂皮肤上浮现暗金色古老图腾印记，
纹路如刺青般自皮肤内透出微光、与皮肤完全齐平无任何凸起，
纹路从胸口位置向外蔓延至肩膀、手肘、手腕、指尖，
月白粗布宫装的宽松袖口被妖气从下向上轻轻掀起露出前臂，
双眼化为金色野兽竖瞳（瞳孔细长如猫），发丝被妖气微微掀起，
体表环绕暗金色细碎光点与流光雾气，半身近景正面构图，
干净深灰渐暗背景，电影级光影，国风写实漫剧，高细节，竖版9:16
```

**关键改动：**

| 改动 | 为什么 |
|---|---|
| "图腾"→"印记 + 刺青" | "图腾"被理解成实体物件，"印记/刺青"才是皮肤纹路 |
| "皮肤内透出微光、与皮肤完全齐平无任何凸起" | 防止再画成3D凸起的铠甲 |
| "裸露的双前臂" | 强调露出皮肤、不是穿装备 |
| "宽松袖口被妖气掀起露出前臂" | 给即梦一个合理理由让袖子露出手臂（否则它会脑补成换装/铠甲） |

### 7.4 修正版负向词

```
铠甲、护臂、护腕、金属手套、利爪、装甲、机甲、面具、胸甲、项圈、
金属饰品、穿戴式装备、cosplay服装、低幼Q版、卡通脸、画风漂移、
多余文字水印、多余手指、现代物件、五官崩坏、欧美脸、写真摄影、
暴露服装、纹身贴纸感、荧光绿、廉色霓虹、双瞳、双头
```

把所有"装备类"翻车词全部堵死。

### 7.5 已知偏差（接受）

修正后服装变成了"无袖坎肩"款（即梦为了让纹路可见，把袖子简化掉了）。
- ❌ 原设定是窄袖宫装
- ✅ 但视觉冲击力极强，符合"觉醒爆发"质感
- ✅ 通过后续镜头 prompt 加"窄袖在妖气冲击下从肘部炸裂撕开"作过渡来圆

**结论：定妆阶段抓核心（纹路是皮肤而非铠甲），细节偏差留给分镜阶段圆。**

## Q8：新角色（小禾、柳娘子）要不要用前面角色做参考图？<a id="q8"></a>

**不要。完全相反——做新角色前必须先移除前一步的参考图。**

### 8.1 后果

如果带着沈念的参考图画小禾，会出现：
- "长得像沈念的小宫女"
- 凤眼出现在小禾脸上
- 月白色衣服变成青色但脸是沈念

### 8.2 操作

每次切换角色前，**显式检查并清空参考图**。即梦的参考图框是"粘性"的，不主动清理会带到下一张。

### 8.3 但同一角色的"形态变体"必须用参考图

| 场景 | 用参考图？ |
|---|---|
| 沈念常态 → 沈念觉醒态 | ✅ 用（同一个人） |
| 沈念 → 小禾 | ❌ 不用（不同人） |
| 小禾 → 柳娘子 | ❌ 不用（不同人） |
| 柳娘子伪善态 → 柳娘子死亡态 | ✅ 用（同一角色） |

## Q9：角色年龄/外貌偏离设定（比如小禾画得太小），是重抽还是接受？<a id="q9"></a>

**优先接受，除非彻底破坏戏剧逻辑。**

### 9.1 真实案例：小禾设定 15 岁，被画成 ~11 岁

第一反应是重抽。但仔细分析后接受了，理由：

1. **戏剧效果其实更好**：小禾在镜头 5/8/12 三场戏（哭喊报信/被太监捂嘴/瞪眼震惊）。一个看起来更小的忠仆，能让"太监捂嘴拖走"的暴力感**更冲击**，沈念的"护主"动机也更强。漫剧惯用手法。

2. **重抽风险高**：即梦对"15岁少女"的稳定输出就在这个面相区间。继续重抽不一定更老，但脸型可能变（鼻子嘴巴比例一变就跟之前不一致）。**已经稳的脸不要轻易换。**

### 9.2 决策框架

| 偏差类型 | 怎么办 |
|---|---|
| 服装色错了（月白画成紫色） | **重抽**，颜色错全集穿帮 |
| 发型错了（双环髻画成披发） | **重抽**，发型是辨识度核心 |
| 年龄偏小/偏大 ±3-5 岁 | **接受**，戏剧上往往更好 |
| 缺少疤痕、雀斑等小细节 | **接受**，分镜阶段补 |
| 表情风格不对（伪善画成凶相） | **重抽**，气质决定角色 |
| 多了/少了配饰 | 看是否破坏身份，破坏则重抽 |

### 9.3 如果非要再大/再小，prompt 加这句

```
14-15岁少女初长成，已有少女轮廓但保留稚气，身高比成年女子矮一头
```
负向词补 `8岁、9岁、10岁、儿童、幼儿、童星感`。

## Q10：反派（柳娘子）怎么避免被画得像主角？<a id="q10"></a>

### 10.1 负向词主动"驱赶"主角特征

柳娘子的负向词必须包含**沈念和小禾的视觉关键词**，防止即梦"风格污染"：

```
年轻少女、稚气、圆脸、双环髻、披发、月白色、淡青色、华丽贵气、
嫔妃感、皇后感、邪恶咧嘴、明显凶相、僵尸妆、妖怪外露、獠牙、
低幼Q版、卡通脸、画风漂移、多余文字水印、多余手指、现代物件、
五官崩坏、欧美脸、写真摄影、暴露服装
```

**重点：**
- "稚气、圆脸、双环髻" → 防止画成小禾
- "月白、淡青、华丽" → 防止画成沈念
- "邪恶咧嘴、獠牙" → 柳娘子是**伪善**，不能脸上写"我是妖"

### 10.2 正向词强调"区分特征"

| 主角特征 | 反派对应特征 |
|---|---|
| 20岁出头、清瘦 | **30岁中年、端庄保养** |
| 凤眼 | **鹅蛋脸、眼尾微吊** |
| 月白粗布 | **深蓝端庄绸料、盘扣宽袖** |
| 无配饰 | **素银簪、管事腰牌** |
| 披发素布带 | **乌黑一丝不苟发髻** |

每个对立面都明确说出来，即梦就不会混。

## Q11："伪善反派"的眼神戏，定妆图要画出来吗？<a id="q11"></a>

**两种思路都行，但更高阶的是"定妆完全无破绽，反差留给镜头"。**

### 11.1 思路 A：定妆就体现"似笑非笑"

prompt 加"嘴角似笑非笑、眼神阴冷算计"。
- 优点：定妆就有反派气质
- 缺点：太早暴露反派身份，"伪善"的反转感弱

### 11.2 思路 B：定妆纯端庄、完全像好人

让定妆看起来就是普通慈眉善目的管事姑姑。
- 优点：**镜头11"砰一拳穿胸化黑烟"的反转冲击力最大**
- 优点："温柔到毛骨悚然"那句台词靠表演镜头细节实现，更高级
- 缺点：定妆没有"识别度"，得靠服装（深蓝管事服+腰牌）辨认

### 11.3 实战选择

**第1集选了思路 B。** 因为：
- 漫剧的核心是反转，铺垫越平淡，反转越炸
- "看起来就是个普通好人"的反派 > "一眼就坏"的反派
- "似笑非笑"通过镜头7的特写 prompt 单独画一次就行

### 11.4 通用原则

> **定妆抓基础气质（端庄/稚气/冷艳），具体表情留给镜头 prompt。**
> 不要在定妆阶段试图把"角色弧光的复杂性"全画出来。

## Q12：场景图要避免哪些坑？<a id="q12"></a>

### 12.1 必须严防"画进人"

即梦做场景图时经常偷偷画进一个剪影/远景人物，破坏空镜效果。

**强制负向词：**
```
人物、人脸、女子、男子、宫女、太监、贵妃、剪影、人影
```

### 12.2 必须强调"破败 + 死寂"

即梦的默认审美偏"明亮华丽"，不主动堵会画成正常寝宫。

**强制负向词：**
```
明亮、华丽、整洁、新房、富丽堂皇、彩色、暖色调、阳光、白天、户外
```

### 12.3 必须包含"复用元素"

场景里会反复入镜的物件必须在定妆场景图里出现：
- 雕花木榻（主要走戏区）
- 角落铜镜（镜头3要用）
- 案台 + 残烛（氛围空镜要用）
- 蛛网、霉斑（破败感锚点）

定妆阶段不画进去，后续镜头就没有视觉锚定。

### 12.4 绝不带任何角色的参考图

场景定妆**必须清空参考图**。如果带着人物参考图，会出现"角色躺在床上"等莫名其妙的画面。

## Q15：什么时候该把手动流程改成 CLI 自动化？<a id="q15"></a>

### 15.1 决策框架

| 阶段 | 推荐方式 | 理由 |
|---|---|---|
| **首集首批定妆**（2-4 张） | **手动** | 需要人工筛选脸/服装，是整集锚点，不能糊涂 |
| **首集剩余定妆 + 分镜出图** | **手动**（沉没成本规避） | 已经用手动定下风格，临时切自动会画风跳变 |
| **第二集开始的所有出图** | **自动化 CLI** | 风格已定型、模板成熟，自动跑性价比最高 |
| **图生视频（Step 3）** | **自动化 CLI** | 视频生成最耗时间，自动化收益最大 |
| **大批量重抽 / 全集风格统一返工** | **自动化 CLI** | 手动跑不动 |

### 15.2 中途切换的代价

如果一集做到一半切换工具，画风会跳变。3 种应对：

| 应对 | 适用 |
|---|---|
| 接受跳变 | 不推荐——观众一眼看出 |
| 回头重做已完成的 | 浪费已完成工作 |
| **本集做完再切换** | ✅ 推荐——以"集"为单位切换粒度 |

### 15.3 第1集决策记录

第1集已用即梦**手动**完成 4 张定妆。
**决定：第1集全程手动跑完，第2集开始接入即梦 CLI 自动化。**

理由：
- 手动定妆已锁画风，剩下出图 + 视频改 CLI 会风格跳变
- 已经手动跑了 30% 的工作量，切换的"重做成本"高于"剩余手动成本"
- 第1集做完正好可以评估 CLI 在第2集的全自动化效果

## Q16：换 AI 工具（即梦↔可灵↔DALL-E 等）时怎么权衡画风一致性？<a id="q16"></a>

### 16.1 各家画风指纹（实测对比）

| 工具 | 风格指纹 | 适用 |
|---|---|---|
| **即梦 图片3.0** | 国风写实、影楼级光感、皮肤通透、东方审美 | 国风短剧主力 |
| **可灵 Kling** | 写实偏冷艳、电影质感、华人脸型自然 | 国风兜底备选，最接近即梦 |
| **DALL-E 3 / gpt-image-1** | 插画感、西方审美、皮肤偏暖、面部比例西化 | 海外投放素材 |
| **Flux Pro** | 好莱坞写实、欧美审美主导、亚洲脸不够自然 | 通用配套素材 |
| **SDXL/SD3** | 高度可控但需 LoRA，原生偏二次元 | 进阶玩家 |
| **Midjourney** | 艺术感最强、电影海报感、不可控性最高 | 单图艺术品而非系列 |

### 16.2 切换工具的"一致性税"

切换工具 = 整集所有人物/场景**必须重做定妆**。原因：

- AI 模型有"美学惯性"——同一 prompt 在不同模型下出来是不同的脸/光感
- 即使保留"参考图"功能，跨模型迁移效果都很差（参考图编码方式不一样）
- 视觉一致性是漫剧的核心质感，跳变 = 观感破产

### 16.3 切换决策表

| 情境 | 推荐 |
|---|---|
| 同一集制作中 | ❌ 绝对不切 |
| 集间过渡（第N集 → 第N+1集） | ⚠️ 可切，但要重做所有定妆 |
| 整剧未开始 | ✅ 选定后不再改 |
| 主工具账号封禁/积分耗尽 | ⚠️ 紧急情况切换备选（提前备好可灵） |

### 16.4 备选策略：双工具栈

国风短剧推荐配置：
- **主**：即梦 CLI（出图 + 视频）
- **备**：可灵 Kling API（账号被封时的紧急后备）
- 两者风格最接近，切换"一致性税"最小

不推荐同时用 即梦 + DALL-E / Flux——审美差异太大，切换等于重做。

## Q17：`curl | bash` 类安装脚本怎么审？<a id="q17"></a>

任何 `curl xxx | bash` 都不该盲跑。审 5 项：

### 17.1 域名核对（最重要）

下载脚本的域名 + 脚本内部下载二进制的 URL 域名，**都必须**落在该厂商**官方主域名**下。

第三方域名、缩短链接（bit.ly/t.cn 等）、不知名 CDN → 直接拒绝。

### 17.2 用 WebFetch 把脚本读出来

不要直接 pipe 到 bash，先 WebFetch 看内容。审下面 4 项：

### 17.3 是否要 sudo / root

合规的用户级工具**不应该**要 sudo。要 root 几乎都是危险信号。
官方即梦 CLI：✅ 装到 `~/.local/bin`，无 sudo。

### 17.4 是否往敏感位置写

| 安全位置 | 危险位置 |
|---|---|
| `~/.local/bin`、`~/bin` | `/usr/local/bin`、`/etc/`、`/usr/bin` |
| `~/.dreamina_cli/`（自家目录） | `/var/`、`/opt/`（不属于该工具的目录） |
| 改 `~/.zshrc` / `~/.bashrc`（仅追加 PATH） | 改 `/etc/profile`、`sudoers` |

### 17.5 是否有可疑行为

- 上传任何**本地文件**到远程 → 危险
- `eval $(curl ...)`、`base64 -d | sh` 等混淆执行 → 危险
- 创建 systemd 服务、cron 任务（用户没要求） → 可疑
- 修改其他工具的配置（如非声明范围） → 可疑

### 17.6 即梦 CLI 审查结论（参考案例）

走完上面 5 步，结论：
- ✅ 域名干净（jimeng.jianying.com + bytednsdoc.com 都是字节系）
- ✅ 用户级安装、无 sudo
- ✅ 只动 `~/.local/bin` + `~/.dreamina_cli` + `~/.zshrc` 追加 PATH
- ✅ 无可疑行为
- ⚠️ 唯一注入点：脚本检测 `/root/.openclaw/workspace/TOOLS.md` 存在则追加内容——但只对装了字节 openclaw 平台的机器生效，对个人机无影响
- ✅ 可以装

## Q18：图 AI 和视频 AI 是什么关系？image prompt 该按谁写？<a id="q18"></a>

### 18.1 两轴架构

```
图 AI（出图工具）→ 图片 → 视频 AI（出视频工具）
       ↑                          ↑
   决定形式               决定内容
prompt 的"写法"        图片"该长啥样"
（语言/格式/参考机制）  （面孔/风格/光感/构图）
```

- **图 AI** = 工具选择（可以是 Gemini / DALL-E / Flux / 即梦 / 可灵任何一个）
- **视频 AI** = 风格基线（决定图片必须长成什么样子，因为它的 image2video 只熟悉自家训练分布）

### 18.2 为什么 image prompt 按视频 AI 来定风格？

视频 AI 的 image2video 本质：
1. 把输入图当首帧
2. 在自家训练分布里推断运动 + 续帧

如果首帧不在视频 AI 训练分布里（比如即梦 image2video 拿到一张 DALL-E 风格的西方写真），运动推断会崩——人脸抖、转场怪、丝绸飘动不自然。

所以 image prompt 描述的"图片该长啥样"必须是**视频 AI 熟悉的样子**，而不是图 AI 自己的默认审美。

### 18.3 同 AI vs 跨 AI 决策表

| 组合 | 锚定句 | 备注 |
|---|---|---|
| 即梦图 + 即梦视频 | ❌ 不需要 | 同家闭环，画风词已隐含 |
| Gemini 图 + 即梦视频 | ✅ **必须拼** 即梦的图像风格锚定句 | 省钱混合方案 |
| DALL-E 图 + Veo 视频 | ✅ 必须拼 Veo 的英文锚定句 | 海外短剧 |
| 即梦图 + Veo 视频 | ⚠️ 不推荐 | 画风跨度大，锚定句也救不动 |

### 18.4 锚定句拼接位置

```
image prompt = 主体 + 外貌锚定 + 动作 + 环境 + 景别 + 画风词 + [视频AI 锚定句]
                                                          ↑
                                                       接在画风词之后
```

锚定句**不要**插进主体描述里（会扰乱构图），就放最后做"风格说明"。

### 18.5 同 AI 是最佳选择吗？

**是。** 理由：
- 同家训练数据闭环，无翻译损耗
- 跨家用锚定句桥接是 90% 还原，不是 100%
- 同家通常还有"参考图机制"互通（即梦的参考图机制喂自家图最稳）

跨家方案存在的价值是**省钱**（比如 Gemini 订阅免费出图，只在视频阶段付费）。

### 18.6 在 skill 里怎么落地？

- `global_style.md` 顶部记两个 AI（"目标视频AI" + "目标图AI"）
- 角色卡的 ① 定妆照 prompt 默认按视频 AI 风格写
- 跨 AI 时所有 image prompt 末尾追加视频 AI 的锚定句（见 `references/platforms.md`）
- 已建好的角色卡末尾要有"跨 AI 用法"区块（含中英双版锚定句），方便手动拼接

### 18.7 锚定句模板速查

视频 AI = 即梦 / Seedance：
```
中文：中国古代东方面孔，国风写实漫剧风格，电影级光影，暗黑宫廷氛围，皮肤通透感，竖版9:16
EN:   cinematic Chinese ancient-fantasy webcomic aesthetic, Eastern Asian face, dramatic chiaroscuro, dark palace atmosphere, vertical 9:16
```

视频 AI = 可灵：
```
中文：中国古代东方面孔，影视级国风写实，自然电影光感，皮肤通透真实质感，竖版9:16
```

视频 AI = Veo：
```
EN:   cinematic Chinese ancient-fantasy aesthetic, photoreal Eastern Asian face, dramatic film lighting, fine film grain, vertical 9:16
```

```
│   └── locations/             ← 场景卡 shared，由 n2d-script 写
```

### 19.3 决策表：什么放共享？什么放每集？

| 资产 | 放哪 | 理由 |
|---|---|---|
| 角色定妆 PNG/prompt | shared | 角色全集统一 |

```
4. 跑即梦（或 CLI 自动）→ 挑图 → PNG 落 出图/common/
5. 索引状态改 ✅
6. 出图/第N集/00_总览.md 引用 shared
7. 后续集自动复用，不再重做
```

### 19.5 进度算法（`_进度.md` 的 `出图` 列）

```
出图 列 = 已存在 PNG / 本集需要的所有 PNG（含 shared 复用 + 本集分镜）
```

举例 — 第2集（假设需要：沈念常态 ✅ + 沈念觉醒态 ✅ + 小禾 ✅ + 冷宫寝殿 ✅（第1集做的）+ 偏殿修炼室 ⬜ + 8 张分镜）：
- 总数 = 5 shared + 8 分镜 = 13
- 已完成 = 4（4 shared 已 ✅）
- 列填 `4/13`

复用让"已完成"数自然跨集累积——第2集开始时就有 4 个起步分。

### 19.6 为什么这种架构？

- **避免重复**：4 张定妆图存 100 次 = 400 张 PNG。共享后 = 4 张。
- **保证一致**：跨集偶然修改某张定妆会立刻让其他集不一致。集中管理避免漂移。
- **降低生成成本**：复用 = 无需重抽，省钱（即梦会员积分）省时间。
- **方便回溯**：所有角色"曾经画过什么样"集中在 `出图/common/角色定妆.md`，无需翻 N 个集子。

### 19.7 跟 `characters/` / `locations/` 的关系

- `common/characters/角色名.md` 是**人物设定文档**（小说级，剧情服务）
- `出图/common/角色定妆.md` 是**生产实战 prompt**（AI 服务，含即梦操作 SOP）
- 前者是设定，后者是执行。前者由编剧维护，后者由制作维护。
- 两者锁定的"妆造拆解"完全一致——`出图/common/` 只是把 `characters/` 的 ① 定妆照 prompt 加上即梦操作、检查清单、参考图依赖等"实战包装"。

## Q20：下一集出料时怎么自动复用前几集的定妆？SOP 是什么？<a id="q20"></a>

### 20.1 强制 5 步 SOP（每集出图 prompt 生成必走）

```
① 扫描共享库
   读 出图/common/00_索引.md
   盘清楚：已有哪些角色（含形态变体）/场景/道具，及状态（✅/⏳/⬜）
   ↓
② 列出本集需求
   读 脚本/第N集/分镜剧本.md + 素材清单.md
   提取本集需要的所有 角色/场景/道具/特殊视觉
   ↓
③ 差集 = 新增
   本集需求 − 共享已有 = 必须新加入共享库的项
   包括："首次出现的全新角色/场景/道具" + "已有角色的新形态变体"
   ↓
④ 追加共享库（仅新增项）
   出图/common/00_索引.md 追加 ⬜ 行
   出图/common/角色|场景|道具定妆.md 追加实战 prompt 块
   ↓
⑤ 建本集 prompt 文件夹
   出图/第N集/00_总览.md（含引用共享）
   出图/第N集/01_分镜出图.md（本集独有分镜）
```

### 20.2 实战例子：第2集 出图 prompt 生成

| 步骤 | 第2集 实际操作 |
|---|---|
| ① 扫描 | 共享库已有：沈念常态 ✅、沈念觉醒态 ✅、小禾 ✅、柳娘子 ✅ |
| ② 列需求 | 本集需要：沈念常态、小禾、偏殿修炼室、系统光幕、12 个分镜场景 |
| ③ 差集 | **新增**：偏殿修炼室 LOC_02、系统光幕（共享道具已占位但 prompt 未写满）|
| ④ 追加 | 共享 `场景定妆.md ②` 占位 → 完整 prompt 块；`道具定妆.md ②` 占位 → 完整 prompt 块 |
| ⑤ 建本集 | `出图/第2集/00_总览.md` 引用共享 + `01_分镜出图.md` 12 镜 |

**复用红利**：第2集开局 **2/16**（沈念常态 + 小禾 直接复用）

### 20.3 跨集累积效应

| 集 | 开局复用 | 新增 shared | 本集分镜 | 出图列 |
|---|---|---|---|---|
| 第1集 | 0 | 4 角色 + 1 场景 + 1 道具 = 6 | 13 | 4/19 |
| 第2集 | 2（沈念常态 + 小禾）| 1 场景 + 1 道具 = 2 | 12 | 2/16 |
| 第3集（预估）| 5+（前面 shared 全继承）| 蛇妖 + 三小妖 + 老槐树 + 枯井 等 | 8-12 | 起步分高 |
| ... | 越往后越累 | 越往后越少 | — | 分母小、分子高 |

到第10+ 集，新增 shared 基本只剩"该集独有的小角色"，每集做 prompt 只需 1-2 张 shared + 8-12 分镜。

### 20.4 跳过第 ① 步的代价（铁律）

```
不扫共享 → 把已存在的定妆当"新"建 → 跨集脸漂移 + 浪费即梦积分 + prompt 库重复
```

无论什么场景，**任何集生成出图 prompt 之前必先扫共享**。这条写进 `SKILL.md` 第3步 item 8 的"强制 SOP"。

### 20.5 死亡角色 / 一次性形态 还放共享吗？

**放**。统一规则 > 节省空间。柳娘子第1集就死了也放 shared——一致性比省 3MB 重要，且未来可能闪回。

### 20.6 跨集判定"是否复用"的 3 个常见误判

| 看起来"是同一个"实际可能不能复用 | 怎么处理 |
|---|---|
| 同一角色的不同形态（觉醒态 vs 银牌态）| **算新增**，建独立 shared 条目 |
| 同一场景的不同时间/光线（白昼 vs 夜晚）| 多数情况可以一张 PNG，分镜 prompt 改光线即可。若反差太大（晨光 vs 暴雨）才分两张 |
| 道具的不同状态（完好木梳 vs 碎成粉末木梳）| 看是否反复入镜——一次性的不进 shared，反复的才进 |

## Q21：skill 越攒越重，要不要拆？4 个 skill 还是单写 agent？<a id="q21"></a>

**结论：拆成 4 个 skill，不要单写 agent。**

### 21.1 起因（2026-05-31）

原 `novel2drama` 单 skill 在 100+ 集制作期间膨胀：拆集 + 全局/角色/场景 + 8 类素材 + 出图 prompt + 出图 CLI 操作 + 视频 prompt + 视频 CLI 操作——全揉在一个 SKILL.md（117 行）+ `references/formats.md`（323 行，含两层架构 §10）+ `references/platforms.md`（147 行）。

具体痛点：
- 任一集只在某一阶段时，无关阶段细节也被加载进上下文（如精修剧本时不需要 CLI 调用细节）
- 不同 AI 工具（图 AI / 视频 AI）的差异散布在多处
- 经验沉淀（本 Q&A）越累越多，单文件 873 行难翻
- 想加新功能（如本机 CLI 探测、并发生图）时不敢动——动哪儿都怕影响其他阶段

### 21.2 决策：skill vs agent

| 维度 | skill | agent |
|---|---|---|
| 用户能否直接触发 | ✅ `/n2d-image` | ❌ 只能被自动调起 |
| 阶段间是否需要人工 review | ✅ 留在主上下文，方便在线打回精修 | ❌ agent 隔离上下文，每次返回都丢状态 |
| 文件读写状态共享（`_进度.md`） | ✅ 都在主上下文里看 | ❌ agent 看不到 |
| 节省上下文 | ✅ 只在被调起时加载 | ✅ 但代价是失忆 |

**这个流水线是「分阶段 + 强交互（用户在线筛图、改本子）」**，正好是 skill 的主场。agent 适合"扔出去几个小时不管它自己干完"的并行工作。

### 21.3 4-skill 结构

```
.claude/skills/
├── novel2drama/                  ← Stage 0：调度（薄，~110 行）
│   ├── SKILL.md                  扫作品根 → 读 _进度.md → 推荐下一步该调哪个 skill
│   ├── Q&A.md                    全阶段沉淀（保留原位）
│   └── references/architecture.md  四阶段流水线总览 + 目录铁律 + 首跑示范
├── n2d-script/                   ← Stage 1：拆集 + 8 类素材
│   ├── SKILL.md
│   ├── scripts/split_novel.py
│   └── references/{formats,platforms}.md
├── n2d-image/                    ← Stage 2：出图 prompt + 生图
│   ├── SKILL.md                  两层架构 + 5 步 SOP + 扫 CLI + 调用/指导
│   └── references/{prompt_format,platforms,cli_registry}.md
└── n2d-video/                    ← Stage 3：出视频 prompt + 生视频
    ├── SKILL.md                  视频 prompt + 扫 CLI + 调用/指导
    └── references/{prompt_format,platforms,cli_registry}.md
```

### 21.4 关于"扫本机生图/视频 CLI 并调用"

- **扫描**：直接 Bash `command -v <name>`，配合 `references/cli_registry.md` 里一份"已知 CLI 清单"
- **调用**：**直接 Bash 调用即可，不需要 agent**
- **何时上 agent**：本集分镜 ≥10 张 / Clip ≥6 段，需要并发生图/生视频时，可 spawn 2-4 个 sub-agent 各跑一段 CLI；主线程统一收集 + 筛选 + 落档
- **第三方逆向 CLI 严禁装**（违 ToS + 封号），仅装官方（dreamina / gemini-cli / ...）

### 21.5 4-skill vs 单大 skill 的权衡

| 方面 | 4-skill | 单大 skill |
|---|---|---|
| 上下文清洁度 | ✅ 高 | ❌ 各阶段细节互相污染 |
| 用户命令记忆负担 | 4 个命令（调度兜底） | 1 个 |
| 阶段并行 | ✅ 第 K 集出图 + 第 K+1 集精修可并行 | 可以但容易混 |
| 公共逻辑重复 | platforms.md 在 3 个 skill 各一份 | 一份 |
| 新增功能扩展 | 改对应阶段 skill，影响范围小 | 改大文件容易破坏其他阶段 |
| 翻车修正定位 | 锁在某个 skill 内 | 全局都受影响 |

**结论**：未来功能只会越加越多，4 个 skill 的隔离收益 > platforms.md 复制 3 份的成本。

### 21.6 拆分的实施约定

- **进度表（`_进度.md`）是 single source of truth**，4 个 skill 都读写它
- **目录约定**（`作品根/{小说,common,脚本,出图,出视频}/` — common 装跨阶段共用资产 + 废料）由 `novel2drama/references/architecture.md` 统一管
- **Q&A.md 保留在 `novel2drama/`**，全阶段共用；各阶段 skill 在 SKILL.md 末尾指向 Q&A.md 的相关问题号
- **`platforms.md` 在 3 个阶段 skill 各放一份**（每份内容相同，~150 行）——self-contained 比共享路径黑魔法稳

### 21.7 后续可能的扩展

- Stage 4 合成（配音 + BGM + 字幕 + 剪辑）当前不在 skill 范围；未来若加入 `n2d-compose`，进度表追加 `成片` 列，调度规则向后扩展即可
- 自动化进度查询脚本可后续抽到 `novel2drama/scripts/`，目前人肉读表 + Claude 解读已够用
