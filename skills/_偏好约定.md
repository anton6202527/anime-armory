# 偏好约定（所有自定义 skill 共用 · 通用机制）

> 这是**共享代码侧**的通用约定，不含任何具体用户的选择。
> 用户的实际选择是**私有的**，落在用户自己的空间，绝不写进 skill 源码。
> 任何 skill 涉及「让用户选」的地方，都按本约定读写，不要把某个平台/后端写死成唯一路径。

## 三句话原则
1. **skill 通用**：不写死平台/后端/分辨率；这些是「选择点」，由用户私有文件决定。
2. **选择私有**：用户的选择存在用户自己的空间（见下），不进 git、不进共享 skill。
3. **持续记录**：选择点首次遇到 → 问一次 → 写入 `_设置.md` → 同项目之后**沉默沿用**；用户随时可改（说一声或直接改文件）。

## 两层私有存储

| 层 | 路径 | 谁写 | 作用 |
|---|---|---|---|
| **作品级** | `<作品根>/_设置.md` | skill 自动读写，用户可手改 | 本作品/曲/小说的实际选择（**权威**，覆盖全局） |
| **全局默认** | 仓库内 `.claude/创作偏好-默认.md`（随库多机同步） | 用户/Claude 维护 | 跨项目的个人默认（如「默认即梦/720p/CosyVoice/Suno」），开新项目时**预填** `_设置.md` |

- `<作品根>`：`制漫剧/<剧名>/`、`制MV/<曲名>/`、`写歌/<曲名>/`、`写小说/<项目>/` 等，与 `_进度.md` 同级。
- 全局默认是**仓库内固定路径** `.claude/创作偏好-默认.md`（本库为私有库，随库同步到各台电脑）；skill 文本引用「全局默认偏好文件」即指它。**仅当本库转为公开/共享模板时**才剥离该文件（见库根 `TODO.md`）。

## 解析顺序（每次要用某选择点时）
1. 读 `<作品根>/_设置.md` 里该选择点 → 有就**直接用，不问**。
2. 没有 → 读全局默认 `创作偏好-默认.md` → 有就用其值**预填**，并**首次告知一句**（"默认 X，要改说一声"），随即写进 `_设置.md`。
3. 全局也没有 → **问用户一次**，写进 `_设置.md`。
4. 之后同项目同选择点：**沉默沿用**，不再问。
5. 无作品根的独立调用（如 novel-title / song-lyrics 单用）：能定位项目就读其 `_设置.md`，否则用全局默认 / 现问，不强行落文件。

> 例外：**合规/不可逆/花钱多**的点（换脸源脸合法性、真人嗓授权、版权来源、删除回流）即便记录过也要确认——安全 > 省事。

## `_设置.md` 格式（人类可读 · 可手改）
```markdown
# _设置 · <作品名>   （你的私有选择；skill 自动读写，也可手改。改主意随时说）

## 选择
- <选择点>: <值>        # <可选值>
...

## 记录
- 2026-06-03 初始化（继承全局默认）
- 2026-06-03 生视频AI 即梦→可灵（用户指定）
```

## 选择点目录（各家族暴露给用户的选择 + 缺省回退）

> 缺省值仅作**全局默认的初始建议**；真正生效的是 `_设置.md` 里的值。

### 制漫剧 novel2drama / n2d-*
- `制作模式`: 配音先行 \| 先出视频后配音　（默认 **配音先行·推荐**：真实配音时长驱动镜头/Clip，音画准、返工少。`先出视频后配音`=**快速demo·不推荐**：镜头时长只能按台词文本估算锁死，后期补真实配音对不上 → 音画不同步/语速怪/可能重切重出视频；仅纯视觉demo/比稿用，过审后仍要回头补真音重定时。两种流程图 + 不推荐的完整理由 + 闸门放行方式见 **novel2drama SKILL「制作模式」节**——选 `先出视频后配音` 时各阶段入口必复述此警告）
- `单集时长`: 前长后短 \| 均衡 \| 快节奏 \| 长集 \| 自定义（均为**时长区间·验收窗口**）　（默认 前长后短：第1集 120–180s / 其余集 60–120s。**只锚①粗胚字数 band，不锚最终边界**——边界靠爽点/钩子等节拍重切、真实时长靠配音实测落进区间；三层模型与各预设区间见 n2d-script 第 1 步）
- `生视频AI`: 即梦 \| 可灵 \| Seedance \| Veo　（默认 即梦）
- `生图AI`: Codex only \| Codex \| 即梦 \| 同视频AI \| Gemini \| DALL-E \| Flux　（默认 Codex only；只用 Codex 内置生图/官方路由，禁止 Dreamina/即梦 CLI 兜底；显式选“即梦”时才允许即梦出图）
- `配音后端`: CosyVoice \| GPT-SoVITS \| MiniMax \| 火山 \| say占位　（默认 CosyVoice，缺凭证回退 say）
- `视频分辨率`: 720p \| 1080p　（默认 720p）
- `画幅`: 9:16 \| 16:9　（默认 9:16 竖屏短剧）
- `对口型`: 关闭 \| 平台原生 \| 后期pass　（默认 关闭；仅正面说话特写且预算允许时启用）
- `BGM来源`: 占位 \| 文件 \| Suno　（默认 占位）
- `字幕语言`: 中文 \| 中英双语 \| 仅英文　（默认 中文·国内投放；海外投放 TikTok/ReelShort/YouTube/北美短剧 选 中英双语 或 仅英文。**翻译是投放选择，不写死在 skill**——选中英/仅英时 n2d-script 才产 `字幕_英文.srt`，否则只产 `字幕_中文.srt`）
- `一致性增强`: 锚点+参考图 \| +LoRA　（默认 锚点+参考图，LoRA 不启用）
- `重抽预算策略`: 预算充足 \| 预算一般 \| 预算不足　（默认 预算充足：每张抽到出图自检三项全过为止，主要人物脸/妆造零漂移容忍；`预算一般`=保主要人物/关键镜过、配角差不多放行；`预算不足`=全程差不多就行最省 credit。详见 n2d-image「重抽预算铁律」三档表）
- `生成粒度`: 逐个 \| 小批 \| 按场景分批 \| 整集　（默认 **逐个**：每出 1 张图/1 个 Clip 就停下让用户审、可即时重抽/改 prompt 再继续；`小批`=每批 N 个一停，N 默认 5 可改；`按场景分批`=同场景连续镜一组一停；`整集`=旧行为一次过、最后统一报告。**任何档都先过共享定妆硬闸门**。⚠️**例外——每次都问**：本点属花钱/token 敏感点，**每次进出图/出视频前都要把四档菜单念给用户选一次，不沉默沿用**；`_设置.md` 里的值只作「默认建议/上次选择」预选，不替代询问。出图/出视频通用，菜单原话见 n2d-image / n2d-video「生成粒度 + 优先序」段）
- `生成优先序`: 关键镜优先 \| 分镜顺序 \| 先易后难　（默认 **关键镜优先**：🔑爽点/反转/钩子/封面候选先出，先定基调先锁主角脸；`分镜顺序`=Clip1→N 叙事序；`先易后难`=单人静态/空镜先、复杂打斗/多人镜后。共享定妆库永远排在最前，与本点无关）
- `脚本批次`: 逐集 \| 小批 \| 整批　（默认 **逐集**：n2d-script 量产剧本改编/分镜设计时每 1 集停下报**边界决策 + 物料清单**再继续；`小批`=每 N 集一停 N 默认 5 可改、附 5-10 集边界复核窗口；`整批`=指定范围一次过最后统一报告。文本不烧钱，**问一次后沉默沿用**，不像 `生成粒度` 每次弹菜单；只管"几集一停"不改三层拆集/P0→P6 边界铁律；第1集永远单独逐集。详见 n2d-script「脚本批次」段）

## 新增选择点 / 新家族
在对应家族下加一行（选项 + 缺省回退），并在该 skill 的「偏好」段引用本文件即可。**不要**在 skill 正文把新选项写成唯一硬路径。
