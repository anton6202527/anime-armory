# 立项访谈 — 把"几个字"补全成创作蓝图

> novel-create 第 0 步的引导手册。目标：从用户的**只言片语 + 碎片**，问出一份能动笔的 `创作蓝图.md`。
> 灵魂在两条：**① 一次问一组、给默认建议让用户确认（不轰炸）；② 用户给过的别再问，先吃再补缺。**

---

## 一、先吃碎片，再问缺口

用户往往不是白纸，他给了"几个字 / 部分风格 / 部分文件"。**先复述你的理解，确认，再补缺**——别让用户重答已给过的。

| 用户给的 | 你先做 | 收进哪 |
|---|---|---|
| 几个字 / 一句话点子 | 复述成一个**初稿 logline**让其确认/修正 | 蓝图 logline |
| 部分风格（"像XX那种""冷峻一点""多对话"） | 提炼成**风格卡草稿**（关键词/句子节奏/对白比/禁忌） | 蓝图 风格卡 |
| 风格样本文件 / 喜欢的段落 | `--ingest` 收进 `素材/`，**读它**抽文风特征 → 风格卡 | 素材/ + 风格卡 |
| 零散设定笔记 / 世界观片段 | `--ingest` 收进 `素材/`，整理进设定圣经草稿，标出**还缺什么** | 素材/ + 设定圣经 |
| 半成品章节 / 开头 | `--ingest` 收进 `素材/`，**读它**反推已隐含的 logline/主角/文风，当文风锚 | 素材/ + 蓝图 + Demo 锚 |

> 复述模板："我理解你想写的是【…一句话…】，风格偏【…】，已经有【…碎片…】。对吗？下面只问还没定的几处。"

## 二、必问六组（按顺序，一次一组）

每组给 **2-3 个候选默认值**，用户选一个或改一个即可，降低"从零想"的负担。

1. **题材 + 目标平台**（最先定，决定后面所有形态）
   - 题材：都市异能 / 玄幻修真 / 古言宅斗 / 科幻 / 悬疑 / 历史…
   - 平台：起点(男频长线) / 番茄(爽快下沉) / 晋江(女频细腻) / **抖音漫剧·红果(短章高频强钩)** / 历史向 / 跨平台
   - → 平台决定**规模档**与**爽点节奏**（漫剧档=1000-1500字/章、每章必爽必钩）。

2. **主角 + 金手指**（故事引擎）
   - 主角是谁、出身、性格底色（一句）
   - **金手指/核心能力**：是什么 + **代价/限制**（无代价的金手指=无张力，必须逼出代价）
   - 默认建议示例："系统流/重生/觉醒异能/老天爷给的天赋"各给一个钩子让其选。

3. **核心爽点**（这本"爽"在哪）
   - 打脸逆袭 / 扮猪吃虎 / 升级变强 / 复仇 / 种田经营 / 智斗 / 救世…
   - 问"读者翻这本书最想看到什么爽"，定 1-2 个主爽点类型。

4. **主线冲突 / 反派**（对抗什么）
   - 主角要对抗的最大阻力：人 / 势力 / 困局 / 命运
   - 一句话主线张力（"他要……，但……拦着”）。

5. **规模 + 视角**
   - 规模档：short(3章试水) / medium(20) / long(40) / 微短剧(50) / 漫剧(90短章)——平台已暗示，确认即可
   - 人称：第一 / 第三限制（默认第三限制）
   - 目标读者（一句）。

6. **风格 + 输出**
   - 风格卡（若一/未提炼）：文风关键词、句子节奏、对白vs内心戏比、禁忌词/套路
   - 没有明确风格 → 记"**Demo 后回填**"，用前 1-3 章定调
   - 输出：txt / docx / outline（默认全要）。

## 三、节奏与边界

- **一次一组、step gate**：问完一组、回填蓝图对应栏、再问下一组。别 6 组一次性甩出去。
- **给默认、可跳过**：用户说"你定"就用平台默认值填，并明确告诉他填了什么、可改。
- **缺口可延后**：风格、目标读者这类可"Demo 后回填"，不卡立项。但 **logline / 主角金手指 / 平台 / 规模**这四样必须当场定——它们决定骨架。
- **够了就开工**：四样硬项 + 一个主爽点 + 一句主线冲突 = 可以跑 `init_project.py` 建骨架了，剩下的边写边补。别把访谈拖成无止境问卷。

## 四、产出对照（访谈 → 蓝图字段）

| 访谈得到 | 写进 创作蓝图.md |
|---|---|
| 初稿 logline | ## 一句话故事 |
| 题材 + 平台 + 基调 | ## 题材 / 平台 / 基调 |
| 主角 + 金手指(代价) + 动机 | ## 主角 |
| 主爽点 | ## 核心爽点 |
| 主线冲突 / 反派 | ## 主线冲突 |
| 目标读者 | ## 目标读者 |
| 规模档 + 人称 | ## 规模 / 视角 |
| 风格卡 / 样本 | ## 风格卡（+ 素材/） |

访谈结束 → 跑 `init_project.py`（第 1 步）→ 把上表填进 `创作蓝图.md`（第 2 步）→ 用户审。
