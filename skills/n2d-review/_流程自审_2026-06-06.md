# n2d 流程自审 2026-06-06

模式②（流程自审）。联网拉 2026-06 市场基准，按三轴 + 模型演进对照 n2d 各 skill + Q&A，只记**真差距**。
默认只产建议，不自动改产线——下方每条带落点，待你拍板再由对应 skill 执行。

> 触发背景：今日新增 `制作模式`(配音先行/先出视频后配音) + `生成粒度/优先序` + `脚本批次`，并补全 n2d-progress 模式路由 + n2d-compose 真音拟合。本次自审确认这几项已闭环（见末「今日改动复核」），重点转向**更广的产线 vs 当前 SOTA**。

## 三轴取证摘要（采集 2026-06，含来源）

- **一致性**：市场已从"每次喂参考图"升级到**可复用角色ID / 数字身份绑定**——Kling 3.0 主体库、Seedream 5.0 Universal Reference（免 LoRA 跨图锁人）、Sora 2 Character Cameo、SenseTime SEKO "数字身份绑定+多角度预设+风格锁"；资产管理用**正/侧/背三视图 + 场景四视图**，"改一次人物资产全剧数千镜头自动同步"；可灵跨集角色相似度 95%+。图模型多参考已到 Seedream 4.5 **14 张**参考、Nano Banana Pro 8 张 + 原生 SynthID 水印。
- **效率**：全链路一句话→自动多分镜（可灵 3.0 一次通过率 90%+）；周期"月级→天级"；单 Clip 更长 = 更少拼接、更少漂移——**Seedance 2.0 单镜 15s**、Kling 3.0 单 clip 最多 6 镜。
- **可控性**：**原生音画联合生成**成主流——Veo 3.1 48kHz 同步对白、Seedance 2.0 原生音视频、Kling Omni 原生 5 语种口型；"完整段落一次出音效/口型/音乐/画面同步"。开源口型 2026 最佳免费 = **MuseTalk**（> Wav2Lip/LatentSync）。
- **模型演进**：图 = Nano Banana Pro(Gemini 3 Pro Image)/GPT Image 2/Seedream 5.0·4.5/Flux 2；视频 = Veo 3.1 / Kling 3.0 Omni / Seedance 2.0；配音 = IndexTTS-2(音色/情绪解耦)/Fish Speech V1.5/VoxCPM2(48k·30语)/Chatterbox/CosyVoice2。

来源见文末。

## 差距清单（按优先级）

### [must] G1 · 防过期：缺"模型矩阵 + 升级触发"快照
- **差距**：各 `platforms.md/backends.md` 散落具体版本名（即梦 Seedance2.0 / 可灵3.0 / Veo3.1），但**没有一处**"当前各轴 SOTA vs n2d 默认 + 何时升级"的对照，版本名会迅速腐烂——这正是本 skill「防过期铁律」要治的元问题。
- **证据**：opencreator / aimlapi / atlascloud 三份 2026 对比（见文末）；本次自审本身就因无快照而要重新满网捞。
- **落点**：新建 `skills/novel2drama/references/模型矩阵.md`（或 Q&A 新条目），三轴各一行：SOTA 能力 / n2d 默认 / 升级触发条件 / 采集日期。由本 skill 每次自审刷新。**写能力不绑版本号**，版本号留在档案。
- **可脚本化**：否（人工刷新）。

### [must] G2 · 单 Clip 时长上限"一刀切 ≤8s"已落后
- **差距**：`n2d-video` 故事板按"即梦 ≤8s 超限拆 Clip"为通用上限。但 Seedance 2.0 单镜 **15s**、Kling 单 clip 多镜——更长单镜 = 更少拼接缝 = **跨镜漂移更少 + 更省**。一刀切 ≤8s 会把本可一镜到底的段落切碎，反伤一致性与成本。
- **证据**：vidguru / 3daistudio 2026-初评测（Seedance 2.0 15s、Kling 6 镜/clip）。
- **落点**：`n2d-video/references/platforms.md` 把"单 Clip 上限"改成**按后端档案**（即梦≤8 / Seedance≤15 / Kling 多镜）；`n2d-script` 阶段2 故事板拆 Clip 时读该上限，而非写死 8s。
- **可脚本化**：部分（拆 Clip 逻辑读档案值）。

### [optional] G3 · 定妆多视图：缺"背面" + 场景四视图
- **差距**：n2d 定妆组 = 正脸/侧脸/半身（3 张），市场标准已是角色**正/侧/背三视图** + **场景四视图**。缺背面会让 over-shoulder / 追逐 / 转身镜没有锁定参考 → 易崩。
- **证据**：sohu 2026 工业化测评、cnblogs 平台测评（正侧背三视图 + 场景四视图）。
- **落点**：`n2d-image/references/角色一致性checklist.md` + `prompt_format.md §定妆`：把背面纳入（至少打斗/追逐题材必出），场景定妆补四视图建议。
- **可脚本化**：否。

### [optional] G4 · 未利用"平台原生角色ID / 主体库"
- **差距**：n2d 把每次出图都当 image2image 重喂参考图；当后端是 Kling 主体库 / Seedream 5.0 Universal Reference / Sora Cameo 时，平台能**注册一次角色 ID、后续按 ID 引用**，比每次喂图更稳更省。n2d 没有这条 opt-in 路径。
- **证据**：gitcode 2026 漫剧工具报告（可灵主体库）、artlist（Seedream 5.0 Universal Reference）。
- **落点**：`n2d-image` SKILL 加 opt-in「后端原生角色ID」段（后端支持则注册→按 ID 引用，不支持回退现有参考图派生）；`platforms.md` 标各后端是否支持。
- **可脚本化**：否（后端能力相关）。

### [optional] G5 · 定妆改动无"下游受影响镜头"传播
- **差距**：市场卖点"改一次人物资产，全剧镜头自动同步"。n2d 定妆库虽共享，但改了某 `定妆_X.png` 后**没有机制列出哪些已出镜头引用了它、需重出**——靠人记。
- **证据**：sohu / cnblogs（资产改一次全剧同步）。
- **落点**：新脚本 `n2d-image/scripts/asset_impact.py`（给定改动的定妆名 → 扫各集分镜 prompt/产物，列出引用镜头 + 标记重出），并入 `n2d-review` 机检家族。
- **可脚本化**：**是**（可进 mechanical_check 体系）。

### [optional] G6 · 口型/配音后端登记滞后于 SOTA
- **差距**：① 口型只列 LatentSync/Wav2Lip，2026 最佳免费已是 **MuseTalk**；② 配音后端缺 **IndexTTS-2**（音色/情绪**解耦**——正好服务 n2d 的情绪/语速念白标注）、Fish Speech V1.5 / VoxCPM2(48k)。
- **证据**：lipsync.com / pixazo（MuseTalk 2026 最佳）、siliconflow / findskill（IndexTTS-2 情绪解耦、Chatterbox 盲测胜 ElevenLabs）。
- **落点**：`n2d-video/references/platforms.md`(对口型工具表) 加 MuseTalk；`n2d-voice/references/backends.md` 加 IndexTTS-2(情绪解耦)/Fish Speech/VoxCPM2。纯档案更新，不动主流程。
- **可脚本化**：否。

### [optional] G7 · 默认图后端 Codex-only 的一致性权衡未点明
- **差距**：用户显式选 Codex-only（尊重该选择，不改默认）。但市场一致性第一梯队是 **Seedream 4.5/5.0（14 张参考·Universal Reference）+ Nano Banana Pro（多参考 + 原生 SynthID 水印）**。对一致性吃紧的项目，多参考后端可能优于 Codex-only；且 Nano Banana 的 **SynthID 加密水印**天然满足 n2d 强制 AI 标识合规闸门。
- **证据**：atlascloud / melies / laozhang 2026 图模型基准。
- **落点**：`n2d-image` SKILL 在 `生图AI` 选择点旁加一句**知情权衡**（一致性关键项可选多参考后端；Nano Banana SynthID 可作合规水印的原生选项）。仍不改 Codex-only 默认。
- **可脚本化**：否。
