# novel-score 评分细则(维度 / 权重档 / 档位 / 判定)

> 百分制。每维先打 **1-10 原始分**,再 ×(权重/10) 汇成百分。题材热度维度必须对标当次联网拉取的 `评分/题材热榜_<date>.md`,不是泛评。

## 七维定义 + 看什么(1-10 锚点)

| 维度 | 看什么(高分) | 低分信号 |
|---|---|---|
| **① 题材热度匹配** | 题材/金手指/人设正踩当下红果·抖音·番茄热榜;套路是上升期不是退潮期;有差异化记忆点 | 题材已退潮/烂大街无新意;与目标平台读者口味错位 |
| **② 开篇黄金三章钩子** | 前300字立冲突/钩子,前3章给身份反转·爽点·悬念,代入快,不铺设定 | 慢热、先交代世界观、主角动机模糊、第1章无钩子 |
| **③ 爽点密度与情绪节奏** | 打脸/逆袭/扮猪吃虎/反转密集,憋—放有度,情绪曲线上抬 | 平铺直叙、爽点稀、节奏拖、爽点给得太碎或太散 |
| **④ 人设与金手指** | 主角立得住有记忆点,金手指吸引力强且有成长想象空间,不圣母不降智不恋爱脑 | 主角脸谱化/降智/圣母,金手指无趣或万能到没张力,配角工具人 |
| **⑤ 剧情结构与主线张力** | 主线清晰、目标明确、悬念递进、章末钩子、伏笔有收、不注水，并持续推进 `设定/读者契约.md` 的核心题旨 | 主线散、流水账、注水拖更感、断更/烂尾迹象、东拼西凑，或写着写着偏成另一本书 |
| **⑥ 文学性 / 文笔** | 画面感、show-don't-tell、对白有戏、文风统一、少AI味；文字准确、可感、有人味，并符合读者契约里的文学质感 | 大段直白告知、用词重复、文风漂、AI味/同质化、辞藻堆砌但不服务人物或叙事 |
| **⑦ 完读 / 留存潜力** | 章末cliffhanger硬断、每章有信息增量/卡点、追读欲强、单章信息密度合适，且读者承诺有递进/延迟/兑现 | 章末收太干、平淡过渡多、读着想划走、信息密度过低或过载，前文承诺长期不推进 |

## 权重档(按 `目标平台` 选)

每维权重(合计 100)。原始分(1-10)×(权重/10)= 该维百分得分。

| 维度 | 商业爽文向(红果/抖音/番茄·**默认**) | 品质向(起点/晋江/历史) |
|---|---|---|
| ① 题材热度匹配 | 20 | 12 |
| ② 开篇黄金三章钩子 | 18 | 14 |
| ③ 爽点密度与节奏 | 18 | 12 |
| ④ 人设与金手指 | 12 | 14 |
| ⑤ 剧情结构主线 | 12 | 18 |
| ⑥ 文学性/文笔 | 8 | 18 |
| ⑦ 完读/留存潜力 | 12 | 12 |
| **合计** | **100** | **100** |

> 跨平台/历史向等其它平台:在两档间按"越偏短视频流量越靠商业向、越偏付费深度阅读越靠品质向"插值,并在报告里写明所用权重。

## 雷点扣分项(命中即在总分上再减,单独列)

不是维度,是**红线/退潮信号**,每条按程度 −3 ~ −10:

- 开篇慢热(前3章无钩子) −5~10
- 题材已明显退潮/被平台限流 −5~10
- 主角降智 / 圣母 / 恋爱脑(商业向尤其致命) −3~8
- 注水拖更、单章无信息增量 −3~8
- 题旨偏移 / 读者承诺违约（连续多章不推进核心题旨或偷换开篇承诺）−4~10
- 三观雷 / 价值观踩平台红线(可能限流封禁) −5~10(命中红线可直接判弃稿/重写该部分)
- AI味重 / 与市面雷同到无识别度 −3~6
- 文学质感明显不匹配目标平台（品质向尤重）−3~8
- 烂尾 / 明显断更感(完本评估时) −3~8

## 书名体检（附加体检项，不计入百分制总分）

评分时若 `_meta.json.title` 已定名，**顺带体检现有书名**——维度沿用 `novel-title/references/title-patterns.md` 的 5 维（各 1-5 分，满分 25）：钩子 / 平台契合 / 角色识别 / 抗撞名 / 可记忆性。

- **不进总分**：书名换起来便宜，不该拖累稿子本身的档位与判定；单列在报告里。
- **抗撞名只做初判**：score 阶段凭常见度估 + 读已有的 `设定/书名撞名检查_*.json`（novel-title 落的）；联网查重仍归 `novel-title`。`unchecked ≠ 不撞名`。
- **needs_rename 条件**：总分 < 15/25，或撞名检查为 `hard_collision`，或 LLM 显式判 needs_rename。
- **路由**：needs_rename 且判定非「弃稿重立」→ `next_actions[]` 追加 `novel-title`（重出候选 + 联网撞名检查）；弃稿重立时不路由（重开自带新名）。
- 书名未定 → 跳过本项，报告里建议先用 `novel-title` 起名。
- **本 skill 只体检现有书名、不出候选**——候选生成 / 排名 / 回写 `_meta.json.title` 全归 `novel-title`。

## 档位 + 判定 + 改写ROI

| 总分 | 档位 | 判定 | 改写ROI 建议 |
|---|---|---|---|
| **≥85** | 爆款潜力 | **过** | 微调即可投/继续写;别过度打磨耽误更新 |
| **70–84** | 合格偏上 | **小改** | 高ROI:润色 + 强化最低的1-2维(常是开篇钩子/爽点),委托 `novel-expand` 或自润 |
| **55–69** | 及格线下 | **大改** | 看题材:**题材对+主线有救**→值得结构级改写(`novel-rewrite` 改主线/换设定,或 `novel-continue` 救烂尾);**题材退潮**→ROI低,倾向重立 |
| **<55** | 不及格 | **弃稿重立** | 多数情况硬改不划算:题材/主线换掉重来更快,委托 `novel-create`;仅当核心创意真有亮点才保留要素重写 |

**判定必须显式回答用户的两问**:
1. **能不能火**:总分+档位+对标热榜一句话结论。
2. **要不要继续改、改哪**:判定 + 改写ROI + 具体到"改第几章、补哪几维、委托哪个 skill"。

## 报告结构(`评分/评分报告_<date>.md`)

1. **概览表**:七维原始分/加权得分 + 雷点扣分 + 总分 + 档位 + 判定(一屏看完)
2. **逐维**:得分 / 证据引文(原文) / 短评 / 改进抓手
2.5 **书名体检**(附加项):5 维分 + 撞名状态 + 换名结论(见上节)
3. **读者契约对齐**:对照 `设定/读者契约.md`,说明核心题旨、读者承诺、文学质感是否被正文兑现
4. **题材热榜对标**:引用 `题材热榜_<date>.md`,点名"踩中/错过/踩雷"了哪些当下热点,标基准日期
5. **判定 & 改写ROI**:能不能火 + 要不要改 + 改哪 + 委托路由
6. **下一步**:具体 skill 调用建议(`novel-rewrite`/`novel-expand`/`novel-continue`/`novel-create` → 之后 `novel-review`)
