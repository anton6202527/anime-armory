## 横切：音画联合生成趋势（可控性）

市场正走向**原生音画联合生成**（Veo 3.1 48kHz 对白、Seedance 2.0 原生音视频、Kling Omni 原生口型），"一次出音效/口型/音乐/画面同步"。

n2d 的取舍是**配音先行 + 剥离 clip 原生音轨**（`-an`，避免与克隆配音双人声；环境底用 `KEEP_CLIP_AUDIO=1` 低音量混入）——这是为了**角色音色一致性 + 节奏可控**的主动选择，不是落后。但每次刷新本表时复核：当原生联合生成的口型/对白质量与可控性追平克隆链路时，是否给"原生音画"开一条 opt-in（尤其纯视觉 demo / 比稿）。

**口型 lip-sync（正面说话特写才值得，默认不做）**：

| | 现状 |
|---|---|
| **SOTA 快照(2026-06)** | 平台原生：Kling Omni 原生 5 语种口型、Veo 3.1 原生对白。后期开源：**MuseTalk**（口型+画质均衡、近实时，当前最佳免费）/ Wav2Lip（SyncNet 同步精度最稳）/ LatentSync（扩散、身份保持好） |
| **n2d 默认** | 默认不做；启用时优先平台原生（可灵 3.0），否则后期 pass 首选 MuseTalk（见 `n2d-video` SKILL「对口型 lip-sync」节） |
| **升级触发** | 正面说话特写多、口型对不上很跳时启用；规模化/真人底片 → Wav2Lip。能不做就靠分镜规避（侧脸/背身/空镜配旁白）最省 |
