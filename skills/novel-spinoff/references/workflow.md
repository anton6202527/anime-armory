# 工作流详解

SKILL.md 是骨架，这里展开各步的"怎么做"。

## 三种时间线模式的差异

| 维度 | 并行外传 | 续接后传 | 分叉外传 |
|---|---|---|---|
| 锚点表 | **必做**，全本扫描 | 只看原作结尾几章定状态 | 分叉点之前的锚点必做，之后忽略 |
| 自由度 | 最低（每个锚点都是硬约束） | 最高 | 中等 |
| 风险点 | OOC / 时间线漂移 / 既定结果冲突 | 续接突兀 / 把原作角色当工具人 | 分叉点选不准 / 平行宇宙感弱 |
| 适合用户 | 想"补完"原作留白 | 想"接着原作往下走" | 想"如果当初 X 选了另一条路…" |

模式在 `init_project.py` 时锁定，后续不应切换。要切只能新开项目。

## 第 2 步深做：留白清单怎么挖

并行外传的核心张力源于"原作里这角色为什么这样做？作者没说"。挖法：

1. 把锚点表里 `mention` 和 `anchor` 都过一遍。
2. 每个 anchor 问三个问题：
   - **动机**：这一刻他为什么这么做？原作给了理由吗？
   - **情绪**：原作描写了他的反应吗？还是只描写了主角看到他这么做？
   - **空白时段**：这次出场到下次出场之间，他在干嘛？原作没写。
3. 每找出一个"原作没说"，记一行到 `角色卡.md` 的"留白清单"。
4. 续写就是把这些留白填进去——这是新文的"私货密度"，也是读者来读外传的理由。

## 第 3 步：书名候选

见独立文档 `references/title-patterns.md`。要点：

- 设定卡完工**之后**才生成候选，确保候选名带得起角色张力。
- **5–8 个候选**，主推平台 / 跨平台稳健 / 高钩子尝试 / 保守雅名 按 2-3 / 1-2 / 1-2 / 0-1 配比。
- 每个候选 5 维评分（钩子 / 平台契合 / 角色识别 / 抗撞名 / 可记忆性，各 1-5 分，满分 25）。
- 用户已有暂定名 → 作为候选 #0 一并打分。
- 候选表写到 `设定/书名候选.md`；用户选定后写进 `_meta.json` 的 `title` 字段。

## 第 4 步：章纲编织法（30+ 章长篇）

**三幕结构 + 锚点织网**：

- **第一幕**（约 1/4 章数）：建立配角的独立世界——他是谁、他在乎什么、原作开头时他在哪里。前几章可以**完全无锚点**（他和主角线还没交汇），把读者拉进他的视角。
- **第二幕**（约 1/2 章数）：和原作主线交汇 + 主要锚点密集出现。每个锚点章前后做一对"赴约章 / 复盘章"——锚点前他怎么走向那一刻、锚点后他怎么消化。这是质感的来源。
- **第三幕**（约 1/4 章数）：他自己的高潮——可以是和原作结局并行的另一条线收束，也可以是原作主线没看到的暗线终局。

**反转点至少两次**：一次在第二幕中段（第 12-15 章左右），一次在第三幕开头（第 22-25 章左右）。

**钩子**：每章末尾留一个"下一章你想看的东西"——悬念 / 误会 / 决定 / 新人物。

## 第 5 步：Demo 节奏（最重要 gate）

**为什么必须有这一步**：30 章里写错的视角/口吻/锚点对齐方式会一路传染，第 1 章错的，第 30 章一定也是错的。Demo gate 把"错了改"的成本压在前 3 章。

**Demo 章数**：
- 长篇 30+ 章 → 前 3 章
- 中篇 8-12 章 → 前 2 章
- 短篇 → 整本就是 demo 量级，跳过本步

**操作节奏**：
1. 写第 1 章。**主对话立即停下**，把章节贴给用户看 + 附 Demo 审核 checklist（见 SKILL.md 第 5 步）。
2. 用户给意见：
   - **过** → 写第 2 章，重复。
   - **要改** → 在第 1 章 Edit 改 + 把意见沉淀到 `设定/角色卡.md` 的 "口吻习惯/不要做的事" 段（这会喂给第 6 步的子代理 prompt）。再次给用户审。
3. 三章都过 → Demo gate 通过，进第 6 步。

**反模式**：
- 一口气写完 3 章再贴 → 第 1 章错的口吻传染到 2-3 章。
- 用户只口头说"挺好"就过 → 让用户对 checklist 逐项过一遍，避免漂在第 10 章才发现"其实当时就有问题"。

## 第 6 步深做：子代理 prompt 模板

派给每个写章子代理的 prompt 大致这样组织（**不要把这段照抄给子代理，按本章实际内容填**）：

```
你是<原作名>的同人作者，正在写一部从 <配角名> 视角的 <模式> 外传。

## 全局设定
[贴：角色卡.md 全文 — 包括"口吻习惯/不要做的事"段，这是 Demo 沉淀的隐式指南]

## 世界观
[贴：世界观.md 全文]

## 章纲（全本）
[贴：章纲.md 全文 — 让你知道前后章在干嘛]

## 文风样本（必须对齐）
[贴：章节/第01章.md 全文 — Demo 第 1 章是用户审过的文风锚点，本章必须保持同样的句长/节奏/词汇密度]

## 你要写的：第 N 章 《<标题>》
- 本章目标字数：5000-8000
- 主线事件：[章纲里的本章 outline]
- 视角情绪基调：[一句话]
- 涉及的锚点（必须对齐）：
  - 锚点 A：原作第 X 章发生 Y 事件，<配角名> 在场，状态 Z，对话要点 W
  - 锚点对齐方式：用事件骨架对齐，不要照抄原作原文
- 上一章末尾衔接：
  [上一章最后 ~500 字]
- 本章末尾要留的钩子：[一句话]

## 写作守则
[贴：pov-craft.md 全文]

## 输出
直接输出本章 markdown：第一行是 `# 第 N 章 <标题>`，之后是正文段落。不要加额外注释或元信息。
```

子代理写完，主对话把它的输出写到 `章节/第NN章.md`，再用 `python3 skills/novel/progress.py set "<作品根>" 第N章 正文初稿 ✅` 更新进度。

## 第 7 步：一致性扫描清单

每组 5 章后跑这个轻量列表：

- **人名**：grep 抽查每个主要角色名出现频次 / 称谓是否前后一致（"师兄"→"道长"漂移？）
- **地名**：grep 同上
- **术语**：修炼境界 / 魔法体系名词前后是否一致
- **时间标记**：本章里说"半年前"，半年前到底是哪章？
- **回忆 / 闪回**：闪回的事件别在主线还没发生时就被回忆了

全本写完跑全量扫描（见 SKILL.md 第 7 步）。

## 何时让用户介入

- **第 2 步后**：报告锚点数、留白点数，让用户拍板要不要调整。
- **第 3 步后**：书名候选 + 推荐 → 用户选定一个。
- **第 4 步后**：章纲必须用户确认。
- **第 5 步**：Demo 每章独立审。**这是最重要的一组 gate**，gate 没过不进第 6 步。
- **第 6 步每写完一组 5 章**：摘要报告 + 用户可决定是否调头。
- **第 7 步全量扫描后**：报告问题清单 + 用户可决定是修小问题还是返工。
