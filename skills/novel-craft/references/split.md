# 拆分标准 — 章 / 集 边界

适用于所有 novel-* 写作 skill（spinoff / expand / continue / condense / 完全原创）的章节划分。**节拍优先，字数兜底**：一章是一个戏剧节拍单元，不是一个字数桶。

本文件是 novel-* 家族的"拆分标准"说明；机器单一真值源在 `novel-craft/scripts/contract.py`。n2d-script 的 `split_novel.py` 是同一标准在漫剧生产线上的脚本实现。

## 核心原则

- **章 = 戏剧节拍单元**：一个完整的"冲突 → 升级 → 反转 / 决断 → 钩子"。
- **章 ≠ 字数硬切**：宁可前后章字数略不均，也别为了凑字数把高潮戏拦腰断成两章。
- **章 ≠ 原作章**：派生作品（spinoff / expand / continue / condense）的新章节边界由你按节拍重切，**和原作章节不必一一对应**。
- **章数随平台**：漫剧友好 vs 短剧 vs 网文长篇 vs 短篇，章数 / 字数差一个量级。
- **粗胚优先**：先按规则切粗胚脚手架，写完后可重切（合章 / 拆章），章节文件名按 `第NN章.md` 重排即可。

## 字数分档（按目标平台）

| 平台 / 用途 | 字数/章 target | 范围 min - max | 章数 / 100k 字总长 | `novel-create --scale` |
|---|---|---|---|---|
| **抖音漫剧** / **红果短剧**（~90 秒/集） | **1000–1500** | 800 – 1800 | ~70-100 | `漫剧` |
| **微短剧**（~3 分钟/集） | 1500–2500 | 1200 – 3000 | ~40-70 | `微短剧` |
| **网文中篇**（起点 / 番茄连载） | 3000–5000 | 2500 – 6000 | ~20-35 | `medium` |
| **网文长篇**（起点 V 后） | 5000–8000 | 4000 – 10000 | ~12-20 | `long` |
| **短篇集** | 6000–10000 | 5000 – 15000 | ~10-17 | `short` |

> **机器配置**：`novel-craft/scripts/contract.py` 的 `SCALE_PROFILES` 与本表 1:1（每档 band 严格相等）。改分档先改 `contract.py`，再同步本说明和测试。

**判断口诀**：节拍 1-2 个 → 1000-2500 字；节拍 3-4 个 → 3000-8000 字。

## 双约束（target / max / min）

- **target**：理想字数（按上表）。
- **max**：硬上限。超过时**强制在段或句边界切开**——但只在已超 max 时切，不在 target 时硬切。
- **min**：硬下限。低于 min 时**并入相邻章**——但只在尾章 / 没法补足时这么做。
- 默认 ratio：`min ≈ target × 0.5`，`max ≈ target × 1.4`（与 n2d-script split_novel.py 同源）。

## 切分边界（优先级从高到低）

1. **戏剧节拍**：节拍闭合处 = 章末（章尾必有钩子）
2. **原作章节**（派生 skill 中）：原作章边界可作参考但非必须
3. **段落**：自然段边界
4. **句子**：句末标点 `。！？……"`
5. **绝不在词中或短语中切**

## 流程

1. **估总长** —— 按 target_platform 反推总章数：总章数 = 总字数 / target。
2. **定弧线** —— 按章数划三幕（≥12 章时）。详见 `outline.md`。
3. **列节拍** —— 每章一个核心节拍，节拍序列就是章纲。
4. **写章纲** —— 详见 `outline.md`。章纲未敲定不进逐章写。
5. **粗胚可重切** —— 写好的章节可合 / 拆，按节拍调整。

## 派生作品的特殊约束

- **spinoff 并行外传**：每个原作锚点至少 1 章覆盖；锚点密集段可拆成"赴约 + 现场 + 复盘"3 章。
- **expand 扩写**：每个原章节扩到 N 个新章节（N = expand ratio）；新章数 ≈ 原章数 × ratio。
- **condense 精简**：相邻同主题章合并；纯支线章砍成一句话或全删。
- **continue 续写**：新章节字数 / 节奏对齐原作末几章的字数 / 节奏，便于读者衔接。

## 元数据剥离

原作转 .txt 后，需剥离这些"非正文"开头：
- 书名《...》/ 简介 / 内容简介 / 一句话简介
- 作品标签 / 看点 / 一句话剧透
- 作者 / 字数 / 平台信息
- 以 `【` / `✅` / `★` / `#` / `—` 开头的元数据行

派生 skill 的 `init_project.py` 应自动剥（n2d-script 的 `strip_frontmatter` 已实现；本 craft 文件只规定规则）。

## 反模式

- **按字数硬切节拍** —— 到 1000 字就切，把高潮戏拦腰断。
- **章数对齐原作章数** —— 派生作品强行 1:1 跟随原作章节，节奏全乱。
- **不留钩子** —— 每章结尾平淡，读者无动力翻下一章。
- **字数过偏** —— 一章 500 字、下一章 12000 字，读者节奏被打散。
- **过早 fixate 章数** —— 章纲写到中段发现节拍不够 / 太多，要敢调整章数。

## 与 n2d-script 的关系

- **概念源** = 本文件（novel-craft/references/split.md）
- **漫剧脚本实现** = `n2d-script/scripts/split_novel.py`（同一标准的工具版）
- 写漫剧脚本 → 直接用 n2d-script；写新小说（spinoff / expand / continue / condense / 原创）→ 引本文件指导章纲编织

novel-* 家族**不依赖** n2d-script，但二者共享同一概念。
