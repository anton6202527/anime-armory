# 钩子库 + 爽点配比守则

`chapter.md` 要求"章末必有钩子"、`outline.md` 要求"爽点节奏"，本篇给**怎么设计**：钩子分类、强度判据、失效检测，以及爽点的波浪式配比。写章、章纲编织、`novel-review` 查"钩子失效/爽点平淡"时都读。

## 钩子五型（章末/段末留悬念，按类型混用，别一种钩到底）

| 型 | 钩什么 | 例 |
|---|---|---|
| **悬念钩** | 一个未解的问题 | "门后站着的人，他十年前亲手埋了。" |
| **转折钩** | 局势骤变 | 眼看赢了，最后一句反转成输。 |
| **危机钩** | 迫在眉睫的威胁 | "三天后，全城都会知道他是谁。" |
| **信息钩** | 读者知道、角色不知道的差 | 主角信任的人，上一章刚被揭是内鬼。 |
| **关系钩** | 人物关系的温度突变 | 冷淡多章的两人，章末一个反常的举动。 |

## 钩子强度判据（弱钩留不住人）

- **具体 > 抽象**：钩"门后是他埋过的人" > 钩"事情没那么简单"。
- **代入主角的得失 > 旁观**：威胁/机会落在主角头上，读者才有损失厌恶。
- **可预期一个动作 > 纯气氛**：好钩子让读者**预判下章第一个动作**；只渲染气氛不算钩。
- **越临近高潮越强**：卷末钩 > 章末钩 > 段末钩，强度分级（配合 `arc-pacing.md`）。

## 钩子失效检测（`novel-review` 机检 + 人判）

- ❌ **钩子与下章无关**：本章末钩 A，下章开头却写 B，A 被遗忘 → 读者被骗一次就掉信任。
- ❌ **钩子不兑现/烂尾**：抛出大悬念后多章不回应，或回应得很敷衍。
- ❌ **同型钩连用**：连续 N 章都用"危机钩"，钝化。轮换钩型。
- ❌ **假钩**：钩子句很玄，揭晓时是无意义的虚晃。
- 检查：每章末钩 → 在 `设定/foreshadowing_ledger.json` 或章纲里标"将在第几章回应"，回应章勾销（见 `foreshadowing.md`）。

## 爽点配比：波浪，不是匀速打脸

网文的"爽"来自**期待→延迟→兑现**的呼吸，而非频繁打脸。匀速打脸 = 钝爽、显廉价。

- **虐—期待—爆发**：先压（受挫/被轻视/失去）→ 蓄读者的期待 → 在读者最想看时给高潮兑现。压得住，爆才响。
- **爽点类型要杂**：打脸/逆袭/扮猪吃虎/获得/解谜/守护/被认可——别只有打脸一种。
- **大小爽错落**：小爽（单章见效）密布日常章，大爽（卷级翻盘）压在卷末。参考节律："三章铺垫蓄势 → 一章中爽 → 多章推进 → 卷末大爽"，按题材松紧调。
- **延迟有度**：蓄太久不兑现 = 憋屈弃书；从不延迟 = 没有期待、爽得廉价。delay 以"读者开始着急但还没失望"为限。
