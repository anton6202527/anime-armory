# 读者契约 / 题旨契约

本文件定义每部小说在 Demo 通过后必须固化的“读者契约”。它不是宣传语，也不是口号，而是后续每一章写作、审稿、评分时共同对齐的质量锚点：**这本书到底在讲什么、答应读者什么、靠什么好看、文字质感该守在哪里**。

路径建议：

```text
设定/读者契约.md
```

## 什么时候写

- `novel-create`：创作蓝图通过后先写一版；Demo 通过后按真实口感修订。
- `novel-rewrite`：改动 spec 通过后写，锁住“保留内核”和“新版本要回答的问题”。
- `novel-continue`：续写方向通过后写，锁住原作未回收伏笔与新章节的阅读承诺。
- `novel-spinoff`：锚点表和角色卡通过后写，锁住配角视角的独特问题，而不是只换摄像机。
- `novel-expand` / `novel-condense`：章节映射通过后写，锁住“保骨架时要保什么阅读体验”。

Demo gate 通过后，把本文件的关键字段同步进 `审稿/demo_gate.json.reader_contract`，后续 `draft_packets.py` 会把它塞进每章任务包。

## 写作标准

### 核心题旨

“题旨”不等于道理说教。它是故事不断回到的核心问题，例如：

- 一个藏拙的人什么时候必须暴露锋芒？
- 权力系统里，弱者怎样保住尊严又不被吞掉？
- 被误解的配角，是否能在不篡改原作的前提下获得自己的命运？

每个 arc 至少要推进一次核心题旨。若连续多章只在刷事件、刷机制、刷打脸，却没有让核心问题更尖锐，审稿时按“题旨偏移 / 注水”处理。

### 读者承诺

读者承诺是前几章明示或暗示给读者的期待。后续章节可以延迟兑现，但不能遗忘或偷换。

常见承诺：

- 爽文承诺：主角会反杀、升级、打脸，但代价也会扩大。
- 情感承诺：两个人的关系会因事件变深，不是只靠误会拖延。
- 悬疑承诺：关键谜面会逐步给证据，不靠临时神降解释。
- 外传承诺：配角的新动机能解释原作留白，但不推翻硬锚点。

### 好看机制

“好看”不是全程高能，而是让读者持续获得：期待、延迟、兑现、反转、余味。任务包和 review 至少检查：

- 本章有没有清晰的阅读动力。
- 本章是否推进主线、关系、秘密、能力代价中的至少一项。
- 章末钩子是否和下一章真的有关。
- 爽点是否由人物选择挣来，而不是作者强塞。
- 文学性是否增强了场景、人物或情绪，而不是装饰性堆词。

### 文学质感

文学性不只等于华丽。对商业文也一样，好的文字要有“准确、可感、有人味”：

- 准确：动词、细节、对白贴角色身份。
- 可感：场景有光线、声音、温度、空间关系。
- 有人味：情绪通过动作和选择呈现，不只靠旁白解释。
- 有节制：高光处可以抬文笔，过渡处保持清爽。

若 `novel-score` 的“文学性 / 文笔”低于 6，或 `novel-review` 出现多条 `prose/theme/reader_promise` 建议级问题，导出前应回到 `demo` 或 `draft` 修订。

## Demo gate 同步字段

`审稿/demo_gate.json` 建议加入：

```json
{
  "reader_contract": {
    "theme": "一句话题旨",
    "dramatic_question": "核心戏剧问题",
    "must_answer": ["终局必须回答的问题"],
    "reader_promises": ["开篇已许诺的爽点/情感/悬念"],
    "aesthetic_register": "文风气质与文学质感",
    "delight_engine": ["好看机制"],
    "banned_drift": ["禁止偏成的题材/口吻/支线"]
  }
}
```

`reader_contract.reader_promises` 与旧字段 `reader_promises` 可并存；旧字段继续兼容，新增字段用于表达更完整的题旨与审美约束。

## Review 回流

`novel-review` 人判时新增三类维度：

- `theme`：题旨偏移、核心问题长期不推进。
- `reader_promise`：前文许诺的爽点/情感/悬念被遗忘或偷换。
- `prose`：文学质感不足、AI 味、辞藻堆砌、对白无戏。

严重度默认：

- 终局或主线 arc 偏离题旨：`blocking`，回 `outline/demo`。
- 连续 3 章读者承诺无推进：`suggestion`；商业连载 / 漫剧源书升 `blocking`。
- 文学质感局部薄弱：`suggestion` 或 `polish`，回 `draft/polish`。
