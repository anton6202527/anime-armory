# 设定圣经 — novel-* 家族统一设定 schema（单一真值源）

跨章一致性靠**一份设定圣经**当锚点（等同图像线的"定妆库 + 锚点句"）。各 novel-* skill **共用本 schema**，不各写各的模板——`novel-create` 从零建、`novel-spinoff` 从原作抽、`novel-rewrite` 在原作上改，落点都是同一套文件、同一套字段。

## 三件套 + 一份宪法

| 文件 | 角色 | 谁建 |
|---|---|---|
| `设定/设定圣经.md` | **宪法/单一真值源**——题材定位、世界观核心规则、力量/金手指体系、势力、术语表、时间线锚点。下游所有章节冲突时以它为准 | create 从访谈建；spinoff/rewrite 从原作抽取 + 标改动 |
| `设定/角色卡.md` | 每个主要角色的不可漂特征 + 弧线 + 关系 + 能力 | 全家族共用下方角色卡字段 |
| `设定/世界观.md` | 空间/势力/规则的展开（设定圣经的世界观小节放不下时） | 同上 |
| `设定/锚点表.json`（spinoff/continue/rewrite） | 与原作对齐的事件骨架锚点（事件/时间/在场人/已知情报） | 派生类专用，见 `novel-spinoff/references/timeline-anchoring.md` |

## 设定圣经.md 字段（宪法）

```markdown
# 设定圣经 — 《<书名>》
## 题材定位
一句话题材 + 目标平台 + 核心爽点类型（复仇/逆袭/打脸/团宠/……）

## 世界观核心规则（3-7 条不可破）
- 规则 1：……（如"修为靠吞噬妖丹，吞噬必反噬心智"）
## 力量 / 金手指体系
- 体系：境界/等级链（按序列出）
- **金手指**：是什么 + **代价是什么**（🔴 铁律：金手指必有代价/限制，无代价=爽点贬值，下游每章都要守这条）
## 主要势力 / 阵营
| 势力 | 立场 | 与主角关系 | 首现章 |
## 关键术语表（锁死写法，防跨章漂移）
| 规范写法 | 别名/禁用写法 | 含义 |
## 时间线锚点（派生作必填，对原作骨架）
| 锚点 ID | 事件 | 时间 | 在场人 | 角色已知情报 |
```

## 角色卡.md 字段（每个主要角色）

```markdown
### <角色名>（复用范围：全篇 / 第N-M章 / 一次性　·　首现章：第N章）
- 身份/定位：
- 外貌锚点：3-5 个不可漂特征压一句（跨章锁形，类比图像线"锚点句"）
- 性格 / 口吻：说话方式、口头禅、行为逻辑（防 OOC 的依据）
- 金手指 / 能力：+ 代价（与设定圣经一致）
- 成长弧线：起点 → 转折 → 终点（防"人物原地踏步/突变"）
- 关键关系：与谁，张力是什么
- 留白清单（spinoff 专用）：原作没说、本作要补的点
```

> **一致性追踪三列**（角色卡 + 势力表 + 术语表都带）：`首现章` / `复用范围` / `代价或约束`——这是把"同一张脸同一套规则"做成可勾选、可回扫的字段（`novel-review` 机检/人判按这三列核对漂移）。

## 各 skill 怎么用本 schema

- **novel-create**：第 3 步从访谈结论**新建**设定圣经 + 角色卡 + 世界观，金手指必写代价。
- **novel-spinoff**：从原作**抽取**世界观/角色到同一 schema，另建 `锚点表.json` 锁原作事件；配角加"留白清单"。
- **novel-rewrite**：`改动spec` 三栏（保留内核/改什么/加什么）→ 落成**新版设定圣经**（新设定也按本 schema、新金手指也必写代价），标"改自原作哪条"。
- **novel-continue**：续写沿用原作设定圣经（无则从原作抽），新增设定追加进去并标首现章。
- **novel-review**：模式① 按角色卡口吻/弧线查 OOC、按设定圣经规则查矛盾、按术语表查漂移、按锚点表查对齐——本 schema 是它的对照基准。

## 防漂移铁律
- **设定圣经是唯一真值**：任何章节与它冲突 → 改章节，不改圣经（除非显式升级圣经并回扫已写章）。
- **金手指必有代价**：无代价的金手指是头号爽点杀手，建卡即写死。
- **新设定追加要标首现章 + 复用范围**：否则跨章/跨作回扫时无从核对。
