# Red-Teaming Agent (剧情漏洞爆破手)

传统审查是"事后补救"；红蓝对抗（Red-Teaming）是**事前爆破**。
在确立了《设定圣经》和《创作蓝图》后，由 Red-Team 扮演刁钻的"找茬读者"，尝试用最功利、最符合逻辑的方式破坏故事。

## 触发时机
在 `novel-create` 第 3 步（建设定圣经）之后，第 5 步（章纲）之前。
运行命令：`logic_sentry.py --red-team`

## 爆破维度 (Attack Vectors)

1. **资源滥用 (Resource Exploit)**：
   *攻击策略*：主角拥有能力 X。如果在场景 Y 中遇到困难 Z，为什么主角不直接用 X 解决？
   *例子*："你设定主角有穿墙术，那为什么去皇宫偷东西还要买通守卫？"
2. **逻辑短路 (Logic Bypass)**：
   *攻击策略*：反派的最终目的是 A。既然他有能力 B，他最快达到 A 的方法是 C。为什么他要用小说里那套复杂的 D 方案？
   *例子*："魔王既然可以直接毁灭城市，为什么还要派手下伪装成商人去下毒？"
3. **金手指零代价 (Consequence Evasion)**：
   *攻击策略*：设定的代价不足以制衡收益，导致战斗毫无悬念。

## 产出 (Red-Team Report)
报告会直接反馈给创作者，强制要求创作者在《设定圣经》中打补丁（如：增加施法前摇、设定魔法免疫结界、解释反派的政治顾虑等）后，方可进入章纲设计。
