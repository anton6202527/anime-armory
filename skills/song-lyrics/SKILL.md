# song-lyrics — 访谈式作词 / 歌词创作

用户只有**主题 / 几个字 / 曲风偏好**，由本 skill **访谈补全 → 写出结构化、可唱、押韵、字数贴旋律的歌词**。是 `song`（写歌线）第 1 步（产 `词/lyrics.md` 给 song-compose）；也可独立调用做一次性作词或问工艺。

**零安装、完全独立**——纯 LLM + 本 skill 自带的作词工艺知识库 `references/songcraft.md`，不依赖任何其他 skill。流程范式（访谈→蓝图→产出）与项目里其它创作 skill 神似，但各写各的、互不引用。

## 偏好（私有 · 用户选择，不写死在本 skill）

本 skill 的可选项**不写死在源码里**。按 `../_偏好约定.md` 读用户私有选择：先读 `<作品根>/_设置.md`；缺则用全局默认 `创作偏好-默认.md` 预填并告知一句；再缺则**首次问一次**→写回 `_设置.md`→同项目之后**沉默沿用**（合规/不可逆/花钱多的点每次仍确认）。

本 skill 涉及的选择点：`歌曲用途`、`目标平台/曲风`、`语言`、`BPM/速度`、`目标时长`（注：独立调用、没有作品根时不强制落文件，按全局默认/现问）。

## 核心原则
- **先访谈再写词**：从几个字问出主题/情绪/曲风/平台/视角/叙事 vs 抒情，一次一组带默认建议，不轰炸。
- **结构先行**：先定段落骨架（intro/verse/pre-chorus/chorus/bridge/outro）+ **hook（记忆点）**，再填词。副歌是全曲核心，先写副歌。
- **可唱 > 文采**：**字数贴旋律/节拍**（每行音节数对齐乐句）、押韵自然、口语顺口、避免拗口堆砌。详见 `references/songcraft.md`。
- **平台/曲风定调**：国风/流行/说唱/民谣/电子/抖音神曲各有套路与 hook 习惯。
- **原创合法**：原创词=用户自有。给已发行旋律填词、商用翻唱 → 词曲版权另属原作者，提示授权。

## 工作流
0. **立项访谈**（必读 `references/songcraft.md §访谈`）：主题/情绪 → 歌曲用途+目标时长 → 曲风+平台 → 语言+BPM/速度 → 叙事还是抒情 → 视角(我/他/对话) → 结构 → 关键意象/想押的韵脚。给默认让用户确认。
1. **建/接骨架**：
   - 新写歌项目 → `python3 <song skill>/scripts/init_project.py --title ... --genre ... --theme ...`（建 `写歌/<曲名>/` + `词/lyrics.md` 段落骨架）。
   - 独立作词 → 直接产出 md，不建项目。
2. **填创作蓝图**（若建了项目）：主题/曲风/情绪/平台/视角写实。→ 用户审。
3. **写副歌 hook 先**：副歌是记忆点，先写它（含 1 句最抓耳的 hook），定全曲韵脚与情绪顶点。→ 用户审。
4. **补主歌/桥段**：verse 叙事推进、pre-chorus 蓄力、bridge 反转/升华，对齐副歌韵脚与字数。
5. **可唱性自查**：逐行核对 `songcraft.md` 清单（字数贴乐句/押韵/顺口/无拗口/hook 够抓）。
6. **交付**：写入 `词/lyrics.md`（结构化，段落标签 + 词），更新 `_进度.md`。下一步 `song-compose` 生成歌。

## 常见错误
| 错误 | 纠正 |
|---|---|
| 上来就写词不问主题/曲风 | 先访谈：主题/情绪/曲风/平台/视角 |
| 先写主歌后写副歌 | 副歌+hook 先写（它是核心与韵脚锚） |
| 词美但不可唱（拗口/字数不齐） | 字数贴乐句、口语顺口，可唱 > 文采 |
| 全曲无记忆点 hook | 副歌必有 1 句最抓耳、可重复的 hook |
| 押韵忽韵脚乱跳 | 一段一韵脚（或规律换韵），别每行乱押 |
| 给已发行旋律填词当原创商用 | 旋律版权另属原作者，提示授权 |
