# 角色一致性 Checklist（出图必对·跨集锁脸锁妆造）

> **目的**：让"同一角色 100 集同一张脸、同一套衣服"成为可勾选的硬流程，而不是凭感觉。
> 一致性不是某一步的事，是一条**建卡 → 定妆 → 复用 → 锚点 → 出视频**的铁律链——任一环断，整集穿帮。
> 本表是 `n2d-image` 出图时的对照清单；规则散见 `n2d-script/references/formats.md`（角色卡）、`novel2drama/Q&A.md` Q3-Q12/Q16/Q18-Q20/Q24，这里汇成一处。

优先级（投入产出比从高到低）：
**① 锚点句每镜拼 > ② 定妆多图参考派生（多视图锁脸·禁纯文生图）> ③ 资产身份注册层（角色 ID / Face Lock / LoRA 状态）> ④ 扫共享库防重建（角色+场景）> ⑤ 不中途换工具 > ⑥ 平台建角色/LoRA/多镜故事板**
前四条是量产地基，**必须做满**；后两条是参考图压不住时的升级。

---

## 一、建卡层（设定先于出图，治"没有锚点可对"）

角色卡（`设定库/characters/<角色>.md`，首次出现即建、全篇唯一）必须含：

- [ ] **妆造拆解**：发型/发色/发饰、妆容、服装（款式·领口·袖型·材质·腰带）、配饰、**色卡（主+辅+点缀）**
- [ ] **识别锚点 3-5 个**：跨集绝不能漂的特征（例：凤眼薄唇 / 乌黑半披素布发带 / 左腕淡疤 / 月白粗布旧宫装）
- [ ] **锚点句**：把锚点压成一句（例「凤眼薄唇·乌黑半披发带·月白粗布·左腕淡疤」）—— 下游每张分镜 prompt 末尾要拼它
- [ ] **形态变体单列**：常态 / 觉醒态 / 银牌态 / 死亡态，各记差异点，**各算独立定妆条目**
- [ ] **反派/易混角色：写"对立特征"**（见第六节）

> 卡没建全 = 后面无据可依。`出图/共享/prompt/角色定妆.md` 是这张卡的"实战 prompt 包装"，妆造拆解必须与角色卡完全一致。

## 二、定妆层（出"参考锚点图"，不是"最美的图"）

**角色定妆组（所有人物必出标准三视图 + 人审拼版，厚锚点）**——单张正脸不再作为人物定妆合格线：

- [ ] **正面主参考**：正脸中性光 + 半身脸部特写 → `定妆_<角色>.png`（下游分镜的主参考图，锁脸）
- [ ] **侧面参考**：标准侧面 → `定妆_<角色>_侧.png`（锁鼻梁 / 下颌 / 发型侧轮廓；侧脸、转头、过肩镜必备）
- [ ] **背面参考**：标准背面 → `定妆_<角色>_背.png`（锁后脑勺发型 / 背部服饰 / 披风 / 腰带绑法；over-shoulder、转身、追逐背身镜不再无参考）
- [ ] **服装参考**：半身或全身 → `定妆_<角色>_半身.png` 或 `定妆_<角色>_全身.png`（二选一；锁衣领 / 袖型 / 腰带 / 配色 / 体态）
- [ ] **人审拼版**：标准三视图 → `定妆_<角色>_三视图.png`（由已通过自检的正面 / 侧面 / 背面生产拆图拼成，给人快速审脸审服装和归档；不作为分镜主参考）
- [ ] **增强设定表 / 表情条**：`定妆_<角色>_设定表.png` 可纳入半身/全身/色卡/配饰；`定妆_<角色>_表情.png`（喜 / 怒 / 哀 / 惊）用于情绪特写多的角色
- [ ] 定妆组全用**同一 prompt 同一模型**，只改"正面 / 侧面 / 背面 / 半身 / 全身 / 表情"那一截，背景妆造一致
- [ ] **命名不要混**：多张可喂给图 AI 的参考 PNG 叫「定妆组」；单张合成总览才叫「三视图 / 设定表」
- [ ] 短线配角 / 路人：仍按标准三视图建人物定妆；若只是人群剪影、无明确身份且不会复用，按场景元素处理，不建角色定妆

**主参考张本身的硬要求**（它要当参考图复用，歪一张毁整集）：
- [ ] **干净浅灰纯色背景 + 柔和均匀光**（不是场景、不上电影级强阴影——便于锁脸锁妆造）
- [ ] **双臂、双手可见**（后续动作戏 / 图腾 / 抓握要用，藏手就废）
- [ ] **竖版 9:16**，模型与该角色所有图**同一个**（换模型脸会跑）
- [ ] **分辨率 ≥1024px·定妆组至少覆盖正/侧/背标准三视图**（可扩到 3–6 角度；2026 多参考后端实战下限：参考图越清晰、角度越全，锁脸越稳）
- [ ] **身份锁定句与锚点句叠加用（别混）**：锚点句锁"特征词不漂"、身份锁定句锁"与参考图①就是同一张脸"（多参考/编辑类模型对后者最敏感）——含角色镜头两句都拼，见 `prompt_format.md §2.2`
- [ ] **挑图优先级**：① 姿态最中性 ② 手臂可见 ③ 脸最像设定 ④ 视觉冲击排最后
- [ ] **挑图 6 条核对**：脸型对 / 发型发色对 / 服装色对 / 配饰对 / 设定细节(疤·雀斑) / 背景纯色
- [ ] 4 张全废 → **同 prompt 再抽一轮**（别改 prompt；即梦同 prompt 不同种子差异大）

**重抽 vs 接受**（治"无止境刷图烧 credit"）：

| 偏差 | 处理 |
|---|---|
| 服装色错（月白→紫）、发型错（双环髻→披发） | **重抽**（全集穿帮） |
| 气质/表情风格错（伪善画成凶相） | **重抽**（气质定角色） |
| 年龄 ±3-5 岁、缺疤/雀斑等小细节 | **接受**（分镜阶段圆） |
| 多/少配饰 | 看是否破坏身份——破坏才重抽 |

> 形态变体（觉醒态…）**必须用常态定妆做参考图**（强度 0.75-0.85），否则"觉醒前/中/后"变三个人。

- [ ] **一致性高危镜预案（2026 公认崩脸带：极端角度 / 大暗部 / 人物在画面太小）**：这三类镜跨图与 image2video 都最易脸漂——出图时**提参考强度 + 补对应角度定妆（侧/背）**，视频用后端 Character-ID/Face-Lock；能在分镜阶段避就避（极端俯仰改常规角度、过小人物改中近景、纯黑环境留一束补光照脸）。分镜侧约定见 `n2d-script/references/分镜语法.md`「一致性高危镜」。

- [ ] **特效/武器/法术形态（剑气/飞剑/法宝/掌风/灵力光效）是否进了共享定妆库、跨镜一致**（和角色脸同等对待，会漂就锁；⚙️仙侠玄幻归 `法宝定妆.md` / `特效定妆.md` 专类，法宝按形态/成长阶段多态）—— 详见 `n2d-script/references/打斗分镜.md §五`、`prompt_format.md §1`
- [ ] **仙侠奇观元素（御剑巡航姿态/祥云、劫雷/护体光/突破光柱、丹炉/丹火/成丹、阵图、地标 establish）是否进库、跨镜跨集一致**（飞行出「巡航定妆」、大阵出「全阵俯视锚」、大场面建「地标场景定妆」复用；斗法对轰双色光束入库、神魂「元神=肉身半透明态派生」治"二我"）—— 详见 `n2d-script/references/仙侠场面分镜.md §一~§八`

### 二点五、资产身份注册层（P0：把定妆变成可调度身份）

`出图/共享/identity_registry.json` 是机器真值；`00_索引.md` 是人读摘要。角色/形态只要会入镜，就要登记成可复用身份，而不是只留几张 PNG：

- [ ] 每个角色/形态都有 `characters[].forms[]`：`form`、`asset_key`、`anchor_phrase` 与角色卡/定妆 prompt 对齐
- [ ] `reference_group` 齐：正面主参考、侧面、背面、服装、三视图拼版；路径为作品根相对路径
- [ ] `identity_adapters.image` 齐：Codex/OpenAI/Dreamina 写 `fallback_reference_group`；可灵/Seedream/Sora 等支持主体库或角色 ID 的后端写 `unregistered|registered|ready`
- [ ] `identity_adapters.video` 齐：Dreamina fallback、Kling Character ID、Seedance Face Lock、Veo reference controls 等状态可查；后端不支持可以 fallback，但不能缺登记
- [ ] `identity_adapters.lora` 齐：默认 `not_needed`；核心长线角色准备训练写 `candidate`；`ready` 必须有 `model_path` + `trigger`
- [ ] `angle_policy` 齐：允许角度、高危角度、需要额外参考的角度/动作；极端俯仰、深暗部、人物过小、多人接触默认高危
- [ ] `drift_forbidden` 齐：脸型、发型、服装主色、标志配饰、体态等零容忍漂移项；审图/审片按这里判
- [ ] 注册状态真实：`registered/ready` 必须写实际 `id` / `handle` / `reference` / `model_path`；空字符串等于假登记，gate 阻断
- [ ] n2d-image 逐镜从 registry 取 `reference_group` 和禁漂项；n2d-video 从 registry 取 Character ID / Face Lock / reference controls，不能在 prompt 现场手写临时 ID

## 三、复用层（共享库 + 5 步 SOP，治"跨集脸漂移"）

**最容易翻车的一环：每集出图前必走 5 步 SOP，第 ① 步跳了必漂移。**

- [ ] **① 扫共享库**：读 `出图/共享/prompt/00_索引.md`，盘清已有角色（含形态变体）/场景/道具及状态
- [ ] **② 列本集需求**：读 `分镜剧本.md` + `素材清单.md`
- [ ] **③ 差集 = 新增**：本集需求 − 共享已有（含"新角色" + "已有角色的新形态"）
- [ ] **④ 仅追加新增项**到共享库（索引 + 定妆 prompt 块）
- [ ] **⑤ 建本集 prompt**：`出图/第N集/prompt/{00_总览,01_分镜出图}.md`，总览**引用**共享、不重画

- [ ] **共享 vs 本集归档**：角色/场景/反复入镜道具定妆 → `出图/共享/图片/`（全篇一份）；一镜一图分镜 → `出图/第N集/图片/`
- [ ] **场景图同样建库复用**：反复出现的场景（宫殿/冷宫/庭院）= 共享定妆，出一次 `出图/共享/图片/定妆_<场景>.png`，跨集所有该场景镜头都引用它做参考图——**别每集重画背景**（背景漂移和脸漂移一样穿帮）。新场景走和角色相同的 5 步 SOP 入库
- [ ] **关键反复场景出多视图（场景四视图·题材自适应）**：贯穿多集的核心场景（主殿/洞府/山门/大阵地）除主图外，按机位需要补不同朝向/景深参考 → `定妆_<场景>_<向>.png`（如正向全景 / 侧向 / 反向（过肩背景）/ 俯瞰），让**跨镜头切换背景自洽**——同一空间这面墙切到那面墙不穿帮。一次性/单镜场景单图即可，**不必铺四视图**（按反复程度自适应，别给路人场景浪费额度）
- [ ] **共享 PNG 先行硬闸门**：本集分镜 PNG 生成前，`出图/共享/prompt/00_索引.md` 中本集引用的共享角色/场景/道具必须全部 ✅，且对应 PNG 文件真实存在；**✅ = 该定妆已过「生成后落档闸门自检」**（妆造逐项对卡 / 三件套同一个人 / 主要角色零漂移，见 `prompt_format.md §1.2` 自检块），出了图但没过自检不算 ✅；缺任一项或自检未过时**禁止生成本集分镜图**，先补共享层并过自检
- [ ] **参考图/图生图复用**：定妆照设为"角色参考图"，每张分镜基于它生成；重生成分镜图时也必须先加载/引用共享定妆 PNG
- [ ] 进度算法 `出图列 = 已有PNG / (共享需要 + 本集分镜)`——复用让分子跨集累积（第2集开局就有起步分）

## 四、锚点层（每镜拼锚点句 + 多图参考派生 —— 双保险）

- [ ] **多图参考派生（铁律）**：含角色的镜头图**一律基于参考图派生**（image-to-image / 多图参考 / 平台角色绑定），**禁纯文生图**。喂：正面主参考（主 ~0.8）+ 侧面锚（~0.5-0.6）+ 背面锚（背身/转身/过肩镜加，~0.5-0.6）+ 半身/全身服装锚（~0.5，全身动作或服装易漂时加）+ 本镜场景定妆（~0.4-0.5）
- [ ] **锚点句 + 多图参考叠加用，不互斥**：参考图锁"像不像"，锚点句锁"特征词不漂"；两条都做，跨镜相似度才稳到 90%+
- [ ] **每张含角色的分镜 image prompt 末尾，拼该角色卡的『锚点句』**（比单纯调参考图强度更稳，直接治脸漂移）
- [ ] prompt 结构：`主体 + 外貌锚定 + 动作 + 环境 + 景别 + 画风词 + [锚点句] + [视频AI锚定句]`
- [ ] 锚点句放**最后做风格说明**，不要插进主体描述（会扰乱构图）
- [ ] 多角色同框：每个角色各拼各的锚点句
- [ ] **多角色同框是高危触发点（`gate.py` 会 WARN 单镜 ≥2 具名角色）**：单图参考后端难保多人各自不漂。处置二选一并登记——① opt-in 官方多参考后端（Seedream 14 图 / Nano Banana Pro 5 人免微调 / 可灵主体库按 ID）；② 沿用当前官方图后端走「**分别出图锁脸 + 合成同框**」（或把两人位置交 n2d-video 用首尾双帧焊住）。别拿单图参考硬抽多人同框反复烧 credit。

## 五、跨工具 / 跨阶段（治"风格中途跳变"）

- [ ] **同一集绝不换图 AI**（换工具=整集所有定妆重做，"一致性税"）；集间要换也得重做全部定妆
- [ ] **切换角色前清空参考图**（即梦参考框是"粘性"的，不清会把上一角色的脸带过来）——只有**同角色形态变体**才保留参考图
- [ ] **场景定妆必须清空人物参考图**（否则会出"角色躺在床上"等鬼画面）
- [ ] **图 AI ≠ 视频 AI 时**：每张 image prompt 末尾拼**目标视频 AI 的图像风格锚定句**（否则 image2video 运动估计崩）
- [ ] **有角色的镜头一律用图生视频**（首帧=定妆/分镜 PNG 锁脸），不用文生视频

## 六、区分层（反派/配角不要"长得像主角"）

- [ ] **定妆生成顺序：主角 → 配角 → 反派（针对性堵主角特征）→ 场景 → 道具**（Q&A Q3）。先把主角定妆锁死，出反派/易混角色时拿主角已定特征当负向词反向"驱赶"，避免画成同一张脸；场景/道具最后出。乱序先出反派会缺主角基准、易撞脸。
- [ ] 反派**负向词**主动写进主角的视觉关键词来"驱赶"（例：柳娘子负向词含"稚气·圆脸·双环髻·月白·淡青"，堵死画成小禾/沈念）
- [ ] **正向词逐条写对立特征**：

| 主角 | 反派对应 |
|---|---|
| 20岁清瘦 | 30岁端庄保养 |
| 凤眼 | 鹅蛋脸·眼尾微吊 |
| 月白粗布 | 深蓝绸料·盘扣宽袖 |
| 无配饰 | 素银簪·管事腰牌 |
| 披发素布带 | 乌黑一丝不苟发髻 |

- [ ] 伪善反派：定妆抓**端庄基础气质**，"似笑非笑"留给镜头特写（铺垫越平淡，反转越炸）

## 七、进阶层（参考图压不住时才上重武器）

- [ ] **后端原生角色ID / 主体库（梯子第②档·先于 LoRA 用尽）**：可灵 主体库 / Custom Model（10-30 段视频）/ Element Library（~4 张图）、Seedream Universal Reference（免 LoRA 跨图锁人）、Sora Character Cameo——**注册一次、按 ID 跨镜跨集引用**，比纯参考图稳、无需 GPU、更省。能用就用；所选官方图后端无持久 ID（含 Dreamina/即梦官方 CLI）则回退第①档参考图派生。详见 `SKILL.md「可选增强：后端原生角色ID / 主体库」`+`platforms.md`「后端原生角色ID / 主体库」节
  - **视频侧同样吃这一档（治 image2video 脸漂）**：出视频后端的原生角色一致能力——可灵 3.0 **Character ID** / Seedance 2.0 **Face Lock** / Veo 3.1 **reference controls**——也要把定妆图喂进去当持久角色参考，不只当首帧（极端角度/大暗部/人物过小最易漂）。见 `n2d-video/references/platforms.md` 各档案「角色一致性」行。
- [ ] **LoRA（可选增强，仅开源 Flux/SDXL 生态）**：只给**贯穿几十集的核心长线角色**训（如女主沈念），一次性角色（柳娘子/三小妖）**不值得**训
  - ROI 拐点：角色反复出现几十集 **且** 参考图已压不住脸 → 才训；先用尽"平台建角色 ID"这一档
  - **启用流程是引导式的，详见 `lora_consistency.md`**（Stage 0 决策门 → 建数据集 → 云训练 → ComfyUI 验证 → 接混合产线 → 固化），一步步问答推进
- [ ] 闭源/可灵/Gemini **不接受自训 LoRA**——自训只能在开源生态出图（绑平台决策，见 Q24）

## 出图前 30 秒速查（最小集）

```
☐ 角色卡有锚点句？        → 没有先建卡（§一）
☐ 人物定妆组齐？          → 标准正/侧/背三视图 + 半身或全身服装参考 + `定妆_<角色>_三视图.png` 拼版（§二）；表情/设定表按剧情增强
☐ 身份注册层齐？          → `identity_registry.json` 有 reference_group / adapters / angle_policy / drift_forbidden（§二点五）
☐ 扫过共享库 00_索引？    → 没扫必漂移（§三①），角色+场景都扫
```
