# LoRA 增强角色一致性（可选 · 引导式）

> **定位**：这是 `n2d-image` 默认一致性方案（锚点句 + 定妆参考图，见 `角色一致性checklist.md §一–五`）之上的**可选增强层**，不是必选项。
> **默认不启用**——绝大多数角色用「参考图 + 锚点句 + 平台建角色」已经够稳。只有**核心长线角色「参考图压不住脸」**时才值得引入 LoRA。
> **重要**：用户选择"启用 LoRA"后，按本文件**一步步问答引导**——一次只推进一个阶段，先确认/问关键决策，拿到回复再进下一步，**不要一次性把五个阶段全丢给用户**。

---

## 何时把 LoRA 作为选项提出来？

在 `n2d-image` 出图过程中，遇到下列信号之一，**主动把 LoRA 作为可选项提给用户**（而不是默认就做）：

- 某个**贯穿几十集的核心角色**（如女主），即便拼了锚点句 + 调高参考图强度，脸/妆造在复杂角度/光照下**仍反复漂移**；
- 该角色 hero 镜（爽点/反转/觉醒/封面）对脸的一致性要求**已超出参考图能压住的上限**；
- 用户主动问"要不要训练 / LoRA / 提高一致性"。

提的时候说清三句话：① 这是**可选增强**，不启用也能继续用参考图出图；② **只对核心长线角色划算**，一次性角色不值得；③ 启用=并一条**开源支线**（Flux/ComfyUI），即梦/可灵等闭源平台**不接受自训 LoRA**，所以是混合产线、不是替换。

**ROI 判据**（帮用户判断该不该做）：

| 该训 LoRA | 不该训 |
|---|---|
| 贯穿几十集、反复出镜的核心角色（如女主沈念） | 只出 1–3 集就退场（柳娘子/三小妖/路人） |
| 多形态（常态/觉醒/银牌）、对脸要求高 | 参考图 + 锚点句已经够稳 |
| 参考图/平台建角色已压不住脸 | 还没试过"平台建角色 ID"这一档（先试它，更轻） |

> 比 LoRA 更轻的一档先用尽：**官方图后端的"建角色 ID / 主体库"**（可灵 Custom Model / Element Library、Seedream Universal Reference、Sora Cameo 等）——无需 GPU、平台内复用。Dreamina/即梦官方 CLI 可作为图片阶段后端，但没有持久主体 ID 时仍属于参考图派生档；LoRA 是这一档也压不住时的重武器。

## 引导流程总览（五阶段，逐步推进）

```
Stage 0 决策门（商用许可 / 本次范围 / 云算力）  ← 先问清，定底模与路径
   ↓
Stage 1 建数据集（现有定妆 → 扩 15–20 张多角度 → 清洗 → 打标 → 1024²）
   ↓
Stage 2 云训练（fal.ai 一行提交 / RunPod 自配）
   ↓
Stage 3 ComfyUI 出图验证（LoRA + IP-Adapter FaceID + ControlNet，和即梦版对比脸一致性）
   ↓
Stage 4 接进产线（混合产线：LoRA 只跑核心角色 hero 镜，回流统一画风）
   ↓
Stage 5 固化（脚本/skill 化，下个核心角色可复用）
```

---

## Stage 0 — 决策门（第一次交互，先问这 3 件）

逐条问，缺省给推荐值；用户回复后据此锁定底模与路线：

1. **商用变现？** —— 决定底模与许可。
   - 仅自用/测试 → 可用 **FLUX.1-dev**（质量最好，非商用许可，零顾虑）。
   - 会商用变现 → FLUX.1-dev 非商用许可是**障碍**：改用 **SDXL** 或 **FLUX.1-schnell（Apache-2.0 倾向，但需自行向 Black Forest Labs 核实当前授权）**，按可商用路线设计。
   - 还没定 → 先按自用跑通，**把"商用许可"记成遗留风险点**，变现前再核实授权（见 Q24.7-E 许可未决项）。
2. **本次范围** —— 决定我做到哪步：仅手动跑通一个 LoRA / 直接脚本化 / 只给方案。
3. **云算力** —— 决定训练入口：**fal.ai（最省事，~$2/次）** / RunPod（更便宜可控，需配环境）/ 用户已有 GPU 或账号。
   - 你是 Mac（无独显）→ **本地训 Flux LoRA 不现实，走云端**（Q24.7-E：Mac 用户走云端更稳）。

> 三件答完再进 Stage 1。**别在没确认商用/算力前就开始建数据集。**

---

## Stage 1 — 建数据集（LoRA 成败 80% 在这）

目标：为该角色凑 **15–20 张「甜区」多角度图**（15–50 可用，>~30 易过拟合）。

- **种子**：该角色现有定妆照可直接当种子（如沈念常态 + 觉醒态）。但 2 张远不够。
- **扩样**：用现有定妆在**即梦/可灵的"角色参考"**里扩出不同**角度/景别/表情**：
  - 角度均衡：约 5 近景脸 / 5 半身 / 5 不同角度（正/侧/微仰俯）/ 5 全身；
  - 表情/光照适度多样（一致性靠角度多样性，不是数量堆）。
- **清洗**：剔除脸崩/手崩/与设定漂移的；保证**这 15–20 张本身就高度一致**（脏数据=学歪）。
- **规格**：统一 **1024×1024**（512 下限），裁好主体。
- **打标**：给每张写 caption，用一个**唯一触发词**（如 `shennian_v1`，别撞常见词）+ 简述（角度/景别/服饰）。
- 落档建议：`制漫剧/<剧名>/设定库/lora/<角色>/dataset/`（图 + caption）。

确认数据集齐 + 一致 → 进 Stage 2。

## Stage 2 — 云训练（提交 → 等 → 取 .safetensors）

**路线 A：fal.ai（推荐首次）**
- 托管式 **FLUX LoRA fast**，~$2/次（默认 1000 步，按步线性计费）；上传数据集 → 拿 `.safetensors`。
- 最省事，无需配环境；先充 $10 左右够多轮。

**路线 B：RunPod（更便宜可控）**
- 租 **RTX 4090(24GB)**：Community ~$0.34/hr、Secure ~$0.69/hr（建议先充 $10）。
- 跑 **Ostris AI-Toolkit / FluxGym**（FLUX.1-dev 是 HF 门控模型，需先接受许可）。
- **FluxGym 基线超参**：每图重复 10、最多 12 epochs、network_dim(rank) 4→16、学习率 8e-4→5e-4（小数据集取低值更稳）。

> ⚠️ Q24.7-E：端到端总价/耗时**无硬数据**，以上是单价；实际多轮调参成本另算。
> 产物 `.safetensors` 落 `设定库/lora/<角色>/<角色>_v1.safetensors`，记下训练参数与触发词。

## Stage 3 — ComfyUI 出图验证（和即梦版对比）

- **底模必须与训练底模一致**（Flux 的 LoRA 不能用在 SDXL 底模，反之亦然）。
- ComfyUI 出图链：**角色 LoRA 强度 0.7–1.0** + **IP-Adapter FaceID 控脸（3–5 张好参考）** + **ControlNet 控姿势**。多数 12GB 显存可跑（8GB 优化后勉强）——本地无显卡则继续用云 GPU 跑 ComfyUI。
- prompt 里带**触发词** + 角色卡锚点句 + 画风词。
- **验收**：拿 LoRA 出的脸和 Codex 参考图版**并排对比**，看复杂角度/光照下脸是否更稳。不如参考图版 → 多半是数据集不一致或步数不当，回 Stage 1/2 调，**别无止境刷**。

## Stage 4 — 接进产线（混合产线，别整集切开源）

- LoRA **只用于该核心角色的 hero 镜 / 关键叙事镜**（爽点/反转/觉醒/封面候选）；其余镜头仍走 Codex/OpenAI 参考图派生。
- **回流统一画风**：LoRA 出的图末尾仍拼视频 AI 的"图像风格锚定句"，与当前项目图后端风格对齐（见 `platforms.md` + Q18）；需要轻重绘统一光感时可回到所选官方后端，包括 Dreamina/即梦官方 CLI。
- **归档/进度**：LoRA 版 PNG 仍落 `出图/共享/图片/`（共享层）或 `出图/第N集/图片/`；在 `00_索引.md` 标注"LoRA 出"。出图列计数规则不变。
- **绝不整集中途从闭源切开源**（画风跳变，违 `角色一致性checklist.md §五` 一致性税）。

## Stage 5 — 固化（让下个核心角色能复用）

- 执行层已抽成 `n2d-lora`：用 `skills/n2d-lora/scripts/lora.py` 管理 `init / dataset / train-job / validate / register / status`。
- 落 `设定库/lora/<CHAR_ID>/<形态>/` 统一管理；每个核心角色/形态都有 `lora_card.json`、`dataset_manifest.json`、`train_job.json`、`validation_report.json` 和最终 `.safetensors`。
- `register` 只接受 `validation_report.verdict=pass`，然后把 `base_model/model_path/trigger/model_hash/validation_report/train_job` 写回 `identity_registry.json`；之后必须跑 `n2d-identity --write`。

## 不确定项（勿当已证实，变现/落地前自行核实，源见 Q24.7-E/F）

- FLUX.1-dev 非商用许可**是否传递到所训 LoRA** → 未决；商用前向 Black Forest Labs 核实当前授权。
- 端到端总价/耗时无硬数据；价格/配额/功能名 2026 迭代极快（即梦 Seedance 2.0、可灵 2.6/3.0 都在变），以官方实时为准。
- 平台"建角色 ID"的具体张数/价格/跨集复用上限以官方文档为准。
