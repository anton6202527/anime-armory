# 资产引用注册层（场景 / 道具 / 独立服装 / VFX）

目标：把关键非人物资产从“索引里的中文名 + PNG”升级成可执行的 ID 映射。角色脸和角色形态归 `identity_registry.json`；会随剧情变化的伤痕/战损归 `visual_state_ledger.json`；场景、道具、独立服装/套装、法宝/光效/VFX 归本层。

机器真值文件：

```text
出图/共享/asset_registry.json
```

`出图/共享/prompt/00_索引.md` 继续做人读索引；`asset_registry.json` 负责 gate、批量任务、出图执行和审片漂移分组。

## 1. 何时建 ID

| 类型 | ID 前缀 | 何时登记 |
|---|---|---|
| 场景 / 地点 | `LOC_` | 反复出现、跨镜轴线/光位/布局要稳定的场景 |
| 道具 | `PROP_` | 剧情关键或反复入镜，结构/件数容易被模型画错的道具 |
| 独立服装 / 套装 | `OUTFIT_` | 可在多个角色、多个形态或跨集单独复用的服装资产；普通角色服装仍放 `CHAR_xx/形态` |
| 法宝 / 光效 / VFX | `VFX_` | 颜色、形状、拖尾、强度需要跨镜统一的特效或法宝表现 |

不要给一次性、无复用价值、没有结构要求的小物件滥建 ID；但只要用了共享场景/道具参考图，逐镜 prompt 就必须写对应 ID。

## 2. 最小 schema

```json
{
  "kind": "n2d_asset_reference_registry",
  "version": 1,
  "updated_at": "2026-06-10",
  "assets": [
    {
      "id": "LOC_01",
      "type": "scene",
      "name": "冷宫寝殿",
      "scope": "第1集起复用",
      "reference_group": {
        "primary": "出图/共享/图片/定妆_冷宫寝殿.png"
      },
      "constraints": {
        "layout": "床榻到门口横轴；床榻/铜镜在画左，门口在画右",
        "axis": "沈念默认画左看画右；柳娘子默认画右看画左",
        "light_anchor": "画左前 3000K 残烛暖主光；画右后门外冷月冷背光"
      },
      "drift_forbidden": ["layout", "axis", "light_direction", "era_style"]
    },
    {
      "id": "PROP_01",
      "type": "prop",
      "name": "赐死托盘",
      "scope": "第1集核心道具",
      "reference_group": {
        "primary": "出图/共享/图片/定妆_赐死托盘.png"
      },
      "constraints": {
        "structure": "暗红托盘；白绫、短匕首、白瓷毒酒壶三件套数量锁定；毒酒壶唯一短颈圆口，无侧嘴/斜嘴/双口/额外壶嘴；短匕首一柄一刃"
      },
      "drift_forbidden": ["item_count", "flask_single_round_mouth", "dagger_single_blade"]
    }
  ]
}
```

## 3. 字段规则

| 字段 | 规则 |
|---|---|
| `kind` | 必须是 `n2d_asset_reference_registry` |
| `assets[].id` | 必须唯一；前缀必须匹配类型：`scene/location -> LOC_`、`prop -> PROP_`、`outfit/costume -> OUTFIT_`、`vfx/effect -> VFX_` |
| `reference_group.primary` | 必填；作品根相对路径；video/compose/review gate 会检查路径真实存在 |
| `constraints` | 必填非空；场景至少锁 `layout/axis/light_anchor/structure` 之一，道具必须锁 `structure` |
| `drift_forbidden` | 必填非空；审图/审片零容忍项 |

道具约束要写“结构拓扑 + 数量”，不要只写材质和颜色。示例：铜镜写单镜面；毒酒壶写唯一短颈圆口；托盘写三件套件数；匕首写一柄一刃。

## 4. 逐镜 prompt 怎么写

```markdown
**资产引用注册层**：`LOC_01` 冷宫寝殿；`PROP_02` 赐死托盘；从 `出图/共享/asset_registry.json` 继承 reference_group / constraints / drift_forbidden；锁冷宫 layout/axis/light_anchor，锁赐死三件套数量和毒酒壶唯一短颈圆口。
```

规则：

- 参考图块含“场景定妆”或 `定妆_<场景>.png`，必须写对应 `LOC_xx`。
- 参考图块含“道具锚”或 `定妆_<道具>.png`，必须写对应 `PROP_xx`。
- 服装如果只是某角色某形态的一部分，不单独写 `OUTFIT_xx`；只有可独立复用或跨角色/跨集复用时才登记。
- 执行端按 ID 读取 `reference_group.primary` 和 `constraints`，不要只靠中文道具名临场猜图。

## 5. gate 规则

`n2d-review/scripts/gate.py` 会检查：

- `asset_registry.json` 是否存在、kind 是否正确、assets 是否非空。
- ID 前缀是否和类型匹配，是否重复。
- 每条是否有 `reference_group.primary`、`constraints`、`drift_forbidden`。
- 场景是否锁布局/轴线/光位，关键道具是否锁结构唯一性。
- 逐镜用了场景/道具参考图但缺 `LOC_xx` / `PROP_xx` 时阻断。

原则：人读索引可以宽松；机器 registry 必须结构化，否则后续批量出图、审片和定向返工无法知道“这个漂移应该对照哪一个资产”。
