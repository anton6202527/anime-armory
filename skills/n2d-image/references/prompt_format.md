# 出图 prompt 两层架构 — 标准格式

把要出的图展开成**开箱即用**的 prompt 块——出图阶段可喂给 Codex 内置生图 / OpenAI / Gemini / Flux / ComfyUI / Dreamina 等图 AI。**分两层**：复用资产（定妆 = 角色/场景/反复入镜道具）放共享，单次资产（分镜）放本集。

---

## 1. 共享层 — `出图/common/`（全篇一份：prompt/ 文件夹 + PNG 同级）

```
出图/common/
├── prompt/                  共享 prompt 文件
│   ├── 00_索引.md           全篇定妆清单 + 状态 + 首现/复用范围 + PNG 路径
│   ├── 角色定妆.md          所有角色（含形态变体）实战 prompt
│   ├── 场景定妆.md          所有场景实战 prompt
│   └── 道具定妆.md          关键道具实战 prompt
├── 定妆_<角色>.png          正脸主参考（与 prompt/ 平级，扁平）
├── 定妆_<角色>_侧.png        最低定妆组·侧脸角度参考（核心角色）
├── 定妆_<角色>_半身.png      最低定妆组·半身服装参考（二选一）
├── 定妆_<角色>_全身.png      最低定妆组·全身服装参考（二选一）
├── 定妆_<角色>_背.png        增强项·背面参考（需要背身/转身镜时补）
├── 定妆_<角色>_表情.png      增强项·表情条（喜/怒/哀/惊）
├── 定妆_<角色>_<形态>.png    形态变体（觉醒态…，按最低定妆组再出）
├── 定妆_<场景>.png           场景定妆（跨集复用，别每集重画）
└── 定妆_<道具>.png
```

> **prompt / 产物分离铁律**：本 skill 任一目录（`common/` 或 `第N集/`）下，所有 prompt md 进 `prompt/` 子目录；生成产物 PNG 平铺在父目录。详见 `novel2drama/references/architecture.md` 二节。

### 1.1 `00_索引.md` 必含字段

- 三张表（角色 / 场景 / 道具），每条含 **ID**、名称、形态、首现集、复用范围、状态（✅/⏳/⬜）、PNG 路径
- **跨 AI 锚定句速查**（三大视频 AI 的中英锚定句）
- **全集统一负面词**
- **图 AI 统一参数**（模型 / 比例 / 精细度 / 数量 / 参考强度档位；Dreamina 参数只作兜底参考）
- **工作流约定**（新角色/场景/道具如何加入共享库）

### 1.2 `角色|场景|道具定妆.md` — 定妆 prompt 块标准格式

```markdown
## ① <ID> <名称>·<形态>（✅/⏳/⬜）

**首现**：第N集 / **复用**：第N-M集
**目标存档**：`出图/common/定妆_xxx.png`
**参考图来源**：（已完成共享图 / 无需参考图）
**角色定妆组**（核心 / 长线角色必出最低三件套；一次性配角可只出主参考一张）：
- **最低有效三件套**：
  - 主参考：`出图/common/定妆_<角色>.png`（正脸中性光，下游分镜的主参考图）
  - 角度参考：`定妆_<角色>_侧.png`（侧脸，锁鼻梁 / 下颌 / 发型侧轮廓）
  - 服装参考：`定妆_<角色>_半身.png` 或 `定妆_<角色>_全身.png`（二选一；锁衣领 / 袖型 / 腰带 / 配色 / 体态）
- **增强项**：`定妆_<角色>_背.png`（背身 / 转身镜多时补）、`定妆_<角色>_表情.png`（喜 / 怒 / 哀 / 惊，可单张多格或分张）
- 形态变体（觉醒态 / 银牌态…）各自至少出最低三件套，均以常态主参考做参考图（强度 0.75-0.85）

## 2. 本集层 — `出图/第N集/`（每集一份：prompt/ 文件夹 + PNG 同级）

```
出图/第N集/
├── prompt/
│   ├── 00_总览.md           本集图清单 + 引用共享库 + 本集分镜进度
│   └── 01_分镜出图.md       本集 N 张分镜实战 prompt（一镜一图，复杂镜可拆 NA/NB）
└── 镜头N_<描述>.png         本集分镜 PNG（与 prompt/ 平级）
```

### 2.1 `00_总览.md` 必含字段

- **本集图数统计**（共享复用 vs 本集分镜，分别列）
- **进度**（已完成 / 总数）
- **已完成图归档**（共享库链接 + 本集出图链接）
- **出图依赖图**（哪些分镜需要哪些定妆作参考）
- **镜头 → 必传参考图 速查表**（路径指向共享库 PNG）
- **不重复**写定妆 prompt（指向共享 `00_索引.md` 链接即可）

### 2.2 `01_分镜出图.md` — 单镜头格式

```markdown
## 镜头 N（节拍标签：如"前3秒冲突"/"反转"）

**剧本描述**：（复制自 分镜剧本.md 画面视觉描述）

**参考图**（**多图参考派生铁律**：镜头图一律基于这些参考图派生，禁纯文生图）：
- `出图/common/定妆_<角色>.png`（正脸主参考，强度 0.8）
- `出图/common/定妆_<角色>_侧.png`（角度锚，强度 0.5-0.6；侧脸 / 转头 / 过肩镜尤其加）
- `出图/common/定妆_<角色>_半身.png` 或 `_全身.png`（服装锚，强度约 0.5；全身动作 / 服装易漂时加）
- `出图/common/定妆_<场景>.png`（场景定妆，强度 0.4-0.5）

**导演视角八维**（分镜图按这八维装配，不是画师视角的"好看插画"；详见 `novel2drama/references/导演视角prompt.md`）：
| 维度 | 本镜填什么 |
|---|---|
| ① 镜头 | 景别(ELS…ECU) + 焦段/景深 |
| ② 机位 | 角度(平/俯/仰)+高度+过肩/正反打（机位即态度）|
| ③ 人物 | 锚点句锁脸锁妆 + 此刻状态(伤痕/凌乱/服装变化) |
| ④ 动作 | 动作最强的一瞬（静帧抓顶点）|
| ⑤ 场景 | 场景锚 + 时间(昼夜) + 环境状态 |
| ⑥ 光影 | 光源动机+方向+质感+调性（光替剧情说话，别均匀打亮）|
| ⑦ 情绪 | 本镜情绪 + 张力(克制/紧张/爆发/释放) + 色调情绪 |
| ⑧ 画质 | 画风词(依global_style)+渲染级+画幅+跨AI锚定句 |

### 正向 prompt（中文）
\`\`\`
（按八维装配顺序：[①镜头][②机位]，[③同一少女·锚点句·此刻状态] 正在 [④动作],
 置身 [⑤场景+时间+环境]，[⑥光影]，整体 [⑦情绪+张力+色调]，[⑧画风+渲染+画幅+视频AI锚定句]）
\`\`\`

### 正向 prompt（英文备用）
\`\`\`
...
\`\`\`

### 负向 prompt
\`\`\`
（统一负面词 + 本镜专属堵漏词）
\`\`\`

### 检查清单（八维自查·最易漏②机位/⑥光影/⑦张力）
1. ✅ 脸型与定妆一致（③人物锚点句已拼）
2. ✅ 服装配色一致 + 此刻状态对（③）
3. ✅ 景别符合分镜要求 + 机位有理由非默认正面平视（①②）
4. ✅ 光影在叙事非均匀打亮（⑥）
5. ✅ 表情符合本镜情绪 + 张力/色调一致（⑦）

### 自检（生成后逐张过 · 落档闸门）
> 这是**生图后**的过/重抽判定，与上面的"出图前 prompt 八维自查"互补。按**筛选宽容铁律**：轻微偏差直接放行，只命中硬伤才重抽；重抽上限由 `重抽预算策略` 档位决定（见 SKILL.md「重抽预算铁律」三档表）。
```markdown
**自检**（轻微偏差放行，只命中硬伤才重抽）：
- [ ] 核心人/物/场景无错位 — 本镜须含：<列出该出现的人/物/场景>
- [ ] 角色脸/妆造未漂移（对照 定妆_<角色>.png 主参考；主要人物零漂移容忍）
- [ ] 无硬性禁忌（景别须=<本镜景别>；无字幕/logo/多余手指/现代物件；<本镜专属禁忌如"不要血浆"/"无清晰文字"/"不展示实体">）
- 重抽预算：<主要人物/关键镜 or 配角/普通镜，按当前策略档填上限> ｜ 实抽__次 → ⬜过 ⬜重抽 ⬜到顶取定妆最一致版
```
- **须含**取自该镜「参考图」行（须出现的角色/场景/道具）；**景别**取自正向 prompt 首段（LS/MCU/CU/ECU…）；**专属禁忌**取自正向 prompt 末尾的本镜约束。
- **关键镜**（爽点/反转/觉醒/威压/封面候选）在标题加 `🔑关键镜` 标记，套用更宽的重抽上限。

### 备注
（特殊处理、降级方案等；如"本镜运动复杂可改为静帧 + 后期添加动态模糊"）
```

## 3. 工作流（新角色/场景/道具首次出现时）

```
第N集 出料阶段发现新角色"X"（n2d-script 阶段已建 characters/X.md 含 ① 定妆 prompt）
  ↓ n2d-image 阶段：
① 出图/common/prompt/00_索引.md 追加 ⬜ 行（ID/名称/首现/复用）
  ↓
② 出图/common/prompt/角色定妆.md 追加完整 prompt 块（本文 §1.2 格式）
   （来源 = characters/X.md 的 ① 定妆 prompt + Stage 2 的实战包装：图 AI 设置 + 检查清单）
  ↓
③ 按 `cli_registry.md` 重新扫描后优先用 Codex/OpenAI/Gemini/Flux/ComfyUI 等自动入口；Dreamina/即梦只兜底 → 挑图 → PNG 落到 出图/common/
  ↓
④ 索引状态改 ✅，填 PNG 路径
  ↓
⑤ 出图/第N集/prompt/00_总览.md 引用此 PNG
  ↓
⑥ 后续集自动复用，不再重做
```

## 4. 与 Stage 1 文件的关系

| 文件 | 谁写 | 角色 |
|---|---|---|
| `脚本/第N集/分镜剧本.md` | n2d-script | 提供"画面视觉描述"原文（艺术层）|
| `脚本/第N集/素材清单.md` | n2d-script | 简版图清单（理论） |
| `设定库/characters/<X>.md` 的 ① Prompt | n2d-script | 角色定妆 prompt 源头 |
| `出图/common/prompt/角色\|场景\|道具定妆.md` | **n2d-image** | **实战定妆 prompt**（含 CLI 调用 + 检查清单 + 已知瑕疵） |
| `出图/第N集/prompt/01_分镜出图.md` | **n2d-image** | **实战分镜 prompt**（含参考图 + 操作） |
| `出图/第N集/prompt/00_总览.md` | **n2d-image** | 本集图清单 + 引用共享 + 进度 |
| `脚本/第N集/故事板.md` | n2d-script | Clip 表，**Stage 3 视频生成**用，不在 image prompt 文件夹中 |

**一句话**：
- `素材清单.md` = **简版图清单**（理论，Stage 1 出）
- `出图/common/` = **全篇定妆库**（按角色/场景维度组织，跨集复用，Stage 2 实战包装）
- `出图/第N集/` = **本集分镜库**（按集 + 镜头维度组织，Stage 2 出）
- 跨 AI 时（图 AI ≠ 视频 AI）每个 image prompt 末尾要拼视频 AI 的"图像风格锚定句"，详见 `platforms.md`。
