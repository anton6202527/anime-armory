# 出图 prompt 两层架构 — 标准格式

把要出的图展开成**开箱即用**的 prompt 块——出图阶段可喂给 `生图AI` 所选官方/已登录图后端（默认 Codex，可选 Dreamina/即梦官方 CLI / Seedream / 可灵主体库 / Nano Banana / Sora Cameo 等）。**分两层**：复用资产（定妆 = 角色/场景/反复入镜道具）放共享，单次资产（分镜）放本集。

---

## 1. 共享层 — `出图/共享/图片/`（全篇一份：prompt/ 文件夹 + 图片/ 文件夹）

```
出图/共享/
├── identity_registry.json   机器真值：角色/形态的参考组 + 后端身份适配 + 角度策略 + 禁漂项
├── prompt/                  共享 prompt 文件
│   ├── 00_索引.md           全篇定妆清单 + 状态 + 首现/复用范围 + PNG 路径
│   ├── 角色定妆.md          所有角色（含形态变体）实战 prompt
│   ├── 场景定妆.md          所有场景实战 prompt
│   ├── 道具定妆.md          关键道具实战 prompt
│   ├── 法宝定妆.md          ⚙️仙侠玄幻可选：法宝/法器（含形态/成长阶段多态）
│   └── 特效定妆.md          ⚙️仙侠玄幻可选：剑气/灵力光效/法术/护体光/阵法（锁颜色/形状/拖尾/强度）
└── 图片/                    共享 PNG 产物（与 prompt/ 平级）
    ├── 定妆_<角色>.png          标准三视图·正面主参考
    ├── 定妆_<角色>_侧.png        标准三视图·侧面参考
    ├── 定妆_<角色>_背.png        标准三视图·背面参考
    ├── 定妆_<角色>_半身.png      服装/体态参考（二选一）
    ├── 定妆_<角色>_全身.png      服装/体态参考（二选一）
    ├── 定妆_<角色>_三视图.png    人审归档·由正/侧/背生产拆图拼成的标准三视图
    ├── 定妆_<角色>_设定表.png    人审归档·含服装/表情增强项时可用这个更宽泛命名
    ├── 定妆_<角色>_表情.png      增强项·表情条（喜/怒/哀/惊）
    ├── 定妆_<角色>_<形态>.png    形态变体
    ├── 定妆_<场景>.png           场景定妆主图
    ├── 定妆_<场景>_<向>.png      场景多视图
    ├── 定妆_<道具>.png
    ├── 定妆_<法宝>.png / 定妆_<法宝>_<阶段>.png
    └── 定妆_<特效>.png
```

> **prompt / 产物分离铁律**：本 skill 任一目录（`common/` 或 `第N集/`）下，所有 prompt md 进 `prompt/` 子目录；生成产物 PNG 进 `图片/` 子目录。详见 `novel2drama/references/architecture.md` 二节。

### 1.1 `00_索引.md` 必含字段

- 表（角色 / 场景 / 道具；**⚙️仙侠玄幻题材再加 法宝 / 特效 两张表**），每条含 **ID**、名称、形态、首现集、复用范围、状态（✅/⏳/⬜）、PNG 路径
- `identity_registry.json` 同步登记提示：角色/形态行必须能对应到 `characters[].id` + `forms[].form`；后端角色 ID / Face Lock / LoRA 状态以 registry 为机器真值，`00_索引.md` 只做人读摘要
- **跨 AI 锚定句速查**（三大视频 AI 的中英锚定句）
- **全集统一负面词**
- **图 AI 统一参数**（Codex/官方 OpenAI/Dreamina/Seedream 等模型 / 比例 / 精细度 / 数量 / 参考强度档位）
- **工作流约定**（新角色/场景/道具/法宝/特效如何加入共享库）

### 1.2 `角色|场景|道具定妆.md` — 定妆 prompt 块标准格式

```markdown
## ① <ID> <名称>·<形态>（✅/⏳/⬜）

**首现**：第N集 / **复用**：第N-M集
**目标存档**：`出图/共享/图片/定妆_xxx.png`
**身份注册**：`出图/共享/identity_registry.json` → `characters[].forms[]`（角色/形态必填；场景/道具/法宝/特效写“无”）
**参考图来源**：（已完成共享图 / 无需参考图）
**角色定妆组**（所有人物必出标准三视图 + 人审拼版；不按主配角或题材省背面）：
- **标准三视图生产拆图**：
  - 正面主参考：`出图/共享/图片/定妆_<角色>.png`（正脸中性光，下游分镜的主参考图）
  - 侧面参考：`图片/定妆_<角色>_侧.png`（侧面，锁鼻梁 / 下颌 / 发型侧轮廓）
  - 背面参考：`图片/定妆_<角色>_背.png`（背面，锁后脑勺发型 / 背部服饰 / 披风 / 腰带绑法）
  - 服装参考：`图片/定妆_<角色>_半身.png` 或 `图片/定妆_<角色>_全身.png`（二选一；锁衣领 / 袖型 / 腰带 / 配色 / 体态）
- **人审拼版**：`图片/定妆_<角色>_三视图.png`（由已通过自检的正面 / 侧面 / 背面生产拆图拼成，用于快速审脸、审服装、归档，不作为分镜主参考图）
- **增强项**：`图片/定妆_<角色>_设定表.png`（可纳入半身/全身/色卡/配饰细节）与 `图片/定妆_<角色>_表情.png`（喜 / 怒 / 哀 / 惊，可单张多格或分张）
- 形态变体（觉醒态 / 银牌态…）各自至少出标准三视图 + 拼版，均以常态主参考做参考图（强度 0.75-0.85）
- **命名约定**：多张可喂给图 AI 的拆分 PNG 叫「定妆组」；单张合成总览才叫「三视图」或「设定表」。不要把多张散图标成“三视图已完成”。

> **⚙️法宝 / 特效定妆（仙侠玄幻题材专类）**：法宝按**形态/成长阶段**出多态（类比角色形态变体，如 `图片/定妆_飞剑_初始.png` / `图片/定妆_飞剑_觉醒.png`）；特效（剑气/灵力光效/法术/护体光/阵法）定**颜色 / 形状 / 拖尾 / 强度**，含特效镜头一律多图参考派生（角色定妆+特效定妆+上一关键帧）防漂。详见 `n2d-script/references/打斗分镜.md §五`。

### 自检（生成后逐张过 · 落档闸门）
> 定妆是**锚点**——脸一漂下游每个镜头都继承错误，所以这层比分镜更该卡。与上面"出图前检查"互补；同样按**筛选宽容铁律**：轻微偏差放行，只命中硬伤才重抽，重抽上限由 `重抽预算策略` 档位决定（定妆套"主要人物/关键镜"那列上限，见 SKILL.md「重抽预算铁律」三档表）。**定妆自检的重点不是单张好看，是"多张之间同一个人 + 与角色卡逐项对得上"。**
```markdown
**自检**（轻微偏差放行，只命中硬伤才重抽）：
- [ ] 妆造拆解逐项对上角色卡（发型 / 妆容 / 服装 / 配饰 / 色卡，缺一项即漂）
- [ ] `identity_registry.json` 已登记本角色/形态的 `reference_group` / `identity_adapters` / `angle_policy` / `drift_forbidden`
- [ ] 标准三视图是**同一个人**（侧面 / 背面的脸型轮廓、发型、后脑勺发型、背部服饰与正面主参考一致）
- [ ] 人审拼版已生成：`图片/定妆_<角色>_三视图.png`，且拼版中的正面 / 侧面 / 背面来自已通过自检的生产拆图
- [ ] 形态变体仍认得出是同一角色（以常态主参考派生，没变成另一个人）
- [ ] 场景定妆：跨集复用基准成立（结构 / 色调 / 材质锁得住，下游引用不会忽大忽小）
- [ ] 关键反复场景（多集核心场景）按机位补了**多视图**（`图片/定妆_<场景>_<向>.png`：正向/侧向/反向/俯瞰），跨镜头切换同一空间背景自洽；一次性场景单图即可，不必铺
- [ ] 无硬性禁忌（无字幕 / logo / 多余手指 / 现代物件；<本项目专属堵漏，如觉醒态"不要铠甲护臂">）
- 重抽预算：<主要角色零漂移容忍·套关键镜上限 or 短线配角·普通上限，按当前策略档填> ｜ 实抽__次 → ⬜过 ⬜重抽 ⬜到顶取定妆最一致版
```
- **过 = 可落档**：PNG 落 `出图/共享/图片/`，并把 `00_索引.md` 该条状态改 ✅（这一步就是「共享先行硬闸门」放行本集分镜图的前提）。
- 主要角色（复用范围标【全篇】或长线核心）**零漂移容忍**；一次性/短线配角走普通上限。

> **执行硬闸门**：共享定妆 prompt 块缺 `检查清单（定妆自查·最易漏③人物/⑥中性光影/一致性）` 或缺 `自检（生成后逐张过 · 落档闸门）` 时，不得提交生图. 定妆不套完整剧情八维，但必须先检查关键视觉特征、中性光、不带戏、跨图一致性。

> **口径说明**：分镜图是剧情镜头，必须按导演视角八维装配，所以使用 `检查清单（八维自查·最易漏②机位/⑥光影/⑦张力）`；定妆图按产线规则是中性档案，不做剧情八维、不带戏剧光和情绪，只锁脸/造型/场景/道具基准，所以使用 `检查清单（定妆自查·最易漏③人物/⑥中性光影/一致性）`。执行时先验证检查结构，再提交生图。

> **中英双 prompt**：中文和英文正向 prompt 默认都写，英文不是可省略备注。中文 prompt 可能被部分平台安全策略误伤或误解，英文版用于同义兜底；提交失败、中文触发规避或画面跑偏时，直接用英文版重试。

## 2. 本集层 — `出图/第N集/图片/`（每集一份：prompt/ 文件夹 + 图片/ 文件夹）

```
出图/第N集/图片/
├── prompt/
│   ├── 00_总览.md           本集图清单 + 引用共享库 + 本集分镜进度
│   └── 01_分镜出图.md       本集 N 张分镜实战 prompt（一镜一图，复杂镜可拆 NA/NB）
└── 图片/
    └── 镜头N_<描述>.png         本集分镜 PNG
```

### 2.1 `00_总览.md` 必含字段

- **本集视觉一致性契约**（⚠️像素层先行 · 出图阶段把视觉变量焊死成像素，凡视频改不动的导演决策都在这里下完）。**真值源在上游**：n2d-script 已在分镜设计阶段把这些写进 `脚本/第N集/storyboard.json` 的 `visual_contract` 种子块（场景光位锚/场景轴线视线/角色状态演进/景别阶梯/色调基线）——本契约是把它**誊抄并细化到出图层**（补每场景具体光位措辞、逐角色锚点对应的状态镜号），**不是凭空重新发明**；种子缺失时回 `/n2d-script` 补 `visual_contract` 再出图。下游 `出视频/第N集/prompt/00_总览.md` 的「本集导演一致性契约」再**继承本契约**，三层同源、不各写一套。`gate.py --stage image` 缺契约或缺任一字段即阻断，与出视频侧同级严谨：
  - **色调基线**：本集/本段主色调 + 哪些色彩或光效只能在指定爽点后出现（如金瞳、妖气、系统光）——与出视频契约同源，色一旦烤进像素，视频改不回。
  - **场景光位锚**：逐场景定「主光方向 + 色温 + 动机光源」（如「冷宫正殿：画左前烛光顶侧光 / 3000K 暖 / 动机=残烛」）。**该场所有分镜继承同一光位锚**，跨镜不许各打各的光；剧情改光（吹烛/开窗/点灯）必须显式写理由。光位飘=剪起来闪，和脸漂同级穿帮。
  - **场景轴线·视线**：逐场景定「主要人物左右站位 + 轴线方向 + 默认视线方向」。**正反打两镜按此对位**——轴线/视线在出图阶段焊进首帧像素，出视频救不回（视频只动作不改站位/视线），所以必须在这里先定。换轴必须有反打/空镜/动作理由。
  - **角色状态演进表**：逐角色列「伤 / 泪 / 妆 / 服 / 发」随镜号推进的时间线（如「沈念：镜3 起左颊新伤→镜3+全程保留；镜7 发髻散乱→不再复原」）。**单调不回退**（出血的镜后面不许干净、乱发不许自愈），**状态不提前泄露**（觉醒前不发光、变身前不出现变体）。首帧逐张照此表烤渐进状态。
  - **景别阶梯**：本集/各段落的景别走向（establish→中景→特写…），**相邻镜不撞同景别同机位**（防 jump cut 廉价感）；要刻意 match cut 的同景别接点须显式标注。
- **本集基础视觉风格契约**（⚠️风格层先行 · 把用户选择的基础风格变成可继承规则）。**真值源在上游**：n2d-script 已在 `storyboard.json` 写 `style_contract` 种子块，本步誊抄并细化到出图层；种子缺失时回 `/n2d-script` 补 `style_contract`。`gate.py --stage image` 缺契约或缺任一字段即阻断：
  - **风格名**：来自 `_设置.md` 的 `基础视觉风格`，如写实电影感 / 国漫写实 / 二次元赛璐璐 / 水墨国风 / 厚涂幻想 / 赛博霓虹 / Q版轻喜 / 自定义。
  - **视觉基调**：该风格的整体审美、角色比例、材质层次和画面密度；不是泛化“高清漂亮”。
  - **镜头与构图**：该风格下可用的景别、焦段感、透视、留白/剪影/线稿等构图原则；每镜从这里取，不随机变透视。
  - **光色策略**：主色 + 强调色 + 强调色出现时机；不同风格可以低饱和、平涂高明度、水墨墨色或霓虹高对比，但必须有统一纪律。
  - **运动边界**：出图首帧为视频留的运动空间要与风格相容；写实可克制手持，赛璐璐可更弹性，水墨偏慢，赛博可更锐利，但都禁止无理由乱甩。
  - **风格禁忌**：随所选风格派生。写实电影感可禁插画化/游戏CG；二次元赛璐璐不应禁“插画感”，而应禁照片皮肤/3D塑料/风格跳变。
- **本集专项镜头模板速查**（⚠️复杂镜头先继承模板，再出首帧/尾帧）。**真值源在上游**：`storyboard.json clips[]` 的 `template/template_contract`。凡打斗、追逐、对话反打、法术爆发、飞行、亲密互动、多人站位，逐 Clip 列出模板 ID、beats、blocking、camera_rule、continuity_must、negative 和模板专属字段；逐镜 prompt 必从这里誊抄，避免临场自由写动作。缺模板或字段不全时回 `/n2d-script` 补 `专项镜头模板库.md` 契约。
- **本集资产身份速查**（⚠️角色身份先继承 registry）。**真值源在共享层**：`出图/共享/identity_registry.json`。凡本集入镜角色/形态，逐项列 `registry id`、`reference_group`、图/视频后端身份状态、允许角度、高危角度、禁漂项；逐镜 prompt 必从这里取角色参考组和禁漂约束，不能只看 `00_索引.md` 临场猜参考图。
- **本集图数统计**（共享复用 vs 本集分镜，分别列）
- **进度**（已完成 / 总数）
- **已完成图归档**（共享库链接 + 本集出图链接）
- **出图依赖图**（哪些分镜需要哪些定妆作参考）
- **镜头 → 必传参考图 速查表**（路径指向共享库图片目录）
- **不重复**写定妆 prompt（指向共享 `00_索引.md` 链接即可）

### 2.2 `01_分镜出图.md` — 单镜头格式

```markdown
## 镜头 N（节拍标签：如"前3秒冲突"/"反转"）

**剧本描述**：（复制自 分镜剧本.md 画面视觉描述）

**参考图**（**多图参考派生铁律**：镜头图一律基于这些参考图派生，禁纯文生图）：
- `出图/共享/图片/定妆_<角色>.png`（正脸主参考，强度 0.8）
- `出图/共享/图片/定妆_<角色>_侧.png`（角度锚，强度 0.5-0.6；侧脸 / 转头 / 过肩镜尤其加）
- `出图/共享/图片/定妆_<角色>_背.png`（背面锚，强度约 0.5；背身 / 转身 / 追逐 / 过肩镜加）
- `出图/共享/图片/定妆_<角色>_半身.png` 或 `图片/定妆_<角色>_全身.png`（服装锚，强度约 0.5；全身动作 / 服装易漂时加）
- `出图/共享/图片/定妆_<场景>.png`（场景定妆，强度 0.4-0.5）

**视线方向**（含角色镜必填 · 出图阶段焊进像素 · 视频改不动）：画左 / 画右 / 镜头外上方…（取自 `00_总览.md` 本场轴线；与反打镜 `Clip K±1` 对位，不越轴）
**光位锚**（继承本场 · 跨镜不各打各的光）：继承 `00_总览.md` 本场「场景光位锚」（主光方向 + 色温 + 动机光源）；本镜默认不改光，若改光写剧情理由
**起幅·运动余量**（出图为视频铺路 · clip 首帧=起幅，非动作顶点）：本镜为 Clip K **首帧=起幅**（动作顶点交尾帧 `镜头N_end.png`，封面/定格图才抓顶点）；按 `故事板.md` 本镜节奏/张力反推的运镜（推近/环绕/跟摇）**预留构图余量**——推近→框略宽、环绕→主体周围留空、跟摇→运动方向留 lead room
**专项镜头模板**（复杂镜必填 · 普通镜写“无”）：若所属 Clip 有 `template/template_contract`，这里誊抄模板 ID + 本镜要落实的 `beats/blocking/camera_rule/continuity_must/negative` + 模板专属字段（如 `impact_frame`、`screen_direction`、`axis`、`effect_asset`、`contact_points`、`screen_positions`）。本镜构图必须服务模板契约：例如反打锁轴线、飞行锁姿态动背景、亲密互动锁接触点、多人数锁主次站位。
**资产身份注册层**（含角色镜必填）：从 `出图/共享/identity_registry.json` 读取本镜角色/形态的 `reference_group`、可用后端角色 ID/主体库、`angle_policy` 和 `drift_forbidden`；普通无人物镜写“无”。若命中高危角度（极端俯仰/深暗部/人物过小/多人接触），必须补对应参考或登记降级方案。

**导演视角八维**（分镜图按这八维装配，不是画师视角的"好看插画"；详见 `novel2drama/references/导演视角prompt.md`）：
| 维度 | 本镜填什么 |
|---|---|
| ① 镜头 | 景别(ELS…ECU) + 焦段/景深；与相邻镜成**景别阶梯**，不撞同景别（见 00_总览） |
| ② 机位 | 角度(平/俯/仰)+高度+过肩/正反打（机位即态度）；**守本场轴线、视线方向对位**（见上「视线方向」字段） |
| ③ 人物 | 锚点句锁脸锁妆 + 此刻状态(伤痕/凌乱/服装变化)，**照 00_总览 角色状态演进表，单调不回退** |
| ④ 动作 | clip 首帧抓**起幅**（留运动余量给视频）、动作顶点交尾帧；仅封面/定格图抓顶点 |
| ⑤ 场景 | 场景锚 + 时间(昼夜) + 环境状态 |
| ⑥ 光影 | **继承本场光位锚**（主光方向+色温+动机光源）+ 质感+调性（光替剧情说话，别均匀打亮，跨镜别飘）|
| ⑦ 情绪 | 本镜情绪 + 张力(克制/紧张/爆发/释放) + 色调情绪 |
| ⑧ 画质 | 画风词(依global_style)+渲染级+画幅+跨AI锚定句 |

## 3. 工作流（新角色/场景/道具首次出现时）

```
第N集 出料阶段发现新角色"X"（n2d-script 阶段已建 characters/X.md 含 ① 定妆 prompt）
  ↓ n2d-image 阶段：
① 出图/共享/prompt/00_索引.md 追加 ⬜ 行（ID/名称/首现/复用）
  ↓
② 出图/共享/identity_registry.json 追加该角色/形态（reference_group / identity_adapters / angle_policy / drift_forbidden）
  ↓
③ 出图/共享/prompt/角色定妆.md 追加完整 prompt 块（本文 §1.2 格式）
   （来源 = characters/X.md 的 ① 定妆 prompt + Stage 2 的实战包装：图 AI 设置 + 检查清单）
  ↓
④ 按 `cli_registry.md` 重新扫描后只用 `生图AI` 所选官方/已登录后端；Dreamina/即梦官方 CLI 可用于图片生成，第三方逆向/web 自动化禁用 → 挑图 → PNG 落到 出图/共享/图片/
  ↓
⑤ 索引状态改 ✅，填 PNG 路径；registry 的 reference_group 路径也必须指向实际 PNG
  ↓
⑥ 出图/第N集/prompt/00_总览.md 引用此 PNG
  ↓
⑦ 后续集自动复用，不再重做
```

## 打斗集出图（含武打/法术时）

> 含武打/法术的集，出图按 `n2d-script/references/打斗分镜.md` 工艺。要点：

- **一招出关键帧定格图**：起手帧 + **命中帧**（供 n2d-video 当首尾帧）。命中帧是打击感核心，画冲击波/火花/凹陷/受击 + 速度线 + motion blur。
- **法术/武器/光效进共享定妆库** `出图/共享/图片/`（如 `图片/定妆_青色剑气.png`），锁颜色/形状/拖尾；含特效镜头走**多图参考派生**（角色定妆 + 特效定妆 + 上一关键帧）防漂。
- prompt 写"动态定格 dynamic action pose / 发力瞬间 / 拧腰送肩 / 拖影"。
- 详见 `n2d-script/references/打斗分镜.md`（§二 五帧拆招、§五 特效一致、§九 示例）。

## 仙侠场面出图（御剑飞行/追逐/渡劫/炼丹炼器/大阵/大场面/斗法对轰/神魂）

> 含这些仙侠奇观的集，出图按 `n2d-script/references/仙侠场面分镜.md` 工艺。要点：

- **奇观元素进共享定妆库** `出图/共享/图片/`（飞剑/祥云、劫雷/护体光/突破光柱、丹炉/丹火/成丹、阵图、地标 establish），和角色脸/法宝特效同等对待，锁颜色/形状/质感——会漂就锁。
- **飞行**：出一张「巡航定妆」（标准飞行姿态）跨镜复用，治"飘"；速度感留给视频（动背景），出图只锁姿态与剑/云形态。
- **渡劫/炼丹/法阵**：**爆发帧单独出图**（雷击命中帧 / 开炉成丹帧 / 全阵激活帧）当 n2d-video 首尾帧——和打斗"命中帧"同地位.
- **大阵**：出一张「全阵俯视锚图」定几何+符文，局部镜从它**多图参考派生**（几何对称 AI 必崩，靠复用治）。
- **大场面 establish**：地标建「场景定妆」跨集复用；按"远全貌→中地标→近人物入画"三镜出，并放比例尺参照物（小人/小舟）衬恢弘。
- **斗法对轰**：撞点（能量对冲炸团）单独出图当首尾帧；双方法术光束**两色区分**（如金 vs 黑）各自入库锁一致。
- **神魂**：元神/神识波纹入库；元神**用肉身定妆做半透明发光态参考派生**（强度 0.75-0.85），治"二我"穿帮；夺舍成败帧（瞳色变）单独出图.
- 含奇观元素的镜头一律**多图参考派生**（角色定妆 + 奇观元素定妆 + 上一关键帧）防漂。
- 详见 `n2d-script/references/仙侠场面分镜.md`（§一~§八 各类拆解，含斗法对轰/神魂；§九 落地速查）。

## 4. 与 Stage 1 文件的关系

| 文件 | 谁写 | 角色 |
|---|---|---|
| `脚本/第N集/分镜剧本.md` | n2d-script | 提供"画面视觉描述"原文（艺术层）|
| `脚本/第N集/素材清单.md` | n2d-script | 简版图清单（理论） |
| `设定库/characters/<X>.md` 的 ① Prompt | n2d-script | 角色定妆 prompt 源头 |
| `出图/共享/prompt/角色|场景|道具定妆.md` | **n2d-image** | **实战定妆 prompt**（含 CLI 调用 + 检查清单 + 已知瑕疵） |
| `出图/第N集/prompt/01_分镜出图.md` | **n2d-image** | **实战分镜 prompt**（含参考图 + 操作） |
| `出图/第N集/prompt/00_总览.md` | **n2d-image** | 本集图清单 + 引用共享 + 进度 |
| `脚本/第N集/故事板.md` | n2d-script | Clip 表，**Stage 3 视频生成**用，不在 image prompt 文件夹中 |

**一句话**：
- `素材清单.md` = **简版图清单**（理论，Stage 1 出）
- `出图/共享/图片/` = **全篇定妆库**（按角色/场景维度组织，跨集复用，Stage 2 实战包装）
- `出图/第N集/图片/` = **本集分镜库**（按集 + 镜头维度组织，Stage 2 出）
- 跨 AI 时（图 AI ≠ 视频 AI）每个 image prompt 末尾要拼视频 AI 的"图像风格锚定句"，详见 `platforms.md`。
