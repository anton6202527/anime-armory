# 出视频后端 + 模型路由（ad-video · 本线自持）

`生视频AI` 默认即梦。`视频模型路由=自动按镜头路由` 时按镜头类型选 primary/fallback；
否则固定 `生视频AI`。后端能力随版本变，正文写能力不绑版本。

## 镜头类型 → 路由建议

| 镜头类型 | primary（能力优先） | fallback | 为什么 |
|---|---|---|---|
| 产品展示 / hero 环绕 | 主体一致性强（Seedance Face/主体 Lock、可灵主体库） | 即梦 | 包装/logo 不能抖花，要稳 |
| 情绪 / 人物特写 | 电影感强（Veo） | 即梦 | 表演与质感 |
| demo 实拍质感 / 手持 | 真实运动后端 | 即梦 | 拟真手持、自然动态 |
| 痛点情境 / 叙事镜 | 即梦 / 通用 | 通用 | 普通叙事 |
| 空镜 / 转场 | 即梦 / 通用 | 通用 | 低风险 |
| end card / 包装定格 | 静帧或极慢运镜 | — | 文字/logo 要稳，必要时 ad-compose 合成 |

## 单 Clip 时长上限按后端

即梦 image2video ≤8s / Seedance ≤15s / 可灵 ≈10s / Veo ≈8s。广告镜短，一般够；能一镜到底就别切碎。

## 出视频规格（`出视频规格`）

- 预算充足：1080p30fps，关键镜多跑挑稳。
- 预算一般：720p24–30fps（默认）。
- 预算不够：720p24fps 一条过。

## 两条硬约束

1. **契约继承**：品牌色/光位/轴线必须从出图继承（`inherit_contract.py` block）。
2. **不混后端当默认**：路由是按镜头选 primary/fallback 并记录在路由表，不是随意混用。
