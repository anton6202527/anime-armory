## 零、批量工业化判断（2026-06-09）

| | 现状 |
|---|---|
| **外部生态** | Veo 3.1 / Kling 3.0 / Runway / Luma 等都已把视频生成做成 API/任务化能力；Veo 3.1 官方 API 支持 8s、720p/1080p/4K、原生音频并按秒计费，Kling 3.0 官方强调 15s、原生多语言音频、多镜头故事板、参考视频/多图一致性，Runway/Luma 则提供 image-to-video、text-to-video、keyframe/extend 等可编排任务。|
| **对 n2d 的含义** | n2d 的“拆镜头 + 资产库 + 模型路由 + gate + 返工队列”路线成立。长剧不应赌单次长视频生成，而应拆成可追踪资产和可重跑 Clip。|
| **当前级别** | 可定位为**工作室级工业化雏形**：能批量、能记账、能审查、能回流，但还不是完全无人值守的重工业产线。|
| **硬瓶颈** | 成本/重抽率、跨集身份一致性、复杂物理交互、口型/音画同步、动作节拍、合规水印/披露、多机调度、投放数据回灌。|
| **升级原则** | 不因某个新模型更强就重写流程；先更新 `platforms.md/backends.md` 能力档案，再让 `n2d-model-router` 按镜头能力和 dashboard 真实通过率/成本选择后端。|

## 横切：音画联合生成趋势（可控性）

市场正走向**原生音画联合生成**（Veo 3.1 48kHz 对白、Seedance 2.0 原生音视频、Kling Omni 原生口型），"一次出音效/口型/音乐/画面同步"。

n2d 的取舍是**配音先行 + 默认丢弃 clip 原生音轨**（避免与克隆配音双人声）——这是为了**角色音色一致性 + 节奏可控**的主动选择，不是落后。已给"原生音画"开 P1 opt-in：低风险空镜/无口型/无台词镜头可把 Veo/Seedance/Kling 环境声或动作音效低音量混入；正面说话、旁白、系统音、克隆音色镜头仍禁止原生人声接管。

**口型 lip-sync（正面说话特写才值得，默认不做）**：

| | 现状 |
|---|---|
| **SOTA 快照(2026-06)** | 平台原生：Kling Omni 原生 5 语种口型、Veo 3.1 原生对白。后期开源：**MuseTalk**（口型+画质均衡、近实时，当前最佳免费）/ Wav2Lip（SyncNet 同步精度最稳）/ LatentSync（扩散、身份保持好） |
| **n2d 默认** | 默认不做；启用时优先平台原生（可灵 3.0），否则后期 pass 首选 MuseTalk（见 `n2d-video` SKILL「对口型 lip-sync」节） |
| **升级触发** | 正面说话特写多、口型对不上很跳时启用；规模化/真人底片 → Wav2Lip。能不做就靠分镜规避（侧脸/背身/空镜配旁白）最省 |
