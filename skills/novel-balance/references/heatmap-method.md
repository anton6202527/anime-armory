# 情节热力图方法

把"全书读起来忽冷忽热"量化成可看的曲线，定位**注水段**和**节奏断裂点**。

## 两条曲线（逐章/逐场景）

1. **情绪起伏度**：每章的冲突强度 + 情绪极性变化幅度。可用确定性近似打底（冲突/情绪关键词密度、对白占比、短句比 → 越高越"燃"），再由 LLM 校准语义。
2. **信息密度**：每章推进了多少**新信息**（新事件/新设定/新关系/锚点）。低密度连续多章 = 注水嫌疑。

## 预警规则

| 预警 | 触发 | 建议 |
|---|---|---|
| 🌊 注水段 | 连续 ≥N 章信息密度低于全书均值的某比例 | 并章 / 砍内心戏（→ `novel-condense`） |
| 📉 节奏脱节 | 相邻章情绪起伏度落差超带宽（高潮后骤平且无缓冲） | 加赴约/复盘缓冲章，或重排序 |
| 🏔️ 高潮过密 | 高燃章扎堆、缺铺垫 | 拉开节奏，给爽点蓄力 |
| 😴 开篇慢热 | 前 3 章情绪/信息双低 | 对照 `novel-craft/references/outline.md` 平台节奏模板重排前 10 章 beat |

## 确定性信号口径（家族单一定义源）

`pacing_analyzer.py` 按以下确定性近似逐章打分（同家族"脚本算确定项、LLM 判语义项"分工）；novel 家族其它 skill 涉及"爽点/节奏"的确定性信号都以此为准：

| 信号 | 算法（确定性） | 缩放 |
|---|---|---|
| 冲突强度 | 冲突/转折关键词命中 / 千字（关键词集见脚本 `CONFLICT_KW`） | 千字密度 0→12 线性映射到 1→10 |
| 信息密度 | 4-gram 词汇多样性（去重比），高=新信息多 | 0.50→0.95 映射到 1→10 |
| 爽点密度 | 爽点关键词命中 / 千字（`PAYOFF_KW`，**与 `novel-simulate` rookie 人格 `kw` 同一口径**） | 千字密度 0→10 映射到 1→10 |

> 口径单点：`novel-simulate` 的 `retention_prior` 与本表 `爽点密度` 共用同一爽点关键词集；`novel-score` 的 `payoff_density` 是 **LLM 语义判定**（另一层），可参考本热力图做先验，但不强依赖脚本数值。

## 与家族联动

- 与 `novel-review` 节奏维度互补：review 人判"这章钩子弱不弱"，balance 给"全书哪段塌"的鸟瞰。
- 热力图里的**高冲突点**自动推给 `novel-promote` 当宣发爆点源。
- 修法回写章纲（`outline.md`），不在本 skill 直接改文。
- **不喂分给 novel-score**：score 独立判"能不能火"，balance 只做节奏体检，二者无数据依赖（见 SKILL.md 边界段）。
