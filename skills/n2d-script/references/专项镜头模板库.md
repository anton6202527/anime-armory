# 专项镜头模板库（复杂镜头）

复杂镜头不要每次从零写 prompt。凡涉及高动作、高空间关系或多人调度的镜头，先在本库选模板，再写 `分镜剧本.md`、`故事板.md` 和 `storyboard.json`。模板的目标不是把画面写死，而是把 AI 最容易失控的变量提前锁住：轴线、站位、动作节拍、起幅落幅、关键帧、负向约束和下游回流点。

## 0. storyboard.json 必填契约

命中本库任一模板的 Clip，必须在 `clips[]` 里写：

```json
{
  "id": "EP01_CLIP04",
  "label": "王敦追入山洞",
  "template": "chase",
  "template_contract": {
    "template_id": "chase",
    "beats": ["起跑", "障碍", "距离变化", "落幅"],
    "blocking": "王敦画面左前向画右深处奔跑，追兵只用影子/脚步声提示",
    "camera_rule": "跟拍或侧向平移，不越轴，不做无理由旋转",
    "continuity_must": ["出入画方向保持左→右", "衣服发型不变", "洞口光位不跳"],
    "negative": ["不要瞬移", "不要新增人物正脸", "不要让角色换方向"]
  }
}
```

通用必填字段：

| 字段 | 要求 |
|---|---|
| `template` | 模板 ID，必须取本库模板；旧项目允许 legacy `multi_person_blocking` |
| `template_contract.template_id` | 与 `template` 完全一致 |
| `beats` | 2-6 个动作/表演节拍，按时间顺序写 |
| `blocking` | 人物左右/前后/距离/出入画方向 |
| `camera_rule` | 允许的机位和运镜边界 |
| `continuity_must` | 必须继承到出图/出视频的连续性约束 |
| `negative` | 该模板最容易翻车的负向约束 |

> gate 会在出图/出视频前检查：复杂镜头缺 `template/template_contract`，或模板字段不全，必须回 `/n2d-script` 补。

---

## 0.1 Motion Control 分级

涉及真实接触和遮挡的模板要先决定是否直接生成，不能留给视频 prompt 临场猜：

| 风险 | 适用模板 | 下游要求 |
|---|---|---|
| required | `fight_exchange`、`intimate_interaction`、`hug_or_pull` | `n2d-model-router` 会写 `motion_control.level=required`；出视频前必须有 `motion_control_manifest.json`，状态为 `ready` 或 `degrade_only` |
| recommended | `multi_character_same_frame`、`ensemble_blocking`、`multi_person_blocking`、`chase`、`flight` | 若曾失败或是关键高光镜，补 pose/depth 控制；否则按模板锁站位/方向并准备降级拆镜 |
| none | 普通单人反应、空镜、低幅度动作 | 不需要 Motion Control manifest |

`ready` manifest 代表已有 pose/depth/instance/contact 控制资产。`degrade_only` 代表不直接生成全身复杂接触，改成手部特写、过肩反打、反应镜、释放帧等可控短镜。OpenPose/DWPose 只能锁姿态；拥抱、抓腕、刀剑命中还要写清 `occlusion_order`、`body_part_ownership` 和接触点，必要时用 depth/instance masks/contact_map。

## 1. `fight_exchange` 打斗攻防

**使用场景**：拳脚、刀剑、掌法、法术近身命中、受击反应。完整武打拆招仍按 `打斗分镜.md`，本模板用于把 Clip 级契约写进 `storyboard.json`。

**拆法**：
- 起手：谁先动，力从哪里来。
- 发力：攻击路径清楚，不让 AI 自由编招。
- 命中/错身：命中帧或擦身帧单独锁图。
- 受击/反应：被打者只做一个明确反应。
- 收势：落到可接下一镜的姿态。

**额外必填字段**：`attack_path`、`impact_frame`、`action_scope`

**提示词规则**：多人混战拆成单人/双人正反打；命中类 Clip 优先首尾双帧；镜头运动只服务打击感，不乱飞。若出现刀剑/手掌/身体接触，出视频前必须 ready Motion Control 或 degrade_only 拆成起手、命中特写、受击反应。

---

## 2. `chase` 追逐

**使用场景**：追杀、逃跑、巷道/山路/洞穴追逐、飞行追逐。仙侠飞行追逐还要同步读 `仙侠场面分镜.md`。

**拆法**：
- 建立方向：先锁屏幕方向和地理关系。
- 距离变化：追近/拉开只能选一个主变化。
- 障碍节拍：门槛、石壁、雨帘、树枝等只放 1-2 个。
- 落幅：停到转弯、回头、跌倒、藏入暗处等可剪点。

**额外必填字段**：`screen_direction`、`distance_curve`、`obstacle_beats`

**提示词规则**：速度感来自背景流动、前景掠过、脚步/衣摆/雨丝，不来自人物变形；同一 Clip 不要同时写追逐、打斗、法术爆发三件事。

---

## 3. `dialogue_shot_reverse` 对话反打

**使用场景**：逼问、谈判、对峙、男女主互相试探、主仆/师徒/敌我正反打。普通台词不必都套；只有需要正反打、过肩、视线对位、权力关系变化时使用。

**拆法**：
- 建立轴线：谁在画左/画右，视线朝向哪里。
- A 面：说话者景别、情绪、压迫感。
- B 面：反打对象反应，不改变左右关系。
- 插入物/手部：必要时用茶杯、剑柄、符纸、手指动作缓冲。
- 落幅：停在眼神或反应，服务下一句台词。

**额外必填字段**：`axis`、`eyeline`、`shot_pairing`

**提示词规则**：反打不是随机换角度；同场景主光、背景深度、人物左右关系必须守住。若要越轴，必须在 `camera_rule` 写明确剧情理由和过渡镜。

---

## 4. `magic_burst` 法术爆发

**使用场景**：灵力爆发、符阵亮起、剑气冲出、雷落、火焰/光束/护盾炸开、斗法撞点。

**拆法**：
- 蓄力：颜色、形状、来源位置先锁。
- 释放：光效路径明确。
- 爆点/撞点：爆发帧或命中帧单独出图。
- 余波：尘、光点、衣摆、地面裂纹等二级动态。
- 落幅：爆后角色状态和场景状态。

**额外必填字段**：`charge_frame`、`release_frame`、`effect_asset`

**提示词规则**：高频复现法术/阵法/剑气必须进 `出图/共享/图片/特效定妆`；不要每镜换颜色、形状和轨迹。爆发只允许照亮既有场景，不新增无关奇观。

---

## 5. `flight` 飞行

**使用场景**：御剑、腾空、坠落、空中掠过、飞檐走壁、云海穿行。

**拆法**：
- 起飞/入画：姿态和方向先锁。
- 巡航：人物姿态尽量固定。
- 机动：只做一个转向、俯冲或急停。
- 抵达/落地：落到明确接点。

**额外必填字段**：`pose_lock`、`background_motion`、`altitude_path`

**提示词规则**：飞行核心是“锁姿态，动背景”。人物脸、发型、衣服不要被速度拉变形；速度感来自云、山、雨、前景掠过和镜头跟随。

---

## 6. `intimate_interaction` 亲密互动

**使用场景**：搀扶、牵手、抚脸、扶肩、疗伤、暧昧靠近、近距离低幅度互动。这里的“亲密”指人物距离与接触调度，不等于情色；若有拥抱、抓腕、推开、拉扯，改用 `hug_or_pull`。

**拆法**：
- 距离变化：远→近或近→分开，只选一个。
- 接触点：手腕、肩、背、额头、衣袖等必须写清。
- 视线/呼吸：用眼神、停顿、身体微动表现关系。
- 落幅：停在接触点或表情反应。

**额外必填字段**：`contact_points`、`distance_boundary`、`body_overlap_limit`

**提示词规则**：多角色接触最容易崩手和肢体重叠；优先用半身/手部特写、过肩、遮挡，而不是全身复杂拥抱。合规边界不清时回避情色化描述。直接生成近距离接触时，manifest 必须写 `contact_points`、`occlusion_order`、`body_part_ownership`。

---

## 7. `hug_or_pull` 拥抱 / 拉扯 / 抓腕

**使用场景**：拥抱、抱住、抓腕、拉袖、推开、拉扯、拽住、扶起时带明显力量方向的接触互动。它比普通亲密互动更容易崩手、穿模、肢体重叠，所以单独模板化。

**拆法**：
- 接触前：先锁两人左右、距离、手的位置。
- 接触点：抓腕/抱肩/拉袖/推胸口等只选一个主接触点。
- 力量方向：谁拉谁、往画左/画右/前景/后景移动，必须写清。
- 释放/停住：落到松手、被拉近、被推开、半步踉跄等可剪点。

**额外必填字段**：`contact_points`、`force_direction`、`body_overlap_limit`、`release_frame`

**提示词规则**：优先用首尾帧或手部特写承接，不要让 AI 在一个 Clip 里完成“冲上去、抓住、拥抱、挣扎、推开”全链路。全身拥抱不稳时，拆成靠近反应、手部接触、脸部反打。没有 ready Motion Control manifest 时，不直接生成全身拥抱/拉扯，写 `status=degrade_only` 并按拆镜执行。

---

## 8. `multi_character_same_frame` 多人同框

**使用场景**：两到三名具名角色必须同时清楚出现在同一画面：双人同框、三人同框、主角与两名关键配角同场对峙、同桌/同车/同殿但需要看清主角脸。

**拆法**：
- 角色槽位：每个具名角色固定画左/画右/中央/前景/后景。
- 脸优先级：只允许 1-2 张脸作为清晰焦点，其余侧脸/背影/虚焦。
- 遮挡规则：谁可以被前景遮挡，谁不能被挡脸。
- 焦点切换：若要 A→B 反应，优先拆正反打，不在同框里乱转。

**额外必填字段**：`character_slots`、`face_priority`、`overlap_rules`

**提示词规则**：多人同框的目标是“槽位稳定”，不是让每个人都完整表演。角色身份从 `identity_registry` 继承；如果出现脸互换、左右互换、服装串色，必须拆成两人 OTS / 反打 / 手部插入。

---

## 9. `ensemble_blocking` 群像站位

**使用场景**：三人以上群戏、群臣/门徒/追兵围堵、宴席、审讯、围观、队伍站位、宗门大殿群像。重点是主次层级和空间关系，不要求人人清脸。

**拆法**：
- 建立层级：主角、对手、群体分别在前/中/后景。
- 站位图：画左/画右/中央/前景/背景写清。
- 焦点层级：一次 Clip 只突出一个说话者或动作点。
- 人群简化：背景人用剪影/虚焦/背影/局部，别要求人人清脸。

**额外必填字段**：`screen_positions`、`focus_hierarchy`、`crowd_simplification`

**提示词规则**：三人以上不追求每张脸都精确；主要人物用定妆参考，次要人物降级为虚焦、剪影或局部。若主角与两名以上角色都要清脸互动，优先拆正反打。

---

## 10. `multi_person_blocking` 多人站位（legacy）

旧项目已写 `multi_person_blocking` 的 Clip 仍可通过 gate；新项目优先按镜头目的拆成：

- 2-3 名具名角色同框：`multi_character_same_frame`
- 3 人以上群戏 / 队列 / 围堵 / 群像：`ensemble_blocking`

**额外必填字段（legacy）**：`screen_positions`、`speaker_focus`、`crowd_simplification`

---

## 11. 选择速查

| 镜头意图 | 首选模板 | 备注 |
|---|---|---|
| 两人拳脚/刀剑/掌法 | `fight_exchange` | 同步读 `打斗分镜.md` |
| 逃跑/追杀/移动压力 | `chase` | 飞行追逐同时读 `仙侠场面分镜.md` |
| 逼问/谈判/眼神压迫 | `dialogue_shot_reverse` | 只在需要轴线反打时套 |
| 符阵/灵力/光束/雷落 | `magic_burst` | 高频特效入库 |
| 御剑/腾空/坠落 | `flight` | 锁姿态动背景 |
| 搀扶/牵手/疗伤 | `intimate_interaction` | 明确接触点和肢体边界 |
| 拥抱/抓腕/推开/拉扯 | `hug_or_pull` | 明确接触点、力量方向、释放帧 |
| 两到三名具名角色同框 | `multi_character_same_frame` | 固定角色槽位和脸优先级 |
| 三人以上群戏/队列/围堵 | `ensemble_blocking` | 主次分层，背景简化 |
| 旧项目多人站位 | `multi_person_blocking` | legacy 兼容；新项目少用 |

**收尾自查**：如果一句 prompt 里同时出现三类以上复杂动作，先拆 Clip；如果需要解释“AI 应该自己理解怎么打/怎么追/怎么站”，说明模板契约还没写够。
