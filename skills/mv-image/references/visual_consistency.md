# MV 单曲视觉一致性包

MV 不需要像 n2d 那样承担多集连续剧的严格跨集一致性，但一支歌内部仍要让观众感到“这是同一个世界、同一个主角、同一套视觉母题”。本文件把 n2d 的一致性经验改写成 MV 版：更宽松、更强调段落情绪和卡点，但不放弃身份锚点。

## 适用范围

- **必须锁**：主角/主唱脸、发型、核心服装轮廓、主色调、视觉风格、反复出现的道具/符号。
- **允许变**：副歌爆发光效、段落换景、色彩强弱、服装外层/妆造细节、转场特效。
- **不追求**：跨集状态账本、长期角色成长、每个场景几何完全一致。

## 视觉一致性包字段

在 `视觉蓝图.md` 或 `设定/` 卡片里尽量明确以下内容，mv-plan / mv-image / mv-video 都要继承：

| 字段 | 用途 |
|---|---|
| `lead_identity_anchor` | 主角/主唱不可漂特征：脸型、发型、服装轮廓、标志配饰 |
| `global_style` | 整支 MV 的画风、镜头质感、颗粒、色彩语法 |
| `palette_anchor` | 2-4 个主色，允许段落内加亮/变暗，不允许整支 MV 跳风格 |
| `section_look` | 每个段落的场景、光位、服装变体和情绪强度 |
| `motif_ledger` | 反复出现的视觉母题：镜子、雨、红线、剑光、霓虹、花瓣、火焰等 |
| `reference_inputs` | 用户指定参考图、后端主体/角色 ID、LoRA 卡等一致性增强资产 |
| `forbidden_drift` | 明确禁止漂移项：换脸、换发色、换衣型、主道具变形、场景风格突变 |

## 三档一致性要求

| 类型 | 要求 | 典型做法 |
|---|---|---|
| 主角/主唱 | 最严格 | 共享定妆 + 锚点句 + 同一生图后端；可用 Seedream/可灵/Sora 主体能力 |
| 段落场景 | 中等 | 同段落继承场景定妆、主色、光位；换段可换景但保留母题 |
| 特效/转场 | 宽松 | 只锁颜色/形状方向，不要求每次粒子完全一样 |

## 出图 prompt 写法

分镜 prompt 末尾固定拼三类锚点：

```text
身份锚点：<lead_identity_anchor>
参考输入：<reference_inputs；无则写 共享定妆+锚点>
视觉锚点：<global_style + palette_anchor>
母题锚点：<motif_ledger 中本段需要出现的符号>
禁止漂移：<forbidden_drift>
```

副歌可以让光效、粒子、镜头角度更夸张，但身份锚点和视觉锚点不变。不要为了“炫”把主角脸、服装轮廓和画风一起换掉。

## 与 n2d 的区别

- n2d 强调跨集状态、角色成长和剧情连续性；MV 强调单曲内的视觉记忆点和段落情绪。
- n2d 的尾帧接力更硬；MV 默认卡点硬切，只有同段落连续动作才用 `_end.png`。
- n2d 的角色 registry 可长期积累；MV 可以用轻量设定卡 + 共享定妆图解决大多数一致性。

## 出图前检查

- [ ] 主角/主唱身份锚点已写进角色卡和每张分镜 prompt？
- [ ] 已向用户提示是否使用指定参考图、后端主体库或 LoRA？
- [ ] 若启用 LoRA，`.safetensors`、trigger、底模和授权说明都已登记？
- [ ] 整支 MV 只用一个官方生图后端？
- [ ] 每个段落的 `section_look` 清楚：场景、光色、服装变体、情绪强度？
- [ ] 反复出现的道具/符号已进 `motif_ledger`？
- [ ] 副歌高能镜只增强光效/机位/动作，没有换脸换风格？
- [ ] `need_end_frame=true` 的连续动作镜已准备尾帧？
